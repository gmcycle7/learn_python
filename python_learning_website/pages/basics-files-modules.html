Files & Modules

This page covers two essential skills: working with files on disk (reading and writing data) and organising your code with modules and imports. We also cover try / except for error handling — because dealing with files is one of the most common places where things can go wrong. By the end of this page you will be able to read and write text files, handle errors gracefully, and structure your Python projects using modules.

Note for MATLAB users: MATLAB uses fopen, fread, fprintf, and fclose for file I/O. Python's approach is similar in spirit but uses different syntax — and the with statement eliminates the need to remember to call fclose. For modules, MATLAB relies on addpath and placing functions in separate .m files. Python uses import, which is more explicit and powerful.

1. Reading Files

The open() Function and File Modes

Python's built-in open() function is the gateway to file operations. It returns a file object that you use to read from or write to the file. The second argument is the mode:

Mode Description File must exist?
"r" Read (default). Opens file for reading. Yes
"w" Write. Creates file or overwrites existing content. No (creates it)
"a" Append. Writes to end of file without erasing existing content. No (creates it)
"r+" Read and write. File must already exist. Yes

Reading the Entire File with .read()

# Read the entire file into a single string
with open("data.txt", "r") as f:
    content = f.read()

print(content)

The .read() method returns the entire file as one string. This is fine for small files, but be careful with very large files — they will consume a lot of memory.

Reading Line by Line

Python gives you several ways to read files line by line: 

# Method 1: .readline() — reads one line at a time
with open("data.txt", "r") as f:
    first_line = f.readline()   # reads line 1
    second_line = f.readline()  # reads line 2
    print(first_line.strip())   # .strip() removes the trailing newline
    print(second_line.strip())
# Method 2: .readlines() — returns a list of all lines
with open("data.txt", "r") as f:
    lines = f.readlines()       # e.g. ["line 1\n", "line 2\n", "line 3\n"]

for line in lines:
    print(line.strip())
# Method 3 (BEST): iterate the file object directly
with open("data.txt", "r") as f:
    for line in f:
        print(line.strip())
Best practice: Method 3 (iterating directly) is the most Pythonic and memory-efficient approach. It reads one line at a time without loading the entire file into memory, making it safe for files of any size.

The with Statement (Context Manager)

You should always use the with statement when working with files. It guarantees that the file is properly closed when the block ends, even if an error occurs: 

# GOOD — with statement (file closes automatically)
with open("data.txt", "r") as f:
    content = f.read()
# f is now closed — no need to call f.close()

# BAD — manual open/close (easy to forget, error-prone)
f = open("data.txt", "r")
content = f.read()
f.close()  # What if an error happens before this line?
Common mistake: Forgetting to close files. If you open many files without closing them, you can run out of file handles and your program will crash. The with statement prevents this entirely. Always use with.
MATLAB comparison: In MATLAB you write fid = fopen('data.txt', 'r') and must remember to call fclose(fid) when done. Python's with statement is like an automatic fclose — it handles cleanup for you, even if your code throws an error.

Encoding

By default, Python uses your system's default encoding. To be explicit (and avoid surprises across platforms), specify the encoding: 

# Specify encoding explicitly — recommended for portability
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()

This is especially important when dealing with files that contain non-ASCII characters (accented letters, CJK characters, emoji, etc.).

2. Writing Files

Writing with "w" Mode (Overwrites!)

# Write mode — creates a new file or OVERWRITES existing content
with open("output.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("This is line 2.\n")
Warning: Opening a file with "w" mode erases all existing content the moment the file is opened. If the file contained important data, it is gone. Always double-check the filename and mode before running your script.

Appending with "a" Mode

# Append mode — adds to the end without erasing
with open("log.txt", "a") as f:
    f.write("New log entry: everything is fine.\n")

Append mode is ideal for log files, running records, or any situation where you want to add data without destroying what is already there.

.write() vs .writelines()

# .write() takes a single string
with open("output.txt", "w") as f:
    f.write("Line 1\n")
    f.write("Line 2\n")

# .writelines() takes a list of strings (does NOT add newlines!)
lines = ["Line 1\n", "Line 2\n", "Line 3\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)
Note: .writelines() does not automatically add newline characters between items. You must include \n in each string yourself.

Writing Formatted Data

# Writing formatted output to a file
students = [
    ("Alice", 92),
    ("Bob", 85),
    ("Charlie", 78),
]

with open("grades.txt", "w") as f:
    f.write("Name        Grade\n")
    f.write("-" * 20 + "\n")
    for name, grade in students:
        f.write(f"{name:<12}{grade}\n")

# grades.txt will contain:
# Name        Grade
# --------------------
# Alice       92
# Bob         85
# Charlie     78
MATLAB comparison: In MATLAB you would use fprintf(fid, '%s %d\n', name, grade). Python's f-strings (f"...") serve the same purpose and are often more readable. The < in {name:<12} means left-align in a 12-character field — similar to MATLAB's %-12s.

A Complete Read-Process-Write Example

# Read numbers from a file, compute stats, write results
with open("numbers.txt", "r") as f:
    numbers = [float(line.strip()) for line in f if line.strip()]

total = sum(numbers)
average = total / len(numbers)
maximum = max(numbers)
minimum = min(numbers)

with open("stats.txt", "w") as f:
    f.write(f"Count:   {len(numbers)}\n")
    f.write(f"Sum:     {total:.2f}\n")
    f.write(f"Average: {average:.2f}\n")
    f.write(f"Max:     {maximum:.2f}\n")
    f.write(f"Min:     {minimum:.2f}\n")

print("Statistics written to stats.txt")

3. try / except — Error Handling

Why Error Handling Matters

When your code interacts with the outside world — reading files, getting user input, connecting to networks — things can go wrong. The file might not exist. The user might type letters when you expect a number. The network might be down. Without error handling, your program crashes with an ugly traceback. With error handling, you can respond gracefully.

Basic try / except Syntax

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")

Output:

Cannot divide by zero!

The code inside try runs normally. If an exception occurs, Python jumps to the matching except block instead of crashing.

MATLAB comparison: MATLAB uses try / catch. Python uses try / except. The concept is identical — only the keyword differs: 
% MATLAB
try
    result = 10 / 0;
catch ME
    disp(ME.message);
end
# Python
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(e)

Catching Specific Exceptions

Always catch specific exceptions rather than a bare except:. This prevents you from accidentally hiding bugs: 

# GOOD — catch specific exceptions
try:
    with open("config.txt", "r") as f:
        data = f.read()
except FileNotFoundError:
    print("config.txt not found. Using defaults.")

# BAD — bare except catches EVERYTHING (including typos and bugs!)
try:
    with open("config.txt", "r") as f:
        data = f.read()
except:          # Don't do this!
    print("Something went wrong.")

Multiple except Blocks

try:
    value = int(input("Enter a number: "))
    result = 100 / value
    print(f"100 / {value} = {result}")
except ValueError:
    print("That's not a valid number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")
except Exception as e:
    print(f"Unexpected error: {e}")

Python checks each except block from top to bottom and runs the first one that matches. Put more specific exceptions first, and more general ones last.

The else and finally Clauses

try:
    with open("data.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("File not found!")
else:
    # Runs ONLY if no exception occurred
    print(f"File has {len(content)} characters.")
finally:
    # Runs ALWAYS, whether or not an exception occurred
    print("File operation attempted.")
Clause When it runs Typical use
try Always (this is the code you are "trying") The risky operation
except Only if an exception occurs in try Handle the error
else Only if no exception occurred Process the result on success
finally Always, no matter what Cleanup (close connections, etc.)

Common Exceptions to Know

Exception When it happens Example
FileNotFoundError File does not exist open("nope.txt")
ValueError Wrong type of value int("hello")
TypeError Wrong type for operation "2" + 2
ZeroDivisionError Division by zero 10 / 0
IndexError List index out of range [1, 2, 3][5]
KeyError Dictionary key not found {"a": 1}["b"]
PermissionError No permission to access file Opening a system-protected file

When to Use try / except (Don't Overuse It)

Error handling is powerful, but don't wrap every line of code in try / except. Use it when:

  • You are doing file I/O (files may not exist, disks may be full)
  • You are parsing user input (users type unexpected things)
  • You are making network requests (connections can fail)
  • You are converting data types (int(), float() can fail)

Don't use try / except to hide bugs. If your code has a logic error, fix the code rather than catching the exception and ignoring it.

Practical Example: Safe File Reader

def read_file_safely(filename):
    """Read a file and return its content, or None if it fails."""
    try:
        with open(filename, "r", encoding="utf-8") as f:
            return f.read()
    except FileNotFoundError:
        print(f"Error: '{filename}' not found.")
        return None
    except PermissionError:
        print(f"Error: No permission to read '{filename}'.")
        return None
    except UnicodeDecodeError:
        print(f"Error: '{filename}' contains invalid UTF-8.")
        return None

# Usage
content = read_file_safely("config.txt")
if content is not None:
    print(f"Read {len(content)} characters.")

4. Importing Modules

What is a Module?

A module is simply a .py file that contains Python code — functions, classes, variables, or runnable statements. Python comes with a large standard library of modules, and you can install thousands more from the internet. Modules let you reuse code without copying and pasting.

Three Ways to Import

# Way 1: import the entire module
import math
print(math.sqrt(16))    # 4.0
print(math.pi)          # 3.141592653589793

# Way 2: import specific items
from math import sqrt, pi
print(sqrt(16))         # 4.0
print(pi)               # 3.141592653589793

# Way 3: import with an alias
import numpy as np      # common convention
arr = np.array([1, 2, 3])
print(arr * 2)          # [2 4 6]
Which style should you use?
  • import module — Best for clarity. You always see where each function comes from.
  • from module import name — Good when you use a function many times and typing the module prefix gets tedious.
  • import module as alias — Use well-known aliases: np for numpy, pd for pandas, plt for matplotlib.pyplot.

Avoid from module import * — it dumps everything into your namespace and can cause name collisions.

MATLAB comparison: In MATLAB, all functions on the path are automatically available — there is no import step. You use addpath to add directories. In Python, you must explicitly import what you need. This is more verbose but prevents name collisions and makes dependencies clear. 
% MATLAB — functions are available if on the path
addpath('my_tools/');
result = my_function(42);

# Python — you must import explicitly
from my_tools import my_function
result = my_function(42)

What Does __name__ == "__main__" Mean?

You will often see this pattern at the bottom of Python scripts: 

# my_script.py

def greet(name):
    return f"Hello, {name}!"

def main():
    print(greet("World"))

if __name__ == "__main__":
    main()

Here is what is happening:

  • Every Python file has a built-in variable called __name__.
  • When you run a file directly (e.g., python my_script.py), Python sets __name__ to "__main__".
  • When you import a file as a module (e.g., import my_script), Python sets __name__ to the module's name ("my_script").

The if __name__ == "__main__": guard means: "Only run this code if I am being executed directly, not imported." This lets the same file work as both a reusable module and a standalone script. 

# Without the guard, importing would print "Hello, World!"
# unintentionally. The guard prevents that.

# Run directly:    python my_script.py  → prints "Hello, World!"
# Import it:       import my_script     → nothing happens (good!)

5. Creating Your Own Modules

Any .py File is a Module

Creating a module is as simple as saving a .py file. Suppose you create a file called helpers.py: 

# helpers.py — your custom module

def celsius_to_fahrenheit(c):
    """Convert Celsius to Fahrenheit."""
    return c * 9 / 5 + 32

def fahrenheit_to_celsius(f):
    """Convert Fahrenheit to Celsius."""
    return (f - 32) * 5 / 9

PI = 3.14159265358979

Now in another file in the same directory, you can import and use it: 

# main.py — uses the helpers module

from helpers import celsius_to_fahrenheit, PI

temp_f = celsius_to_fahrenheit(100)
print(f"100 C = {temp_f} F")     # 100 C = 212.0 F
print(f"PI = {PI}")              # PI = 3.14159265358979
# Or import the entire module
import helpers

temp_c = helpers.fahrenheit_to_celsius(72)
print(f"72 F = {temp_c:.1f} C")  # 72 F = 22.2 C

The __name__ Guard in Your Own Modules

Add the guard to make your module both importable and runnable: 

# helpers.py — with a __name__ guard

def celsius_to_fahrenheit(c):
    return c * 9 / 5 + 32

def fahrenheit_to_celsius(f):
    return (f - 32) * 5 / 9

if __name__ == "__main__":
    # This only runs when you execute: python helpers.py
    # It does NOT run when someone does: import helpers
    print("Running tests...")
    assert celsius_to_fahrenheit(0) == 32
    assert celsius_to_fahrenheit(100) == 212
    assert fahrenheit_to_celsius(32) == 0
    print("All tests passed!")

This is a common pattern: put test code or demo code inside the __name__ guard, and keep the rest of the file clean for importing.

Organising a Small Project

Here is how a typical small Python project might be structured: 

my_project/
    main.py          # entry point — run this
    helpers.py       # utility functions
    file_io.py       # file reading/writing functions
    data/
        input.txt
        output.txt
# main.py
from helpers import celsius_to_fahrenheit
from file_io import read_temperatures, write_results

temperatures = read_temperatures("data/input.txt")
converted = [celsius_to_fahrenheit(t) for t in temperatures]
write_results("data/output.txt", converted)
print("Done!")
# file_io.py
def read_temperatures(filename):
    """Read a file of temperatures (one per line) and return as list of floats."""
    with open(filename, "r") as f:
        return [float(line.strip()) for line in f if line.strip()]

def write_results(filename, temperatures):
    """Write temperatures to a file, one per line."""
    with open(filename, "w") as f:
        for temp in temperatures:
            f.write(f"{temp:.1f}\n")
MATLAB comparison: In MATLAB, each public function lives in its own .m file (e.g., celsius_to_fahrenheit.m). In Python, you can put many functions in a single .py file and import just the ones you need. This gives you more flexibility in how you organise your code.

Putting It All Together

Here is a complete example that combines file reading, error handling, modules, and writing output: 

# analyze_scores.py — a complete mini-program

import os
from datetime import datetime

def load_scores(filename):
    """Load student scores from a file. Each line: 'Name,Score'"""
    scores = {}
    try:
        with open(filename, "r", encoding="utf-8") as f:
            for line_num, line in enumerate(f, start=1):
                line = line.strip()
                if not line:
                    continue
                try:
                    name, score = line.split(",")
                    scores[name.strip()] = float(score.strip())
                except ValueError:
                    print(f"Warning: skipping malformed line {line_num}: '{line}'")
    except FileNotFoundError:
        print(f"Error: '{filename}' not found.")
    return scores

def compute_stats(scores):
    """Compute basic statistics from a dict of scores."""
    if not scores:
        return None
    values = list(scores.values())
    return {
        "count": len(values),
        "average": sum(values) / len(values),
        "highest": max(values),
        "lowest": min(values),
        "top_student": max(scores, key=scores.get),
    }

def write_report(filename, stats):
    """Write a statistics report to a file."""
    with open(filename, "w", encoding="utf-8") as f:
        f.write(f"Report generated: {datetime.now().strftime('%Y-%m-%d %H:%M')}\n")
        f.write("=" * 40 + "\n")
        f.write(f"Students:     {stats['count']}\n")
        f.write(f"Average:      {stats['average']:.1f}\n")
        f.write(f"Highest:      {stats['highest']:.1f}\n")
        f.write(f"Lowest:       {stats['lowest']:.1f}\n")
        f.write(f"Top student:  {stats['top_student']}\n")
    print(f"Report written to '{filename}'.")

if __name__ == "__main__":
    scores = load_scores("scores.csv")
    if scores:
        stats = compute_stats(scores)
        write_report("report.txt", stats)
    else:
        print("No scores to analyse.")

Exercise 6: File Copier with Error Handling

Write a function copy_file(source, destination) that copies the content of one text file to another. Handle errors for missing source file and permission problems.

Show Solution 
def copy_file(source, destination):
    """Copy text content from source to destination."""
    try:
        with open(source, "r", encoding="utf-8") as src:
            content = src.read()
    except FileNotFoundError:
        print(f"Error: source file '{source}' not found.")
        return False
    except PermissionError:
        print(f"Error: no permission to read '{source}'.")
        return False

    try:
        with open(destination, "w", encoding="utf-8") as dst:
            dst.write(content)
    except PermissionError:
        print(f"Error: no permission to write to '{destination}'.")
        return False

    print(f"Copied '{source}' to '{destination}' ({len(content)} characters).")
    return True

# Usage
copy_file("original.txt", "backup.txt")

            f.write(f"{name:<15} {grade:6.1f}  {status}\n")
        f.write("=" * 30 + "\n")
        f.write(f"Class average: {average:.1f}\n")
        f.write(f"Highest: {max(grades):.1f}\n")
        f.write(f"Lowest:  {min(grades):.1f}\n")

    print(f"Report written to '{output_file}'.")

process_grades("students.csv", "grade_report.txt")
