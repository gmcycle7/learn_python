Section B — Getting Started with Python

Before you can write a single line of Python, you need Python installed on your machine and a basic understanding of how to run code. This section walks you through every step — installing Python, verifying the installation, running code interactively and from files, managing virtual environments, and installing third-party packages. By the end, you will have a fully working Python development setup.

Note for MATLAB Users

In MATLAB, everything comes bundled: you install MATLAB and you immediately have an editor, a command window, a path manager, and every toolbox you purchased. Python is different. Python itself is just the language interpreter. You choose your own editor (VS Code, PyCharm, etc.), manage packages separately with pip, and organize project dependencies with virtual environments. This flexibility is powerful, but it means there are a few setup steps before you start coding. Don't worry — once you do it once, it becomes second nature.

1. How to Install Python

Python is free and open-source. The official distribution is available at python.org/downloads. Always download the latest stable release (Python 3.12 or newer as of this writing). Do not install Python 2 — it reached end-of-life in January 2020.

Windows

  1. Go to python.org/downloads.
  2. Click the large yellow "Download Python 3.x.x" button. This downloads the Windows installer (.exe file).
  3. Run the downloaded installer. You will see a window titled "Install Python 3.x.x" with two checkboxes at the bottom.
  4. CRITICAL: Check the box that says "Add python.exe to PATH" before clicking anything else. This checkbox is at the very bottom of the installer window and is unchecked by default.
  5. Click "Install Now" (the recommended option). The installer will show a progress bar.
  6. When you see "Setup was successful", click Close.
  7. Open a new Command Prompt or PowerShell window (existing windows won't see the new PATH) and type python --version to verify.

Warning — "Add to PATH" on Windows

If you forget to check "Add python.exe to PATH", Windows will not recognize the python command in your terminal. You would need to either re-run the installer and select Modify to add it, or manually add Python's install directory (e.g., C:\Users\YourName\AppData\Local\Programs\Python\Python312\) to your system's PATH environment variable. Save yourself the headache — always check that box.

macOS

  1. Go to python.org/downloads.
  2. Click the "Download Python 3.x.x" button. This downloads a .pkg installer.
  3. Open the .pkg file. You will see a standard macOS installer wizard. Click Continue through the introduction, license agreement (click Agree), and destination selection.
  4. Click Install. You may be asked for your administrator password.
  5. When the installer finishes, it opens a Finder window showing the Python folder. You can close this.
  6. Open Terminal (found in Applications > Utilities) and type python3 --version to verify.

Note: macOS may come with an older system Python (Python 2 on older versions, or a stub that prompts you to install developer tools). Always use the python3 command on macOS to ensure you are running Python 3.

Alternative with Homebrew:

# Install Homebrew first if you don't have it:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Then install Python:
brew install python

Linux

Most Linux distributions come with Python 3 pre-installed. Check first:

python3 --version

If Python 3 is not installed, or you want a newer version, use your distribution's package manager:

# Ubuntu / Debian
sudo apt update
sudo apt install python3 python3-pip python3-venv

# Fedora
sudo dnf install python3 python3-pip

# Arch Linux
sudo pacman -S python python-pip

Tip — Which Python Version?

Always install the latest stable Python 3 release. As of this writing, Python 3.12+ is recommended. Avoid Python 3.6 or older, as many modern libraries have dropped support for them. You can check which versions are currently supported at devguide.python.org/versions.

2. How to Check Your Python Version

After installation, open a terminal (Command Prompt or PowerShell on Windows, Terminal on macOS/Linux) and run one of the following commands: 

# Check the Python version
python --version
# Output: Python 3.12.4

# On macOS/Linux you may need to use python3
python3 --version
# Output: Python 3.12.4

If you get an error like 'python' is not recognized as an internal or external command on Windows, Python is not on your PATH. Revisit the installation step above.

Find Where Python Is Installed

# Windows — find the Python executable location
where python
# Output: C:\Users\YourName\AppData\Local\Programs\Python\Python312\python.exe

# macOS / Linux — find the Python executable location
which python3
# Output: /usr/local/bin/python3

These commands are useful for debugging when you have multiple Python versions installed and want to confirm which one your terminal is using.

Check pip Version Too

# Check pip version
pip --version
# Output: pip 24.0 from ...site-packages/pip (python 3.12)

# Or on macOS/Linux
pip3 --version

3. How to Run Python from the Terminal (Interactive Mode)

Python has a built-in interactive interpreter, often called the REPL (Read-Eval-Print Loop). This is similar to the MATLAB Command Window — you type an expression, press Enter, and immediately see the result.

Starting the Interactive Interpreter

# Start interactive Python
python

# Or on macOS/Linux
python3

You will see something like this:

Python 3.12.4 (main, Jun  7 2024, 00:00:00) [GCC 14.1.1] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

The >>> prompt means Python is waiting for your input. Try typing some expressions: 

>>> 2 + 3
5
>>> "Hello, " + "World!"
'Hello, World!'
>>> 10 / 3
3.3333333333333335
>>> 2 ** 10
1024
>>> type(42)
<class 'int'>
>>> import math
>>> math.sqrt(144)
12.0
>>> [x ** 2 for x in range(5)]
[0, 1, 4, 9, 16]

Exiting the Interactive Interpreter

There are several ways to exit:

>>> exit()       # Built-in function — works everywhere
>>> quit()       # Also works everywhere
# Or use a keyboard shortcut:
# Ctrl+Z then Enter  (Windows)
# Ctrl+D             (macOS / Linux)

Interactive Mode vs. Scripts

The interactive interpreter is great for quick experiments and testing small snippets, much like MATLAB's Command Window. However, for anything beyond a few lines, you should write your code in a .py file (a "script") and run it from the terminal. We cover this next.

4. How to Run a .py File

Real Python development happens in .py files — plain text files containing Python code. Here is the complete workflow:

Step 1: Create a Python File

Open any text editor (VS Code, Notepad++, Sublime Text, or even plain Notepad) and type the following: 

# hello.py — My first Python script

name = "World"
print(f"Hello, {name}!")
print("Welcome to Python.")
print(f"2 + 3 = {2 + 3}")

Save this file as hello.py in a folder you can easily navigate to (e.g., C:\Users\YourName\projects\hello.py on Windows or ~/projects/hello.py on macOS/Linux).

Warning — File Extension Matters

Make sure the file is saved with the .py extension, not .py.txt. If you are using Windows Notepad, select "All Files (*.*)" in the "Save as type" dropdown, then type the filename as hello.py. Better yet, use a proper code editor like VS Code (free).

Step 2: Navigate to the File's Folder

# Windows
cd C:\Users\YourName\projects

# macOS / Linux
cd ~/projects

Step 3: Run the Script

# Windows
python hello.py

# macOS / Linux
python3 hello.py

You should see this output:

Hello, World!
Welcome to Python.
2 + 3 = 5

The Workflow Summary

  1. Edit your .py file in a text editor.
  2. Save the file.
  3. Switch to your terminal.
  4. Run with python filename.py.
  5. Read the output (or error messages).
  6. Repeat — go back to step 1 and make changes.

This edit-save-run cycle is the fundamental Python workflow. Editors like VS Code let you run scripts with a single keypress, but under the hood, it is still doing the same thing.

5. How to Use Virtual Environments

Why This Matters

Imagine you have two projects: Project A needs numpy 1.24 and Project B needs numpy 1.26. If you install packages globally (system-wide), you can only have one version of numpy at a time, and switching between projects becomes a nightmare. Virtual environments solve this problem. Each project gets its own isolated folder of packages, so Project A and Project B can each have exactly the versions they need, without any conflicts.

Think of a virtual environment as giving each project its own private copy of Python and its packages. It is one of the most important habits to develop as a Python programmer. Always use a virtual environment for every project.

Creating a Virtual Environment

Python 3 includes the venv module out of the box (no installation needed). Navigate to your project folder and run: 

# Create a virtual environment named "myenv"
python -m venv myenv

# Or on macOS/Linux
python3 -m venv myenv

This creates a new folder called myenv inside your project directory. This folder contains a copy of the Python interpreter, a pip executable, and a site-packages directory where installed packages will go. You typically name it venv, .venv, or env — any name works.

Activating the Virtual Environment

Creating the environment is not enough — you must activate it so that your terminal uses the environment's Python and pip instead of the global ones. 

# Windows (Command Prompt)
myenv\Scripts\activate

# Windows (PowerShell)
myenv\Scripts\Activate.ps1

# Windows (Git Bash)
source myenv/Scripts/activate

# macOS / Linux
source myenv/bin/activate

When the environment is active, you will see the environment name in parentheses at the beginning of your terminal prompt: 

(myenv) C:\Users\YourName\projects>
(myenv) user@machine:~/projects$

While the environment is active, python and pip refer to the versions inside the virtual environment — not the global ones.

Deactivating the Virtual Environment

# Works on all platforms — just type:
deactivate

The (myenv) prefix disappears, and your terminal goes back to using the global Python.

Complete Virtual Environment Workflow

# 1. Create a project folder
mkdir my_project
cd my_project

# 2. Create a virtual environment
python -m venv venv

# 3. Activate it
# Windows:   venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

# 4. Install packages (they go into the venv only)
pip install numpy requests

# 5. Do your work...
python my_script.py

# 6. When done, deactivate
deactivate

Warning — Don't Commit Your venv Folder

The virtual environment folder (venv/, myenv/, etc.) can be hundreds of megabytes and is specific to your operating system. Never add it to version control (Git). Instead, add it to your .gitignore file. Others can recreate the environment using your requirements.txt (covered in the next section).

MATLAB Comparison

MATLAB does not have virtual environments. When you install a toolbox, it is available to every MATLAB project on your machine. This is simpler but can cause version conflicts. Python's virtual environment system gives you fine-grained control over dependencies per project — a major advantage for reproducibility and collaboration.

6. How to Install Packages with pip

Python's standard library is extensive, but the real power of the ecosystem comes from third-party packages — libraries written by the community and published on PyPI (the Python Package Index). There are over 500,000 packages available. You install them using pip, Python's package installer.

Installing a Package

# Make sure your virtual environment is active first!

# Install a single package
pip install numpy

# Install a specific version
pip install numpy==1.26.4

# Install multiple packages at once
pip install numpy requests matplotlib pandas

pip will download the package from PyPI and install it (along with any dependencies it needs). You will see output showing the download progress and a success message like Successfully installed numpy-1.26.4.

Listing Installed Packages

# See all installed packages and their versions
pip list

# Example output:
# Package    Version
# ---------- -------
# numpy      1.26.4
# pip        24.0
# requests   2.31.0
# setuptools 69.5.1

Freezing Dependencies to requirements.txt

# Export exact versions of all installed packages
pip freeze > requirements.txt

# View the file
cat requirements.txt
# Output:
# certifi==2024.2.2
# charset-normalizer==3.3.2
# numpy==1.26.4
# requests==2.31.0
# urllib3==2.2.1

The requirements.txt file is a plain text list of packages with pinned versions. Share this file with collaborators or include it in your Git repository so anyone can recreate your exact environment: 

# Someone else (or you on a new machine) can install everything with:
pip install -r requirements.txt

Upgrading and Uninstalling Packages

# Upgrade a package to the latest version
pip install --upgrade numpy

# Upgrade pip itself
pip install --upgrade pip

# Uninstall a package
pip uninstall requests

Searching for Packages

To find packages, browse pypi.org directly in your web browser. Some popular packages you will encounter as a MATLAB user transitioning to Python:

7. Common Beginner Mistakes in Setup

Almost every Python beginner hits one or more of these issues. Read through them now so you can avoid hours of frustration later.

Mistake 1 — Not Adding Python to PATH (Windows)

Symptom: You type python in the terminal and get 'python' is not recognized as an internal or external command'.

Fix: Re-run the Python installer, choose Modify, and make sure the PATH option is checked. Alternatively, use the full path to python.exe (e.g., C:\Users\YourName\AppData\Local\Programs\Python\Python312\python.exe), but this is tedious.

Mistake 2 — Using the Wrong Python Version

Symptom: Code that uses modern features (f-strings, walrus operator, match statements) throws SyntaxError.

Fix: Run python --version to check. If you see Python 2.x, you need to install Python 3 and use the python3 command. If you see Python 3.6 or older, upgrade to a current version.

Mistake 3 — Mixing pip and pip3

Symptom: You install a package with pip install numpy but when you run your script with python3, you get ModuleNotFoundError: No module named 'numpy'.

Fix: pip might be linked to Python 2 on some systems while pip3 is linked to Python 3. The safest way to install packages is: python3 -m pip install numpy. This guarantees the package is installed for the same Python that you run.

Mistake 4 — Installing Packages Globally Instead of in a Virtual Environment

Symptom: Package version conflicts between projects. System Python breaks after a bad install. Permission errors (ERROR: Could not install packages due to an EnvironmentError).

Fix: Always create and activate a virtual environment before running pip install. Check that your prompt shows (venv) before installing.

Mistake 5 — Forgetting to Activate the Virtual Environment

Symptom: You created a virtual environment, installed packages into it, but your script cannot find them.

Fix: Make sure you see (venv) or (myenv) in your terminal prompt. If not, run the activate command again. You must activate the environment every time you open a new terminal window.

Mistake 6 — Running python Instead of python3 on macOS/Linux

Symptom: On some macOS/Linux systems, python points to Python 2 (or does not exist), while python3 is the correct command.

Fix: Always use python3 and pip3 on macOS/Linux unless you are inside a virtual environment (where python is always correct).

Mistake 7 — Editing Code but Running the Old Version

Symptom: You change your code but the output does not change.

Fix: Make sure you saved the file before running it again. This is a surprisingly common oversight, especially for MATLAB users who are accustomed to the editor auto-saving before running.

8. Difference Between python, python3, pip, pip3, venv, and conda

These terms can be confusing when you are just starting out. Here is a clear comparison:

Command / Tool What It Is When to Use It Notes
python The Python interpreter Running scripts, starting the REPL On Windows, this is typically Python 3. On macOS/Linux, this may point to Python 2 on older systems, or may not exist at all.
python3 The Python 3 interpreter (explicit) Running scripts on macOS/Linux Always refers to Python 3. Use this on macOS/Linux to be safe. Inside a virtual environment, python and python3 are the same.
pip Python package installer Installing third-party packages May be linked to Python 2 on some systems. Safest usage: python -m pip install ...
pip3 Package installer for Python 3 (explicit) Installing packages on macOS/Linux Guarantees packages install for Python 3. Inside a venv, pip and pip3 are equivalent.
venv Python's built-in virtual environment module Creating isolated project environments Included with Python 3.3+. Lightweight, no extra installation needed. Usage: python -m venv myenv
conda Third-party package and environment manager (from Anaconda/Miniconda) Data science projects, managing complex dependencies Manages both Python packages and non-Python dependencies (e.g., C libraries). Heavier than venv. Popular in data science. You do not need conda to start learning Python — venv + pip is sufficient.
virtualenv Third-party virtual environment tool Older projects or when you need extra features Predates venv. Faster environment creation, supports older Python. For most beginners, the built-in venv is all you need.
pyenv Tool for managing multiple Python versions When you need several Python versions side by side Not a package manager — it manages Python itself. Useful for developers who work on projects requiring different Python versions. macOS/Linux only (there is a Windows port called pyenv-win).

Recommended Setup for Beginners

Don't overcomplicate things. For your first few months with Python, all you need is:

  • Python 3 installed from python.org
  • venv for virtual environments (built-in, no installation needed)
  • pip for installing packages (comes with Python)
  • VS Code as your text editor (free, with excellent Python support)

You can explore conda, pyenv, and other tools later when you have a specific need for them.
