Common Errors & How to Fix Them

Every programmer makes mistakes — even seasoned professionals. The difference between a beginner and an experienced developer is not that the expert never sees errors; it is that the expert knows how to read the error message, understand what went wrong, and fix it quickly. This page is your complete reference for the 20 most common Python errors you will encounter as a beginner.

How to use this page: Each error includes the exact error message, a plain-English explanation, code that triggers the error, the fix, and tips to avoid it in the future. Bookmark this page and come back whenever you hit an error you don't recognize.

1. How to Read a Traceback

When Python encounters an error, it prints a traceback (also called a stack trace). This is not meant to intimidate you — it is a roadmap that tells you exactly where the problem is. Let's walk through one step by step.

Example traceback

Traceback (most recent call last):
  File "calculator.py", line 12, in <module>
    result = divide(10, 0)
  File "calculator.py", line 5, in divide
    return a / b
ZeroDivisionError: division by zero

Step-by-step breakdown

Line What It Tells You
Traceback (most recent call last): The start of the traceback. The most recent call is listed last (at the bottom).
File "calculator.py", line 12, in <module> The file name, the line number, and the scope. <module> means top-level code (not inside a function).
result = divide(10, 0) The actual line of code that was executing at that point.
File "calculator.py", line 5, in divide The error originated deeper — inside the divide function, at line 5.
return a / b This is the exact line that caused the error.
ZeroDivisionError: division by zero The error type and a description. This is always the last line and the most important one.
Key rule: Always read the traceback from the bottom up. The last line tells you what went wrong, and the lines above tell you where it went wrong.

Tips for reading tracebacks

  • Start at the bottom. The error type and message are on the last line.
  • Look at the last file/line reference that is YOUR code. Ignore lines from library files you did not write.
  • Read the code snippet shown. Python prints the offending line for you.
  • Check the line number. Open your file and go directly to that line.
  • Sometimes the real bug is one line above. A missing colon or parenthesis on line 5 may cause an error reported on line 6.

2. General Debugging Strategy

When you encounter an error, resist the urge to randomly change things. Instead, follow this systematic 5-step approach:

Step 1: Read the error message

This sounds obvious, but most beginners skip it. The error message almost always tells you exactly what is wrong. Read the entire traceback, starting from the bottom.

Step 2: Find the exact line

The traceback tells you which file and line number caused the error. Go to that line. Remember that the real problem might be on the line just above the reported line (especially for syntax errors).

Step 3: Reproduce with a minimal example

If the code is complex, try to isolate the problem. Create a tiny script that reproduces the same error. This helps you understand the root cause without distractions.

Step 4: Add print statements

When you are unsure what a variable contains, add print() calls before the error line to inspect values and types: 

# Debugging: What is x right before the error?
print(f"x = {x}, type = {type(x)}")
result = x + 10  # This line fails

Step 5: Search for the error message

If you still can't figure it out, copy the error message and search online. Sites like Stack Overflow have answers for virtually every Python error you will ever encounter. Include the error type in your search, for example: "TypeError unsupported operand type str int python".

Pro tip: As you gain experience, Steps 1 and 2 will be enough to solve 90% of your errors. The debugging process becomes second nature very quickly.

4. SyntaxError: invalid syntax

Error message: SyntaxError: invalid syntax

What it means

Python tried to read your code but found something it doesn't understand. Your code violates the grammar rules of the Python language. This is like a grammatical error in English — the sentence doesn't make sense to the interpreter.

Common causes and fixes

Cause 1: Missing colon after if, for, def, etc.

# BAD — missing colon
if x > 10
    print("big")

# GOOD
if x > 10:
    print("big")

Cause 2: Missing or mismatched parentheses

# BAD — missing closing parenthesis
print("Hello"

# GOOD
print("Hello")

Cause 3: Missing or mismatched quotes

# BAD — opening quote but no closing quote
message = "Hello world

# GOOD
message = "Hello world"

Cause 4: Using = instead of == in a condition

# BAD — assignment in a condition
if x = 5:
    print("five")

# GOOD — comparison
if x == 5:
    print("five")
How to fix it: Look at the line number in the error message. Check for missing colons, unmatched parentheses, unmatched quotes, or typos. Sometimes the real problem is on the line before the reported line.

Avoidance tip: Use a code editor with syntax highlighting. It will visually show you mismatched brackets and quotes by coloring your code differently when something is wrong.

5. SyntaxError: unexpected EOF while parsing

Error message: SyntaxError: unexpected EOF while parsing

What it means

"EOF" stands for "End Of File." Python reached the end of your code while it was still expecting more. Something is incomplete — an open parenthesis without a close, a function body with no code inside, or a block that is empty.

Common causes and fixes

Cause 1: Empty code block

# BAD — empty if block
if x > 10:

# GOOD — use pass as a placeholder
if x > 10:
    pass

Cause 2: Unclosed parenthesis, bracket, or brace

# BAD — missing closing bracket
numbers = [1, 2, 3

# GOOD
numbers = [1, 2, 3]

Cause 3: Incomplete function definition

# BAD — function body is missing
def greet(name):

# GOOD
def greet(name):
    print(f"Hello, {name}!")
How to fix it: Check if you have an empty code block (add pass as a placeholder), an unclosed bracket, or an incomplete statement at the end of your file.

Avoidance tip: When you type an opening bracket or parenthesis, immediately type the closing one too, then move your cursor back inside to fill in the contents.

6. IndentationError: expected an indented block

Error message: IndentationError: expected an indented block

What it means

After a colon (:) — such as after if, for, def, or class — Python expects at least one indented line of code. You either forgot to indent or left the block empty.

Code that causes the error

# BAD — no indented block after the colon
def say_hello():
print("Hello!")   # This line is not indented

How to fix it

# GOOD — indent the body of the function
def say_hello():
    print("Hello!")
# If you don't have code yet, use pass
def say_hello():
    pass  # Placeholder — will add code later
How to fix it: Indent the line(s) after the colon by 4 spaces. If you intentionally want an empty block, use pass.

Avoidance tip: Configure your editor to insert 4 spaces when you press Tab. After typing a colon and pressing Enter, most editors will auto-indent for you.

7. IndentationError: unexpected indent

Error message: IndentationError: unexpected indent

What it means

A line is indented more than Python expects. This usually happens when you accidentally add extra spaces or mix tabs and spaces.

Code that causes the error

# BAD — the second print has extra indentation
name = "Alice"
    print(name)   # IndentationError!

How to fix it

# GOOD — both lines at the same indentation level
name = "Alice"
print(name)
# Another common case: inconsistent indentation inside a block
if True:
    print("A")
        print("B")   # BAD — too much indentation

# GOOD
if True:
    print("A")
    print("B")
How to fix it: Make sure all lines at the same logical level have exactly the same indentation. Remove extra spaces or fix tab/space mixing.

Avoidance tip: Turn on "show whitespace" in your editor so you can see whether you are using tabs or spaces. Stick to spaces only (4 per level) and never mix tabs with spaces.

8. NameError: name 'x' is not defined

Error message: NameError: name 'x' is not defined

What it means

You are trying to use a variable, function, or name that Python has never heard of. Either you haven't created it yet, you made a typo, or it is defined in a different scope.

Common causes and fixes

Cause 1: Typo in a variable name

# BAD — typo: "mesage" instead of "message"
message = "Hello"
print(mesage)   # NameError: name 'mesage' is not defined

# GOOD
message = "Hello"
print(message)

Cause 2: Using a variable before assigning it

# BAD — using total before it exists
print(total)   # NameError
total = 100

# GOOD — assign first, then use
total = 100
print(total)

Cause 3: Forgetting to import a module

# BAD — math was never imported
result = math.sqrt(16)   # NameError: name 'math' is not defined

# GOOD
import math
result = math.sqrt(16)
How to fix it: Check the spelling carefully. Make sure the variable is assigned before the line that uses it. If it is from a module, make sure you have an import statement.

Avoidance tip: Use descriptive but consistent variable names. A code editor with autocomplete will help you avoid typos by suggesting names you have already defined.

9. TypeError: unsupported operand type(s)

Error message: TypeError: unsupported operand type(s) for +: 'str' and 'int'

What it means

You tried to perform an operation (like +, -, *) between two types that Python doesn't know how to combine. The most common case is trying to add a string and a number.

Code that causes the error

# BAD — can't add a string and an integer directly
age = 25
message = "I am " + age + " years old"   # TypeError!

How to fix it

# Fix 1: Convert the number to a string
age = 25
message = "I am " + str(age) + " years old"

# Fix 2: Use an f-string (recommended)
age = 25
message = f"I am {age} years old"

# Fix 3: Use format()
age = 25
message = "I am {} years old".format(age)

Another common case: multiplying a string by a float

# BAD
print("ha" * 2.5)   # TypeError: can't multiply str by non-int

# GOOD
print("ha" * 2)     # Output: haha
How to fix it: Convert values to compatible types before combining them. Use str() to turn numbers into strings, or int() / float() to turn strings into numbers. Better yet, use f-strings for building messages.

Avoidance tip: When combining text and numbers, always use f-strings: f"The answer is {value}". This automatically handles the conversion for you.

10. TypeError: 'type' object is not subscriptable

Error message: TypeError: 'list' object is not callable or TypeError: 'type' object is not subscriptable

What it means

You are confusing () (parentheses, for calling functions) with [] (square brackets, for indexing). Or you are using square brackets on a type name like list instead of creating an instance first.

Common causes and fixes

Cause 1: Using parentheses to index a list

# BAD — using () instead of []
fruits = ["apple", "banana", "cherry"]
print(fruits(0))   # TypeError: 'list' object is not callable

# GOOD — use square brackets for indexing
print(fruits[0])   # Output: apple

Cause 2: Naming a variable the same as a built-in type

# BAD — overwriting the built-in 'list' name
list = [1, 2, 3]
new_list = list([4, 5, 6])   # TypeError: 'list' object is not callable

# GOOD — don't use built-in names as variable names
numbers = [1, 2, 3]
new_list = list([4, 5, 6])   # Works fine

Cause 3: Using brackets on a type name in older Python

# In Python 3.8 and earlier, this fails:
def greet(names: list[str]):   # TypeError in Python < 3.9
    pass

# Fix for older Python: use typing module
from typing import List
def greet(names: List[str]):
    pass
How to fix it: Use [] for indexing collections and () for calling functions. Avoid naming your variables list, dict, str, int, or any other built-in name.

Avoidance tip: Never name a variable the same as a Python built-in. Use descriptive names like numbers, names, or items instead of list.

11. TypeError: function takes X positional arguments but Y were given

Error message: TypeError: greet() takes 1 positional argument but 2 were given

What it means

You called a function with a different number of arguments than it expects. You either passed too many or too few values.

Code that causes the error

# The function expects 1 argument
def greet(name):
    print(f"Hello, {name}!")

# BAD — passing 2 arguments
greet("Alice", "Bob")   # TypeError: takes 1 but 2 given

# BAD — passing 0 arguments
greet()   # TypeError: missing 1 required argument

How to fix it

# GOOD — pass exactly the number of arguments expected
greet("Alice")   # Output: Hello, Alice!

# If you need to accept variable arguments, use default values or *args
def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

greet("Alice")              # Uses default: Hello, Alice!
greet("Alice", "Hi there")  # Custom: Hi there, Alice!
How to fix it: Check the function definition to see how many parameters it expects. Match your function call to that number. Use default parameter values if you want optional arguments.

Avoidance tip: Hover over a function in your editor (or press Ctrl+click) to see its signature and parameter list before calling it.

12. ValueError: invalid literal for int()

Error message: ValueError: invalid literal for int() with base 10: 'hello'

What it means

You tried to convert a string to an integer (or float), but the string doesn't contain a valid number. Python can't turn "hello" or "12.5" into an int.

Code that causes the error

# BAD — "hello" is not a number
number = int("hello")   # ValueError!

# BAD — "12.5" has a decimal, can't convert directly to int
number = int("12.5")    # ValueError!

# BAD — empty string
number = int("")         # ValueError!

How to fix it

# GOOD — convert a valid numeric string
number = int("42")       # Works: 42

# For decimal strings, convert to float first
number = int(float("12.5"))   # Works: 12

# Validate user input before converting
user_input = input("Enter a number: ")
if user_input.isdigit():
    number = int(user_input)
else:
    print("That's not a valid number!")

# Or use try/except for robust handling
try:
    number = int(user_input)
except ValueError:
    print("Please enter a valid whole number.")
How to fix it: Always validate strings before converting them to numbers. Use .isdigit() for simple checks, or try/except ValueError for robust handling.

Avoidance tip: Whenever you convert user input to a number, wrap it in a try/except block. Users will type unexpected things.

13. IndexError: list index out of range

Error message: IndexError: list index out of range

What it means

You tried to access an element at a position that doesn't exist in the list. Remember: Python uses zero-based indexing, so a list with 3 items has valid indices 0, 1, and 2 (not 3).

Code that causes the error

colors = ["red", "green", "blue"]   # indices: 0, 1, 2

# BAD — index 3 doesn't exist (only 0, 1, 2)
print(colors[3])    # IndexError!

# BAD — using len() directly as an index
print(colors[len(colors)])   # IndexError! len is 3, but max index is 2

# BAD — accessing an empty list
empty = []
print(empty[0])   # IndexError!

How to fix it

colors = ["red", "green", "blue"]

# GOOD — use valid indices
print(colors[0])    # red
print(colors[2])    # blue
print(colors[-1])   # blue (last item)

# GOOD — check the length first
index = 5
if index < len(colors):
    print(colors[index])
else:
    print(f"Index {index} is out of range (list has {len(colors)} items)")

# GOOD — use try/except
try:
    print(colors[10])
except IndexError:
    print("That index doesn't exist!")
How to fix it: Check that your index is less than len(list). Remember that the last valid index is len(list) - 1. Use negative indexing (list[-1]) to safely access the last element.

Avoidance tip: When looping, use for item in list instead of for i in range(len(list)). The first form never causes an IndexError.

14. KeyError

Error message: KeyError: 'age'

What it means

You tried to access a dictionary key that doesn't exist. Unlike lists (which use numeric indices), dictionaries use named keys — and if the key isn't there, Python raises a KeyError.

Code that causes the error

person = {"name": "Alice", "city": "London"}

# BAD — "age" is not a key in this dictionary
print(person["age"])   # KeyError: 'age'

How to fix it

person = {"name": "Alice", "city": "London"}

# Fix 1: Use .get() — returns None (or a default) if key is missing
age = person.get("age")          # Returns None
age = person.get("age", "N/A")   # Returns "N/A"

# Fix 2: Check if the key exists first
if "age" in person:
    print(person["age"])
else:
    print("Age not found")

# Fix 3: Use try/except
try:
    print(person["age"])
except KeyError:
    print("Age not found")
How to fix it: Use .get(key, default) instead of [key] when a key might not exist. This is the most Pythonic approach and prevents crashes.

Avoidance tip: Make .get() your default way to access dictionary values. Only use [key] when you are 100% certain the key exists.

15. AttributeError

Error message: AttributeError: 'str' object has no attribute 'append'

What it means

You tried to call a method or access an attribute that doesn't exist on the object's type. For example, strings don't have an .append() method — that belongs to lists.

Common causes and fixes

Cause 1: Using the wrong method for the type

# BAD — strings don't have .append()
name = "Alice"
name.append(" Smith")   # AttributeError!

# GOOD — use string concatenation or f-strings
name = "Alice"
name = name + " Smith"
# Or: name = f"{name} Smith"

Cause 2: Misspelling a method name

# BAD — typo: "uper" instead of "upper"
text = "hello"
print(text.uper())   # AttributeError: 'str' has no attribute 'uper'

# GOOD
print(text.upper())   # HELLO

Cause 3: Variable is None when you expect something else

# BAD — sort() returns None, not the sorted list
numbers = [3, 1, 2]
sorted_numbers = numbers.sort()   # sort() returns None!
print(sorted_numbers.reverse())   # AttributeError: 'NoneType' has no attribute 'reverse'

# GOOD — sort() modifies in place, or use sorted()
numbers = [3, 1, 2]
numbers.sort()
print(numbers)   # [1, 2, 3]

# Or use sorted() to get a new list
sorted_numbers = sorted([3, 1, 2])
print(sorted_numbers)   # [1, 2, 3]
How to fix it: Check what type your variable actually is with type(variable), then look up the correct methods for that type. Watch out for methods that return None (like .sort() and .append()).

Avoidance tip: Use dir(object) in the Python shell to see all available methods for any object. For example, dir("hello") shows all string methods.

16. FileNotFoundError

Error message: FileNotFoundError: [Errno 2] No such file or directory: 'data.txt'

What it means

You tried to open or read a file that doesn't exist at the path you specified. Either the file name is wrong, the file is in a different folder, or it was never created.

Code that causes the error

# BAD — file doesn't exist at this path
with open("data.txt", "r") as f:
    content = f.read()   # FileNotFoundError!

How to fix it

import os

# Fix 1: Check if the file exists first
if os.path.exists("data.txt"):
    with open("data.txt", "r") as f:
        content = f.read()
else:
    print("File not found!")

# Fix 2: Use try/except
try:
    with open("data.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("File 'data.txt' not found. Check the path.")

# Fix 3: Use the full absolute path to be explicit
with open("/Users/alice/projects/data.txt", "r") as f:
    content = f.read()
How to fix it: Double-check the file path and name (spelling, capitalization, extension). Use os.path.exists() to verify the file exists before opening it, or use try/except.

Avoidance tip: Print the current working directory with os.getcwd() to understand where Python is looking for files. Use absolute paths when in doubt.

17. ZeroDivisionError

Error message: ZeroDivisionError: division by zero

What it means

You divided a number by zero. This is mathematically undefined, and Python refuses to do it.

Code that causes the error

# BAD — dividing by zero
result = 10 / 0   # ZeroDivisionError!

# BAD — this can happen with variables too
total = 100
count = 0
average = total / count   # ZeroDivisionError!

How to fix it

# Fix 1: Check before dividing
total = 100
count = 0

if count != 0:
    average = total / count
else:
    average = 0   # or handle it however makes sense
    print("Cannot compute average: no items")

# Fix 2: Use try/except
try:
    average = total / count
except ZeroDivisionError:
    average = 0
    print("Warning: divided by zero, defaulting to 0")
How to fix it: Always check that the divisor is not zero before dividing. This is especially important when the divisor comes from user input, a calculation, or a list length.

Avoidance tip: Any time you divide by a variable (not a literal number), ask yourself: "Could this ever be zero?" If yes, add a check.

18. ModuleNotFoundError

Error message: ModuleNotFoundError: No module named 'pandas'

What it means

You tried to import a module that Python can't find. Either the package isn't installed, you misspelled the name, or you're using a different Python environment than where the package was installed.

Code that causes the error

# BAD — pandas is not installed
import pandas   # ModuleNotFoundError!

# BAD — typo in module name
import numpyy   # ModuleNotFoundError!

How to fix it

# Fix 1: Install the missing module
# In your terminal (not in Python):
# pip install pandas

# Fix 2: Check spelling
import numpy   # not "numpyy"

# Fix 3: Check which Python you're using
# In your terminal:
# python --version
# pip list   (shows all installed packages)
How to fix it: Install the module using pip install module_name in your terminal. Verify the correct spelling. Make sure you are using the same Python environment where the package was installed.

Avoidance tip: Keep track of your project's dependencies. Use pip list to see what's installed, and pip freeze > requirements.txt to save a list.

19. ImportError

Error message: ImportError: cannot import name 'Calculator' from 'math'

What it means

The module exists, but the specific name you are trying to import from it does not. You might have misspelled the function or class name, or you might be importing from the wrong module.

Code that causes the error

# BAD — 'Calculator' doesn't exist in the math module
from math import Calculator   # ImportError!

# BAD — misspelled function name
from math import squareroot   # ImportError!

How to fix it

# GOOD — use the correct name
from math import sqrt
print(sqrt(16))   # 4.0

# Check what's available in a module
import math
print(dir(math))   # Lists all names in the module
How to fix it: Check the module's documentation for the correct name. Use dir(module) to see all available names. Verify the spelling exactly.

Avoidance tip: Use your editor's autocomplete after typing from module import — it will show you what names are actually available.

20. UnboundLocalError

Error message: UnboundLocalError: cannot access local variable 'count' where it is not associated with a value

What it means

Inside a function, you tried to read a variable before assigning it. This usually happens when you have a global variable and try to modify it inside a function without using the global keyword. Python sees the assignment and treats the variable as local — but then you try to read it before the assignment runs.

Code that causes the error

count = 0   # global variable

def increment():
    count = count + 1   # UnboundLocalError!
    # Python sees "count = ..." and treats count as LOCAL.
    # But "count + 1" tries to READ local count, which doesn't exist yet.

increment()

How to fix it

# Fix 1: Use the global keyword (use sparingly)
count = 0

def increment():
    global count
    count = count + 1

increment()
print(count)   # 1

# Fix 2 (better): Pass the value as a parameter and return the new value
def increment(current_count):
    return current_count + 1

count = 0
count = increment(count)
print(count)   # 1
How to fix it: Either use global variable_name at the top of your function (not recommended for large programs), or pass the variable as a parameter and return the new value (recommended).

Avoidance tip: Avoid modifying global variables inside functions. Instead, pass values in as parameters and return results. This makes your code easier to understand and debug.

21. RecursionError

Error message: RecursionError: maximum recursion depth exceeded

What it means

A function keeps calling itself forever. Python has a safety limit (usually 1000 calls deep) to prevent your program from crashing your computer. When a recursive function has no proper stopping condition (called a "base case"), it triggers this error.

Code that causes the error

# BAD — no base case, calls itself forever
def countdown(n):
    print(n)
    countdown(n - 1)   # Never stops!

countdown(5)   # RecursionError after ~1000 calls

How to fix it

# GOOD — add a base case to stop the recursion
def countdown(n):
    if n <= 0:       # Base case: stop when n reaches 0
        print("Done!")
        return
    print(n)
    countdown(n - 1)

countdown(5)
# Output: 5 4 3 2 1 Done!

Another example: factorial

# BAD — missing base case
def factorial(n):
    return n * factorial(n - 1)   # Never stops!

# GOOD — with base case
def factorial(n):
    if n <= 1:       # Base case
        return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120
How to fix it: Every recursive function must have a base case — a condition that causes the function to return without calling itself again. Make sure the recursive call moves toward that base case.

Avoidance tip: Before writing a recursive function, define two things: (1) the base case (when should it stop?), and (2) how each recursive call gets closer to the base case. If in doubt, use a loop instead.

22. StopIteration

Error message: StopIteration

What it means

You called next() on an iterator that has no more items to give. Iterators are "used up" after you go through all their elements. Calling next() one more time after the last item raises StopIteration.

Code that causes the error

numbers = iter([10, 20])   # Create an iterator with 2 items

print(next(numbers))   # 10
print(next(numbers))   # 20
print(next(numbers))   # StopIteration — no more items!

How to fix it

# Fix 1: Use a for loop instead — it handles StopIteration automatically
numbers = iter([10, 20])
for num in numbers:
    print(num)
# Output: 10, 20 (no error)

# Fix 2: Provide a default value to next()
numbers = iter([10, 20])
print(next(numbers, "done"))   # 10
print(next(numbers, "done"))   # 20
print(next(numbers, "done"))   # "done" (no error)

# Fix 3: Use try/except
numbers = iter([10, 20])
try:
    while True:
        print(next(numbers))
except StopIteration:
    print("All items processed")
How to fix it: Use a for loop to iterate safely (it handles StopIteration automatically). If you must use next() manually, provide a default value: next(iterator, default).

Avoidance tip: As a beginner, prefer for loops over manual next() calls. You will almost never need to call next() directly in everyday code.

23. PermissionError

Error message: PermissionError: [Errno 13] Permission denied: '/etc/config.txt'

What it means

You tried to read, write, or delete a file, but your user account doesn't have permission to do so. This commonly happens when trying to write to system directories, accessing files owned by another user, or trying to modify a read-only file.

Code that causes the error

# BAD — trying to write to a protected system directory
with open("/etc/my_config.txt", "w") as f:
    f.write("data")   # PermissionError!

# BAD — trying to overwrite a read-only file
with open("readonly_file.txt", "w") as f:
    f.write("new data")   # PermissionError if file is read-only

How to fix it

# Fix 1: Write to a directory you have permission for
with open("my_config.txt", "w") as f:   # Current directory, not /etc/
    f.write("data")

# Fix 2: Handle the error gracefully
try:
    with open("output.txt", "w") as f:
        f.write("data")
except PermissionError:
    print("Cannot write to that file. Check your permissions.")

# Fix 3: Check if the file is writable before attempting
import os
filepath = "output.txt"
if os.access(filepath, os.W_OK):
    with open(filepath, "w") as f:
        f.write("data")
else:
    print(f"No write permission for {filepath}")
How to fix it: Write files to your own directories (like your home folder or project folder). Use try/except PermissionError to handle cases where permissions are uncertain. On Linux/Mac, you may need to change file permissions with chmod.

Avoidance tip: Always write to locations you control — your project folder or a dedicated output directory. Avoid writing to system directories unless you are certain you have permission.

24. Putting It All Together: Error Handling Patterns

Now that you know the common errors, here are patterns for handling them gracefully in real code.

Pattern 1: The basic try/except

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")
    result = 0

Pattern 2: Catching multiple errors

try:
    value = int(input("Enter a number: "))
    result = 100 / value
except ValueError:
    print("That's not a valid number!")
except ZeroDivisionError:
    print("You can't divide by zero!")
except Exception as e:
    print(f"Unexpected error: {e}")

Pattern 3: Using else and finally

try:
    f = open("data.txt", "r")
except FileNotFoundError:
    print("File not found!")
else:
    # Only runs if no exception was raised
    content = f.read()
    print(f"Read {len(content)} characters")
    f.close()
finally:
    # Always runs, whether or not there was an exception
    print("File operation complete")

Pattern 4: Safe dictionary access with defaults

config = {"debug": True, "port": 8080}

# Instead of risking a KeyError:
host = config.get("host", "localhost")    # Returns "localhost"
port = config.get("port", 3000)           # Returns 8080 (key exists)
print(f"Server: {host}:{port}")

Pattern 5: Safe list access

def safe_get(lst, index, default=None):
    """Safely get an item from a list by index."""
    try:
        return lst[index]
    except IndexError:
        return default

scores = [85, 92, 78]
print(safe_get(scores, 0))     # 85
print(safe_get(scores, 10))    # None
print(safe_get(scores, 10, 0)) # 0
Remember: Error handling is not about preventing errors from ever happening — it is about handling them gracefully when they do. Your programs should fail informatively, not crash silently or mysteriously.

Summary

  • Read the traceback bottom-up — the last line tells you what went wrong, the lines above tell you where.
  • SyntaxError and IndentationError mean your code can't even be parsed — check colons, brackets, quotes, and spacing.
  • NameError usually means a typo or a missing import.
  • TypeError means you are mixing incompatible types or passing the wrong number of arguments.
  • ValueError means the value is the wrong kind (e.g., converting "hello" to an int).
  • IndexError and KeyError mean you are accessing something that doesn't exist — use .get() for dicts and check len() for lists.
  • AttributeError means you called a method that doesn't exist on that type — check the type with type().
  • FileNotFoundError and PermissionError are file-system issues — check paths and permissions.
  • Use try/except to handle errors gracefully in production code.
  • When in doubt, add print() statements to inspect variable values and types.
Errors are your friends, not your enemies. Every error message is Python trying to help you. The more errors you encounter and fix, the faster you learn. Don't be afraid of the red text — read it, understand it, fix it, and move on.
