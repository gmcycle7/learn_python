Functions & Lambda

Organise your code into reusable blocks. Learn how to define functions, pass arguments, return values, and use anonymous lambda expressions.

Note for MATLAB users: In MATLAB, functions typically live in their own .m files (or at the end of a script since R2016b). Python is far more flexible — you can define a function anywhere in your script, and you can put as many functions as you like in a single file. MATLAB uses the function keyword; Python uses def. Return values are explicit in Python (you write return), whereas MATLAB assigns to output variables declared in the function signature.

1. Defining Functions

A function is a named block of code that performs a specific task. You define it once, then call it as many times as you need. This avoids repeating the same code and makes your programs easier to read, test, and maintain.

Syntax

def function_name(parameters):
    """Optional docstring."""
    # body — indented code
    return value  # optional
  • def — the keyword that starts every function definition.
  • function_name — use snake_case (lowercase with underscores). Descriptive names are best: calculate_area, not ca.
  • parentheses — hold the parameters (inputs). Empty () if no inputs.
  • colon — required at the end of the def line.
  • indented body — everything inside the function must be indented (usually 4 spaces).

Example 1 — A Simple Function

# Define the function
def greet():
    print("Hello, welcome to Python!")

# Call it
greet()          # Hello, welcome to Python!
greet()          # Hello, welcome to Python!

Example 2 — Function with a Parameter

def greet_user(name):
    print(f"Hello, {name}! Welcome to Python.")

greet_user("Alice")   # Hello, Alice! Welcome to Python.
greet_user("Bob")     # Hello, Bob! Welcome to Python.

Example 3 — Function that Calculates

def circle_area(radius):
    pi = 3.14159
    area = pi * radius ** 2
    return area

result = circle_area(5)
print(result)   # 78.53975
MATLAB comparison: 
% MATLAB — function in its own file circle_area.m
function area = circle_area(radius)
    area = pi * radius^2;
end

In MATLAB you would save this in a file called circle_area.m. In Python, the function sits right in your script alongside other code. No special file required.

Common mistake: Forgetting the colon at the end of the def line, or forgetting to indent the body. Both will cause a SyntaxError. 
# Wrong — missing colon
def greet()
    print("Hi")    # SyntaxError!

# Wrong — no indentation
def greet():
print("Hi")        # IndentationError!

2. Parameters and Arguments

The terms parameter and argument are often used interchangeably, but there is a subtle difference:

  • Parameter — the variable name in the function definition (the placeholder).
  • Argument — the actual value you pass when calling the function.
def add(a, b):      # a and b are parameters
    return a + b

result = add(3, 5)  # 3 and 5 are arguments

Positional Arguments

By default, arguments are matched to parameters by position (left to right): 

def describe_pet(animal, name):
    print(f"I have a {animal} named {name}.")

describe_pet("cat", "Whiskers")   # I have a cat named Whiskers.
describe_pet("Whiskers", "cat")   # I have a Whiskers named cat.  (Wrong!)
Watch out: Order matters with positional arguments! Swapping them gives a logically wrong (but syntactically valid) result.

Keyword Arguments

You can specify arguments by name, which makes the order irrelevant: 

describe_pet(name="Whiskers", animal="cat")
# I have a cat named Whiskers.  (Correct regardless of order)

Default Values

You can give parameters a default value. If the caller does not provide that argument, the default is used: 

def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

greet("Alice")               # Hello, Alice!
greet("Alice", "Good morning")  # Good morning, Alice!
Rule: Parameters with default values must come after parameters without defaults. 
# Wrong — default before non-default
def greet(greeting="Hello", name):   # SyntaxError!
    print(f"{greeting}, {name}!")

Example — Combining Positional, Keyword, and Defaults

def create_profile(name, age, city="Unknown", hobby="coding"):
    print(f"{name}, age {age}, from {city}, enjoys {hobby}.")

create_profile("Alice", 30)
# Alice, age 30, from Unknown, enjoys coding.

create_profile("Bob", 25, city="London")
# Bob, age 25, from London, enjoys coding.

create_profile("Carol", 28, hobby="painting", city="Tokyo")
# Carol, age 28, from Tokyo, enjoys painting.

3. Return Values

The return statement sends a result back to the caller. Without return, a function returns None.

Returning a Single Value

def square(n):
    return n ** 2

result = square(7)
print(result)   # 49

Returning Multiple Values (Tuple Unpacking)

Python can return multiple values separated by commas. Under the hood this returns a tuple, which you can unpack into separate variables: 

def min_max(numbers):
    return min(numbers), max(numbers)

low, high = min_max([4, 1, 8, 3])
print(low)    # 1
print(high)   # 8
MATLAB comparison: MATLAB uses bracket syntax for multiple outputs: [low, high] = minmax(data). Python uses tuple unpacking, which feels very similar.

Returning None

If a function has no return statement (or uses return without a value), it returns None: 

def say_hello(name):
    print(f"Hello, {name}!")
    # no return statement

result = say_hello("Alice")   # Hello, Alice!
print(result)                  # None
Common mistake: Confusing print() with return. print() displays text on screen but does not send a value back to the caller. return sends a value back but does not display anything. 
def bad_add(a, b):
    print(a + b)       # displays 8, but returns None

def good_add(a, b):
    return a + b       # returns 8, caller decides what to do

x = bad_add(3, 5)     # prints 8, but x is None!
y = good_add(3, 5)    # y is 8

Early Return

You can use return to exit a function early: 

def divide(a, b):
    if b == 0:
        return "Error: division by zero"
    return a / b

print(divide(10, 3))   # 3.3333333333333335
print(divide(10, 0))   # Error: division by zero

4. Scope — Local vs. Global

Scope determines where a variable can be seen and used. In Python, every function creates its own local scope.

Local Variables

Variables created inside a function exist only inside that function: 

def my_function():
    secret = 42        # local variable
    print(secret)

my_function()           # 42
print(secret)           # NameError: name 'secret' is not defined

Global Variables

Variables created outside any function are global — they can be read inside functions, but not modified without the global keyword: 

greeting = "Hello"     # global

def say_hi():
    print(greeting)    # reading global — OK

say_hi()               # Hello
counter = 0

def increment():
    counter = counter + 1   # Error! Python thinks counter is local
                            # because you're assigning to it

increment()   # UnboundLocalError

The global Keyword (Use Sparingly!)

counter = 0

def increment():
    global counter         # tells Python: use the global variable
    counter = counter + 1

increment()
increment()
print(counter)   # 2
Best practice: Avoid using global in most cases. It makes code harder to understand, test, and debug because any function could secretly change a variable. Instead, pass values as arguments and return results: 
# Better approach — no global keyword needed
def increment(counter):
    return counter + 1

count = 0
count = increment(count)   # 1
count = increment(count)   # 2

The LEGB Rule (Simplified)

When Python encounters a variable name, it searches for it in this order:

  1. Local — inside the current function.
  2. Enclosing — inside any enclosing (outer) function (for nested functions).
  3. Global — at the module (file) level.
  4. Built-in — Python's built-in names (print, len, etc.).
x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)       # local

    inner()
    print(x)           # enclosing

outer()
print(x)               # global
As a beginner: You mostly need to remember two things: (1) variables inside a function are local, and (2) if Python cannot find a variable locally, it looks outward. The full LEGB rule will become important when you start writing nested functions.

5. Docstrings

A docstring is a string literal that appears as the first statement in a function. It documents what the function does. Use triple quotes ("""...""") for docstrings: 

def celsius_to_fahrenheit(celsius):
    """Convert a temperature from Celsius to Fahrenheit.

    Args:
        celsius: Temperature in degrees Celsius.

    Returns:
        Temperature in degrees Fahrenheit.
    """
    return celsius * 9 / 5 + 32

Why Bother?

  • You (or someone else) can read the docstring months later to understand what the function does.
  • The built-in help() function displays docstrings automatically:
help(celsius_to_fahrenheit)

# Output:
# Help on function celsius_to_fahrenheit in module __main__:
#
# celsius_to_fahrenheit(celsius)
#     Convert a temperature from Celsius to Fahrenheit.
#     ...

Accessing the Docstring Directly

print(celsius_to_fahrenheit.__doc__)
# Convert a temperature from Celsius to Fahrenheit.
# ...
MATLAB comparison: In MATLAB, you write help text as comments at the top of a function file. Python uses a dedicated docstring (a real string, not a comment), which tools can parse automatically. The idea is the same: document your functions!

Example — Good Docstring Habits

def calculate_bmi(weight_kg, height_m):
    """Calculate Body Mass Index (BMI).

    Args:
        weight_kg: Weight in kilograms.
        height_m: Height in metres.

    Returns:
        BMI as a float.

    Example:
        >>> calculate_bmi(70, 1.75)
        22.857142857142858
    """
    return weight_kg / height_m ** 2

6. Lambda Functions

A lambda is a small anonymous function (a function without a name). It is defined in a single line using the lambda keyword: 

# Syntax: lambda parameters: expression

double = lambda x: x * 2
print(double(5))    # 10

add = lambda a, b: a + b
print(add(3, 7))    # 10

How Lambda Compares to def

# Regular function
def square(x):
    return x ** 2

# Lambda equivalent
square = lambda x: x ** 2

# Both work the same way
print(square(4))   # 16

When to Use Lambda

Lambdas are most useful as short, throwaway functions passed to other functions like sorted(), map(), or filter(): 

# Sort a list of tuples by the second element
pairs = [(1, "b"), (3, "a"), (2, "c")]
sorted_pairs = sorted(pairs, key=lambda x: x[1])
print(sorted_pairs)   # [(3, 'a'), (1, 'b'), (2, 'c')]
# Use with map() to double every number
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, numbers))
print(doubled)   # [2, 4, 6, 8, 10]
# Use with filter() to keep only even numbers
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens)   # [2, 4]

When NOT to Use Lambda

Avoid lambda when:
  • The logic is complex or spans more than one expression — use a regular def function instead.
  • You need to reuse the function multiple times — give it a proper name with def.
  • You need a docstring — lambdas cannot have docstrings.
  • Readability suffers — if you have to squint to understand it, use def.
# Bad — too complex for a lambda
process = lambda x: x.strip().lower().replace(" ", "_") if isinstance(x, str) else str(x)

# Better — use def
def process(x):
    """Clean a value for use as a key."""
    if isinstance(x, str):
        return x.strip().lower().replace(" ", "_")
    return str(x)

7. Functions as First-Class Objects

In Python, functions are first-class objects. This means you can:

  • Assign a function to a variable.
  • Pass a function as an argument to another function.
  • Return a function from another function.

Assigning Functions to Variables

def shout(text):
    return text.upper() + "!"

yell = shout              # yell now refers to the same function
print(yell("hello"))      # HELLO!

Passing Functions as Arguments

def apply_twice(func, value):
    """Apply a function to a value two times."""
    return func(func(value))

def add_three(x):
    return x + 3

result = apply_twice(add_three, 10)
print(result)   # 16  (10 + 3 = 13, then 13 + 3 = 16)

A Practical Example — Custom Sorting

students = [
    {"name": "Alice", "grade": 88},
    {"name": "Bob", "grade": 95},
    {"name": "Carol", "grade": 72},
]

def get_grade(student):
    return student["grade"]

# Pass the function as the sorting key
ranked = sorted(students, key=get_grade, reverse=True)
for s in ranked:
    print(f"{s['name']}: {s['grade']}")
# Bob: 95
# Alice: 88
# Carol: 72
Key insight: When you pass a function as an argument, you write the function name without parentheses: key=get_grade, not key=get_grade(). Parentheses would call the function immediately, which is not what you want here.

8. *args and **kwargs

Sometimes you want a function that accepts any number of arguments. Python provides two special syntaxes for this.

*args — Variable Positional Arguments

The *args parameter collects extra positional arguments into a tuple: 

def add_all(*args):
    """Add any number of values together."""
    print(f"Received: {args}")     # args is a tuple
    return sum(args)

print(add_all(1, 2))           # Received: (1, 2) → 3
print(add_all(1, 2, 3, 4, 5)) # Received: (1, 2, 3, 4, 5) → 15

Example — Flexible Greeting

def greet_all(*names):
    for name in names:
        print(f"Hello, {name}!")

greet_all("Alice", "Bob", "Carol")
# Hello, Alice!
# Hello, Bob!
# Hello, Carol!

**kwargs — Variable Keyword Arguments

The **kwargs parameter collects extra keyword arguments into a dictionary: 

def build_profile(name, **kwargs):
    """Build a user profile dictionary."""
    profile = {"name": name}
    profile.update(kwargs)
    return profile

user = build_profile("Alice", age=30, city="London", job="engineer")
print(user)
# {'name': 'Alice', 'age': 30, 'city': 'London', 'job': 'engineer'}

Combining *args and **kwargs

def super_function(*args, **kwargs):
    print(f"Positional: {args}")
    print(f"Keyword:    {kwargs}")

super_function(1, 2, 3, name="Alice", age=30)
# Positional: (1, 2, 3)
# Keyword:    {'name': 'Alice', 'age': 30}
Naming convention: The names args and kwargs are just convention. You could use *values and **options — what matters is the * and ** prefix. However, sticking with args and kwargs is strongly recommended because every Python developer recognises them instantly.

Parameter Order Rule

When combining all parameter types, they must appear in this order: 

def example(pos1, pos2, default="x", *args, **kwargs):
    pass

# 1. Regular positional parameters
# 2. Parameters with default values
# 3. *args
# 4. **kwargs
MATLAB comparison: MATLAB uses varargin for variable-length input arguments and nargin to check how many arguments were passed. Python's *args and **kwargs are more elegant and explicit — you can iterate over them directly without checking argument counts.

Exercises

Practice what you have learned. Try each exercise on your own before revealing the solution.

Exercise 1 — Rectangle Area

Write a function rectangle_area(width, height) that returns the area of a rectangle. Test it with a few different values.

Show Solution 
def rectangle_area(width, height):
    """Return the area of a rectangle."""
    return width * height

print(rectangle_area(5, 3))    # 15
print(rectangle_area(10, 2))   # 20
print(rectangle_area(7.5, 4))  # 30.0
Exercise 2 — Temperature Converter

Write a function f_to_c(fahrenheit) that converts Fahrenheit to Celsius using the formula: C = (F - 32) * 5/9. Return the result rounded to 2 decimal places.

Show Solution 
def f_to_c(fahrenheit):
    """Convert Fahrenheit to Celsius."""
    celsius = (fahrenheit - 32) * 5 / 9
    return round(celsius, 2)

print(f_to_c(32))    # 0.0
print(f_to_c(212))   # 100.0
print(f_to_c(98.6))  # 37.0
Exercise 3 — Default Greeting

Write a function greet(name, greeting="Hello") that prints "{greeting}, {name}!". Call it with and without the greeting argument.

Show Solution 
def greet(name, greeting="Hello"):
    """Print a personalised greeting."""
    print(f"{greeting}, {name}!")

greet("Alice")                # Hello, Alice!
greet("Bob", "Good morning")  # Good morning, Bob!
Exercise 4 — Return Multiple Values

Write a function stats(numbers) that receives a list of numbers and returns three values: the minimum, maximum, and average. Unpack them into separate variables.

Show Solution 
def stats(numbers):
    """Return min, max, and average of a list."""
    minimum = min(numbers)
    maximum = max(numbers)
    average = sum(numbers) / len(numbers)
    return minimum, maximum, average

low, high, avg = stats([4, 8, 15, 16, 23, 42])
print(f"Min: {low}, Max: {high}, Avg: {avg:.2f}")
# Min: 4, Max: 42, Avg: 18.00
Exercise 5 — Lambda Sort

Given the list words = ["banana", "pie", "Washington", "a"], use sorted() with a lambda to sort the words by their length (shortest first).

Show Solution 
words = ["banana", "pie", "Washington", "a"]
by_length = sorted(words, key=lambda w: len(w))
print(by_length)   # ['a', 'pie', 'banana', 'Washington']
Exercise 6 — *args Sum

Write a function multiply_all(*args) that multiplies all the arguments together and returns the result. If no arguments are given, return 1.

Show Solution 
def multiply_all(*args):
    """Multiply all arguments together."""
    result = 1
    for num in args:
        result *= num
    return result

print(multiply_all(2, 3, 4))    # 24
print(multiply_all(5, 10))      # 50
print(multiply_all())            # 1
Exercise 7 — **kwargs Profile Builder

Write a function make_profile(name, **kwargs) that builds and returns a dictionary. The dictionary should always have a "name" key, plus any extra keyword arguments passed in. Test it by creating a profile with age, city, and hobby.

Show Solution 
def make_profile(name, **kwargs):
    """Build a profile dictionary from keyword arguments."""
    profile = {"name": name}
    for key, value in kwargs.items():
        profile[key] = value
    return profile

p = make_profile("Alice", age=30, city="London", hobby="chess")
print(p)
# {'name': 'Alice', 'age': 30, 'city': 'London', 'hobby': 'chess'}
Exercise 8 — Docstring Practice

Write a function is_even(n) that returns True if n is even, False otherwise. Add a proper docstring with Args, Returns, and an Example section. Then call help(is_even) to verify the docstring displays correctly.

Show Solution 
def is_even(n):
    """Check whether a number is even.

    Args:
        n: An integer to check.

    Returns:
        True if n is even, False otherwise.

    Example:
        >>> is_even(4)
        True
        >>> is_even(7)
        False
    """
    return n % 2 == 0

print(is_even(4))    # True
print(is_even(7))    # False
help(is_even)
Exercise 9 — Pass a Function

Write a function apply_to_list(func, lst) that applies a given function to every element in a list and returns the new list. Test it by passing a function that triples a number.

Show Solution 
def apply_to_list(func, lst):
    """Apply func to every element and return a new list."""
    return [func(item) for item in lst]

def triple(x):
    return x * 3

numbers = [1, 2, 3, 4, 5]
result = apply_to_list(triple, numbers)
print(result)   # [3, 6, 9, 12, 15]

# Also works with a lambda:
result2 = apply_to_list(lambda x: x ** 2, numbers)
print(result2)  # [1, 4, 9, 16, 25]
Exercise 10 — Scope Challenge

Without running the code, predict what this prints. Then run it to check your answer: 

x = 10

def outer():
    x = 20
    def inner():
        x = 30
        print("inner:", x)
    inner()
    print("outer:", x)

outer()
print("global:", x)
Show Solution 
# Output:
# inner: 30
# outer: 20
# global: 10

# Each function has its own local x.
# The global x is never modified.

Summary

Concept Syntax / Key Point
Define a function def name(params): body
Positional args Matched by order: func(a, b)
Keyword args Matched by name: func(name="Alice")
Default values def f(x, y=10):
Return values return value (or multiple: return a, b)
Scope LEGB: Local > Enclosing > Global > Built-in
Docstrings Triple-quoted string as first line of function body
Lambda lambda x: expression — for short, one-time functions
First-class functions Pass functions as arguments, assign to variables
*args Collects extra positional args into a tuple
**kwargs Collects extra keyword args into a dict
Why this matters: Functions are the fundamental building block of well-organised code. Once you are comfortable defining and calling functions, you can break any complex problem into small, manageable, testable pieces. Every serious Python program you write will be built on functions.
