常見錯誤與修正方法

每個程式設計師都會犯錯 — 即使是經驗豐富的專業人士。 初學者和有經驗的開發者之間的差別不在於 專家永遠不會看到錯誤；而是專家知道如何 閱讀錯誤訊息、理解 出了什麼問題，並快速修正。本頁是你 作為初學者會遇到的 20 個最常見 Python 錯誤的 完整參考。

如何使用本頁：每個錯誤都包含確切的 錯誤訊息、淺顯易懂的解釋、觸發錯誤的程式碼、 修正方法，以及未來避免的技巧。將本頁加入書籤， 每當你遇到不認識的錯誤時回來查看。

1. 如何閱讀追蹤回溯

當 Python 遇到錯誤時，它會印出一個追蹤回溯 （也稱為堆疊追蹤）。這不是要嚇你的 — 它是一張路線圖，準確告訴你問題在哪裡。讓我們 一步一步地走過一個。

範例追蹤回溯

Traceback (most recent call last):
  File "calculator.py", line 12, in <module>
    result = divide(10, 0)
  File "calculator.py", line 5, in divide
    return a / b
ZeroDivisionError: division by zero

逐步分析

行 它告訴你什麼
Traceback (most recent call last): 追蹤回溯的開始。最近的呼叫列在最後（底部）。
File "calculator.py", line 12, in <module> 檔案名稱、行號和作用域。<module> 表示頂層程式碼（不在函式內部）。
result = divide(10, 0) 在該點正在執行的實際程式碼行。
File "calculator.py", line 5, in divide 錯誤源頭更深處 — 在 divide 函式內，第 5 行。
return a / b 這是導致錯誤的確切程式碼行。
ZeroDivisionError: division by zero 錯誤型別和描述。這總是最後一行，也是最重要的一行。
關鍵規則：總是由下往上閱讀追蹤回溯。最後一行告訴你出了什麼問題，上面的行告訴你在哪裡出了問題。

閱讀追蹤回溯的技巧

  • 從底部開始。錯誤型別和訊息在最後一行。
  • 找到最後一個屬於你的程式碼的檔案/行參考。忽略你沒有撰寫的函式庫檔案中的行。
  • 閱讀顯示的程式碼片段。Python 為你印出了出錯的那一行。
  • 檢查行號。開啟你的檔案，直接跳到那一行。
  • 有時真正的錯誤在上一行。第 5 行缺少冒號或括號可能導致第 6 行報告錯誤。

2. 通用除錯策略

當你遇到錯誤時，不要隨意修改。 相反地，遵循這個系統化的 5 步驟方法：

步驟 1：閱讀錯誤訊息

這聽起來很明顯，但大多數初學者會跳過它。錯誤訊息 幾乎總是準確地告訴你出了什麼問題。閱讀整個追蹤回溯， 從底部開始。

步驟 2：找到確切的行

追蹤回溯告訴你哪個檔案和行號導致了錯誤。 轉到那一行。記住真正的問題可能在報告行的 上一行（特別是對於語法錯誤）。

步驟 3：用最小範例重現

如果程式碼很複雜，嘗試隔離問題。建立一個能 重現相同錯誤的小腳本。這幫助你理解 根本原因而不受干擾。

步驟 4：加入 print 語句

當你不確定變數包含什麼時，在錯誤行之前加入 print() 呼叫來檢查值和型別： 

# 除錯：在錯誤之前 x 是什麼？
print(f"x = {x}, type = {type(x)}")
result = x + 10  # 這行失敗

步驟 5：搜尋錯誤訊息

如果你仍然無法弄清楚，複製錯誤訊息並在網上搜尋。 像 Stack Overflow 這樣的網站幾乎有你會遇到的 每個 Python 錯誤的答案。在搜尋中包含錯誤型別， 例如："TypeError unsupported operand type str int python"。

專家提示：隨著你獲得經驗，步驟 1 和 2 就足以 解決 90% 的錯誤。除錯過程很快就會變成第二天性。

4. SyntaxError: invalid syntax

錯誤訊息： SyntaxError: invalid syntax

它的意思

Python 嘗試讀取你的程式碼但發現了它不理解的東西。 你的程式碼違反了 Python 語言的語法規則。這就像 英文中的語法錯誤 — 句子對直譯器來說沒有意義。

常見原因和修正

原因 1：if、for、def 等後面缺少冒號

# 錯誤 — 缺少冒號
if x > 10
    print("big")

# 正確
if x > 10:
    print("big")

原因 2：缺少或不匹配的括號

# 錯誤 — 缺少右括號
print("Hello"

# 正確
print("Hello")

原因 3：缺少或不匹配的引號

# 錯誤 — 有左引號但沒有右引號
message = "Hello world

# 正確
message = "Hello world"

原因 4：在條件中使用 = 而不是 ==

# 錯誤 — 條件中的賦值
if x = 5:
    print("five")

# 正確 — 比較
if x == 5:
    print("five")
如何修正：查看錯誤訊息中的行號。檢查是否缺少冒號、不匹配的括號、不匹配的引號或打字錯誤。有時真正的問題在報告行的上一行。

避免技巧：使用有語法高亮的程式碼編輯器。當有東西出問題時，它會用不同的顏色來顯示不匹配的括號和引號。

5. SyntaxError: unexpected EOF while parsing

錯誤訊息： SyntaxError: unexpected EOF while parsing

它的意思

「EOF」代表「End Of File（檔案結尾）」。Python 在仍然期待更多內容時到達了程式碼的結尾。某些東西不完整 — 一個沒有關閉的括號、沒有程式碼的函式本體，或一個空的區塊。

常見原因和修正

原因 1：空的程式碼區塊

# 錯誤 — 空的 if 區塊
if x > 10:

# 正確 — 使用 pass 作為佔位符
if x > 10:
    pass

原因 2：未關閉的括號、方括號或大括號

# 錯誤 — 缺少右方括號
numbers = [1, 2, 3

# 正確
numbers = [1, 2, 3]

原因 3：不完整的函式定義

# 錯誤 — 函式本體缺失
def greet(name):

# 正確
def greet(name):
    print(f"Hello, {name}!")
如何修正：檢查你是否有空的程式碼區塊（加入 pass 作為佔位符）、未關閉的括號，或檔案結尾處不完整的語句。

避免技巧：當你輸入左括號時，立即也輸入右括號，然後將游標移回中間填入內容。

6. IndentationError: expected an indented block

錯誤訊息： IndentationError: expected an indented block

它的意思

在冒號（:）之後 — 例如 if、for、def 或 class 之後 — Python 期待至少一行縮排的程式碼。你要麼忘記縮排，要麼讓區塊留空了。

導致錯誤的程式碼

# 錯誤 — 冒號後沒有縮排的區塊
def say_hello():
print("Hello!")   # 這行沒有縮排

如何修正

# 正確 — 縮排函式的本體
def say_hello():
    print("Hello!")
# 如果你還沒有程式碼，使用 pass
def say_hello():
    pass  # 佔位符 — 稍後會加入程式碼
如何修正：將冒號後的行縮排 4 個空格。如果你故意要一個空區塊，使用 pass。

避免技巧：設定你的編輯器在按 Tab 時插入 4 個空格。輸入冒號並按 Enter 後，大多數編輯器會自動縮排。

7. IndentationError: unexpected indent

錯誤訊息： IndentationError: unexpected indent

它的意思

某行的縮排比 Python 預期的多。這通常發生在你不小心多加了空格或混用了 tab 和空格時。

導致錯誤的程式碼

# 錯誤 — 第二個 print 有多餘的縮排
name = "Alice"
    print(name)   # IndentationError!

如何修正

# 正確 — 兩行在同一縮排層級
name = "Alice"
print(name)
# 另一個常見情況：區塊內不一致的縮排
if True:
    print("A")
        print("B")   # 錯誤 — 縮排太多

# 正確
if True:
    print("A")
    print("B")
如何修正：確保同一邏輯層級的所有行有完全相同的縮排。移除多餘的空格或修正 tab/空格的混用。

避免技巧：在編輯器中啟用「顯示空白字元」，這樣你就能看到你是用 tab 還是空格。只使用空格（每層 4 個），永遠不要混用 tab 和空格。

8. NameError: name 'x' is not defined

錯誤訊息： NameError: name 'x' is not defined

它的意思

你嘗試使用 Python 從未聽過的變數、函式或名稱。要麼你還沒有建立它、你打了錯字，或它定義在不同的作用域中。

常見原因和修正

原因 1：變數名稱打字錯誤

# 錯誤 — 打字錯誤："mesage" 而不是 "message"
message = "Hello"
print(mesage)   # NameError: name 'mesage' is not defined

# 正確
message = "Hello"
print(message)

原因 2：在賦值之前使用變數

# 錯誤 — 在 total 存在之前使用它
print(total)   # NameError
total = 100

# 正確 — 先賦值，再使用
total = 100
print(total)

原因 3：忘記匯入模組

# 錯誤 — math 從未被匯入
result = math.sqrt(16)   # NameError: name 'math' is not defined

# 正確
import math
result = math.sqrt(16)
如何修正：仔細檢查拼寫。確保變數在使用它的行之前被賦值。如果它來自模組，確保你有 import 語句。

避免技巧：使用描述性但一致的變數名稱。有自動完成功能的程式碼編輯器會通過建議你已經定義的名稱來幫助你避免打字錯誤。

9. TypeError: unsupported operand type(s)

錯誤訊息： TypeError: unsupported operand type(s) for +: 'str' and 'int'

它的意思

你嘗試在 Python 不知道如何組合的兩個型別之間執行操作（如 +、-、*）。最常見的情況是嘗試將字串和數字相加。

導致錯誤的程式碼

# 錯誤 — 不能直接將字串和整數相加
age = 25
message = "I am " + age + " years old"   # TypeError!

如何修正

# 修正 1：將數字轉換為字串
age = 25
message = "I am " + str(age) + " years old"

# 修正 2：使用 f-string（推薦）
age = 25
message = f"I am {age} years old"

# 修正 3：使用 format()
age = 25
message = "I am {} years old".format(age)

另一個常見情況：字串乘以浮點數

# 錯誤
print("ha" * 2.5)   # TypeError: can't multiply str by non-int

# 正確
print("ha" * 2)     # 輸出：haha
如何修正：在組合之前將值轉換為相容的型別。使用 str() 將數字轉為字串，或 int() / float() 將字串轉為數字。更好的是，使用 f-string 來建構訊息。

避免技巧：當組合文字和數字時，總是使用 f-string：f"The answer is {value}"。這會自動為你處理轉換。

10. TypeError: 'type' object is not subscriptable

錯誤訊息： TypeError: 'list' object is not callable 或 TypeError: 'type' object is not subscriptable

它的意思

你混淆了 ()（括號，用於呼叫函式）和 []（方括號，用於索引）。或者你對型別名稱如 list 使用了方括號而不是先建立實例。

常見原因和修正

原因 1：用括號來索引串列

# 錯誤 — 使用 () 而不是 []
fruits = ["apple", "banana", "cherry"]
print(fruits(0))   # TypeError: 'list' object is not callable

# 正確 — 使用方括號進行索引
print(fruits[0])   # 輸出：apple

原因 2：變數名稱和內建型別同名

# 錯誤 — 覆蓋了內建的 'list' 名稱
list = [1, 2, 3]
new_list = list([4, 5, 6])   # TypeError: 'list' object is not callable

# 正確 — 不要用內建名稱作為變數名稱
numbers = [1, 2, 3]
new_list = list([4, 5, 6])   # 正常運作

原因 3：在舊版 Python 中對型別名稱使用方括號

# 在 Python 3.8 及更早版本中，這會失敗：
def greet(names: list[str]):   # Python < 3.9 中的 TypeError
    pass

# 舊版 Python 的修正：使用 typing 模組
from typing import List
def greet(names: List[str]):
    pass
如何修正：使用 [] 來索引集合，使用 () 來呼叫函式。避免將變數命名為 list、dict、str、int 或任何其他內建名稱。

避免技巧：永遠不要將變數命名為和 Python 內建名稱相同。使用描述性名稱如 numbers、names 或 items 而不是 list。

11. TypeError: function takes X positional arguments but Y were given

錯誤訊息： TypeError: greet() takes 1 positional argument but 2 were given

它的意思

你用和函式期望的不同數量的引數來呼叫函式。你傳了太多或太少的值。

導致錯誤的程式碼

# 函式期望 1 個引數
def greet(name):
    print(f"Hello, {name}!")

# 錯誤 — 傳了 2 個引數
greet("Alice", "Bob")   # TypeError: takes 1 but 2 given

# 錯誤 — 傳了 0 個引數
greet()   # TypeError: missing 1 required argument

如何修正

# 正確 — 傳入確切的引數數量
greet("Alice")   # 輸出：Hello, Alice!

# 如果你需要接受可變引數，使用預設值或 *args
def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

greet("Alice")              # 使用預設值：Hello, Alice!
greet("Alice", "Hi there")  # 自訂：Hi there, Alice!
如何修正：檢查函式定義看它期望多少個參數。讓你的函式呼叫匹配那個數量。如果你想要可選引數，使用預設參數值。

避免技巧：在編輯器中將游標懸停在函式上（或按 Ctrl+點擊）來查看其簽名和參數列表，然後再呼叫它。

12. ValueError: invalid literal for int()

錯誤訊息： ValueError: invalid literal for int() with base 10: 'hello'

它的意思

你嘗試將字串轉換為整數（或浮點數），但字串不包含有效的數字。Python 無法將 "hello" 或 "12.5" 轉換為 int。

導致錯誤的程式碼

# 錯誤 — "hello" 不是數字
number = int("hello")   # ValueError!

# 錯誤 — "12.5" 有小數點，不能直接轉換為 int
number = int("12.5")    # ValueError!

# 錯誤 — 空字串
number = int("")         # ValueError!

如何修正

# 正確 — 轉換有效的數字字串
number = int("42")       # 有效：42

# 對於有小數的字串，先轉換為 float
number = int(float("12.5"))   # 有效：12

# 在轉換之前驗證使用者輸入
user_input = input("Enter a number: ")
if user_input.isdigit():
    number = int(user_input)
else:
    print("That's not a valid number!")

# 或使用 try/except 進行穩健處理
try:
    number = int(user_input)
except ValueError:
    print("Please enter a valid whole number.")
如何修正：在轉換字串為數字之前總是驗證它們。使用 .isdigit() 進行簡單檢查，或使用 try/except ValueError 進行穩健處理。

避免技巧：每當你將使用者輸入轉換為數字時，都包在 try/except 區塊中。使用者會輸入意想不到的東西。

13. IndexError: list index out of range

錯誤訊息： IndexError: list index out of range

它的意思

你嘗試存取串列中不存在的位置的元素。記住：Python 使用零基索引，所以有 3 個項目的串列有效索引是 0、1 和 2（不是 3）。

導致錯誤的程式碼

colors = ["red", "green", "blue"]   # 索引：0、1、2

# 錯誤 — 索引 3 不存在（只有 0、1、2）
print(colors[3])    # IndexError!

# 錯誤 — 直接用 len() 作為索引
print(colors[len(colors)])   # IndexError！len 是 3，但最大索引是 2

# 錯誤 — 存取空串列
empty = []
print(empty[0])   # IndexError!

如何修正

colors = ["red", "green", "blue"]

# 正確 — 使用有效索引
print(colors[0])    # red
print(colors[2])    # blue
print(colors[-1])   # blue（最後一個項目）

# 正確 — 先檢查長度
index = 5
if index < len(colors):
    print(colors[index])
else:
    print(f"Index {index} is out of range (list has {len(colors)} items)")

# 正確 — 使用 try/except
try:
    print(colors[10])
except IndexError:
    print("That index doesn't exist!")
如何修正：檢查你的索引小於 len(list)。記住最後一個有效索引是 len(list) - 1。使用負索引（list[-1]）來安全地存取最後一個元素。

避免技巧：迴圈時使用 for item in list 而不是 for i in range(len(list))。前者永遠不會引起 IndexError。

14. KeyError

錯誤訊息： KeyError: 'age'

它的意思

你嘗試存取不存在的字典鍵。與串列（使用數字索引）不同，字典使用命名的鍵 — 如果鍵不在那裡，Python 會引發 KeyError。

導致錯誤的程式碼

person = {"name": "Alice", "city": "London"}

# 錯誤 — "age" 不是這個字典中的鍵
print(person["age"])   # KeyError: 'age'

如何修正

person = {"name": "Alice", "city": "London"}

# 修正 1：使用 .get() — 鍵不存在時回傳 None（或預設值）
age = person.get("age")          # 回傳 None
age = person.get("age", "N/A")   # 回傳 "N/A"

# 修正 2：先檢查鍵是否存在
if "age" in person:
    print(person["age"])
else:
    print("Age not found")

# 修正 3：使用 try/except
try:
    print(person["age"])
except KeyError:
    print("Age not found")
如何修正：當鍵可能不存在時，使用 .get(key, default) 而不是 [key]。這是最 Pythonic 的做法，能防止程式崩潰。

避免技巧：讓 .get() 成為你存取字典值的預設方式。只有在你 100% 確定鍵存在時才使用 [key]。

15. AttributeError

錯誤訊息： AttributeError: 'str' object has no attribute 'append'

它的意思

你嘗試呼叫物件型別上不存在的方法或存取不存在的屬性。例如，字串沒有 .append() 方法 — 那屬於串列。

常見原因和修正

原因 1：對該型別使用了錯誤的方法

# 錯誤 — 字串沒有 .append()
name = "Alice"
name.append(" Smith")   # AttributeError!

# 正確 — 使用字串串接或 f-string
name = "Alice"
name = name + " Smith"
# 或：name = f"{name} Smith"

原因 2：方法名稱拼寫錯誤

# 錯誤 — 打字錯誤："uper" 而不是 "upper"
text = "hello"
print(text.uper())   # AttributeError: 'str' has no attribute 'uper'

# 正確
print(text.upper())   # HELLO

原因 3：變數是 None 但你期望別的東西

# 錯誤 — sort() 回傳 None，不是排序後的串列
numbers = [3, 1, 2]
sorted_numbers = numbers.sort()   # sort() 回傳 None！
print(sorted_numbers.reverse())   # AttributeError: 'NoneType' has no attribute 'reverse'

# 正確 — sort() 就地修改，或使用 sorted()
numbers = [3, 1, 2]
numbers.sort()
print(numbers)   # [1, 2, 3]

# 或使用 sorted() 來取得新串列
sorted_numbers = sorted([3, 1, 2])
print(sorted_numbers)   # [1, 2, 3]
如何修正：用 type(variable) 檢查你的變數實際上是什麼型別，然後查找該型別的正確方法。注意回傳 None 的方法（如 .sort() 和 .append()）。

避免技巧：在 Python shell 中使用 dir(object) 查看任何物件的所有可用方法。例如，dir("hello") 顯示所有字串方法。

16. FileNotFoundError

錯誤訊息： FileNotFoundError: [Errno 2] No such file or directory: 'data.txt'

它的意思

你嘗試開啟或讀取在你指定路徑不存在的檔案。要麼檔案名稱錯了、檔案在不同的資料夾中，或它從未被建立。

導致錯誤的程式碼

# 錯誤 — 這個路徑的檔案不存在
with open("data.txt", "r") as f:
    content = f.read()   # FileNotFoundError!

如何修正

import os

# 修正 1：先檢查檔案是否存在
if os.path.exists("data.txt"):
    with open("data.txt", "r") as f:
        content = f.read()
else:
    print("File not found!")

# 修正 2：使用 try/except
try:
    with open("data.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("File 'data.txt' not found. Check the path.")

# 修正 3：使用完整的絕對路徑以明確表示
with open("/Users/alice/projects/data.txt", "r") as f:
    content = f.read()
如何修正：仔細檢查檔案路徑和名稱（拼寫、大小寫、副檔名）。使用 os.path.exists() 在開啟前驗證檔案是否存在，或使用 try/except。

避免技巧：用 os.getcwd() 印出當前工作目錄，以了解 Python 在哪裡尋找檔案。不確定時使用絕對路徑。

17. ZeroDivisionError

錯誤訊息： ZeroDivisionError: division by zero

它的意思

你把一個數字除以零。這在數學上是未定義的，Python 拒絕執行。

導致錯誤的程式碼

# 錯誤 — 除以零
result = 10 / 0   # ZeroDivisionError!

# 錯誤 — 變數也可能這樣
total = 100
count = 0
average = total / count   # ZeroDivisionError!

如何修正

# 修正 1：除之前先檢查
total = 100
count = 0

if count != 0:
    average = total / count
else:
    average = 0   # 或按合理的方式處理
    print("Cannot compute average: no items")

# 修正 2：使用 try/except
try:
    average = total / count
except ZeroDivisionError:
    average = 0
    print("Warning: divided by zero, defaulting to 0")
如何修正：在除法之前總是檢查除數不為零。當除數來自使用者輸入、計算結果或串列長度時尤其重要。

避免技巧：每當你除以一個變數（不是字面數字）時，問自己：「這可能為零嗎？」如果可能，加一個檢查。

18. ModuleNotFoundError

錯誤訊息： ModuleNotFoundError: No module named 'pandas'

它的意思

你嘗試匯入 Python 找不到的模組。要麼套件未安裝、你拼錯了名稱，或你使用的 Python 環境和安裝套件的環境不同。

導致錯誤的程式碼

# 錯誤 — pandas 未安裝
import pandas   # ModuleNotFoundError!

# 錯誤 — 模組名稱打字錯誤
import numpyy   # ModuleNotFoundError!

如何修正

# 修正 1：安裝缺少的模組
# 在你的終端機中（不是在 Python 中）：
# pip install pandas

# 修正 2：檢查拼寫
import numpy   # 不是 "numpyy"

# 修正 3：檢查你使用的 Python 版本
# 在你的終端機中：
# python --version
# pip list   （顯示所有已安裝的套件）
如何修正：在終端機中使用 pip install module_name 安裝模組。驗證正確的拼寫。確保你使用的是安裝套件的同一個 Python 環境。

避免技巧：追蹤你專案的依賴。使用 pip list 查看已安裝的套件，使用 pip freeze > requirements.txt 儲存清單。

19. ImportError

錯誤訊息： ImportError: cannot import name 'Calculator' from 'math'

它的意思

模組存在，但你嘗試從中匯入的特定名稱不存在。你可能拼錯了函式或類別名稱，或你從錯誤的模組匯入。

導致錯誤的程式碼

# 錯誤 — math 模組中不存在 'Calculator'
from math import Calculator   # ImportError!

# 錯誤 — 函式名稱拼寫錯誤
from math import squareroot   # ImportError!

如何修正

# 正確 — 使用正確的名稱
from math import sqrt
print(sqrt(16))   # 4.0

# 檢查模組中有哪些可用的名稱
import math
print(dir(math))   # 列出模組中的所有名稱
如何修正：查看模組的文件以取得正確的名稱。使用 dir(module) 查看所有可用的名稱。精確驗證拼寫。

避免技巧：在輸入 from module import 後使用編輯器的自動完成 — 它會顯示實際可用的名稱。

20. UnboundLocalError

錯誤訊息： UnboundLocalError: cannot access local variable 'count' where it is not associated with a value

它的意思

在函式內部，你嘗試在賦值之前讀取一個變數。這通常發生在你有一個全域變數並嘗試在函式內修改它而沒有使用 global 關鍵字時。Python 看到賦值就把變數當作局部的 — 但你在賦值執行之前就嘗試讀取它。

導致錯誤的程式碼

count = 0   # 全域變數

def increment():
    count = count + 1   # UnboundLocalError!
    # Python 看到 "count = ..." 就把 count 當作局部變數。
    # 但 "count + 1" 嘗試讀取還不存在的局部 count。

increment()

如何修正

# 修正 1：使用 global 關鍵字（謹慎使用）
count = 0

def increment():
    global count
    count = count + 1

increment()
print(count)   # 1

# 修正 2（更好）：將值作為參數傳入並回傳新值
def increment(current_count):
    return current_count + 1

count = 0
count = increment(count)
print(count)   # 1
如何修正：在函式頂部使用 global variable_name（大型程式不推薦），或將變數作為參數傳入並回傳新值（推薦）。

避免技巧：避免在函式內修改全域變數。改為透過參數傳入值並回傳結果。這讓你的程式碼更容易理解和除錯。

21. RecursionError

錯誤訊息： RecursionError: maximum recursion depth exceeded

它的意思

一個函式不斷地呼叫自己。Python 有一個安全限制（通常是 1000 次深度）以防止你的程式讓電腦當機。當遞迴函式沒有適當的停止條件（稱為「基礎情況」）時，就會觸發這個錯誤。

導致錯誤的程式碼

# 錯誤 — 沒有基礎情況，永遠呼叫自己
def countdown(n):
    print(n)
    countdown(n - 1)   # 永遠不停！

countdown(5)   # 約 1000 次呼叫後 RecursionError

如何修正

# 正確 — 加入基礎情況來停止遞迴
def countdown(n):
    if n <= 0:       # 基礎情況：n 到 0 時停止
        print("Done!")
        return
    print(n)
    countdown(n - 1)

countdown(5)
# 輸出：5 4 3 2 1 Done!

另一個範例：階乘

# 錯誤 — 缺少基礎情況
def factorial(n):
    return n * factorial(n - 1)   # 永遠不停！

# 正確 — 有基礎情況
def factorial(n):
    if n <= 1:       # 基礎情況
        return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120
如何修正：每個遞迴函式必須有一個基礎情況 — 讓函式不再呼叫自己而直接回傳的條件。確保遞迴呼叫朝向那個基礎情況前進。

避免技巧：在撰寫遞迴函式之前，定義兩件事：(1) 基礎情況（什麼時候應該停止？），以及 (2) 每次遞迴呼叫如何更接近基礎情況。如果有疑問，改用迴圈。

22. StopIteration

錯誤訊息： StopIteration

它的意思

你對一個沒有更多項目可以提供的迭代器呼叫了 next()。迭代器在你走過所有元素後就「用完了」。在最後一個項目之後再呼叫 next() 會引發 StopIteration。

導致錯誤的程式碼

numbers = iter([10, 20])   # 建立有 2 個項目的迭代器

print(next(numbers))   # 10
print(next(numbers))   # 20
print(next(numbers))   # StopIteration — 沒有更多項目了！

如何修正

# 修正 1：改用 for 迴圈 — 它自動處理 StopIteration
numbers = iter([10, 20])
for num in numbers:
    print(num)
# 輸出：10、20（沒有錯誤）

# 修正 2：為 next() 提供預設值
numbers = iter([10, 20])
print(next(numbers, "done"))   # 10
print(next(numbers, "done"))   # 20
print(next(numbers, "done"))   # "done"（沒有錯誤）

# 修正 3：使用 try/except
numbers = iter([10, 20])
try:
    while True:
        print(next(numbers))
except StopIteration:
    print("All items processed")
如何修正：使用 for 迴圈來安全迭代（它自動處理 StopIteration）。如果你必須手動使用 next()，提供預設值：next(iterator, default)。

避免技巧：作為初學者，偏好使用 for 迴圈而不是手動呼叫 next()。在日常程式碼中你幾乎不需要直接呼叫 next()。

23. PermissionError

錯誤訊息： PermissionError: [Errno 13] Permission denied: '/etc/config.txt'

它的意思

你嘗試讀取、寫入或刪除檔案，但你的使用者帳號沒有權限這麼做。這通常發生在嘗試寫入系統目錄、存取其他使用者擁有的檔案，或嘗試修改唯讀檔案時。

導致錯誤的程式碼

# 錯誤 — 嘗試寫入受保護的系統目錄
with open("/etc/my_config.txt", "w") as f:
    f.write("data")   # PermissionError!

# 錯誤 — 嘗試覆寫唯讀檔案
with open("readonly_file.txt", "w") as f:
    f.write("new data")   # 如果檔案是唯讀的則 PermissionError

如何修正

# 修正 1：寫入你有權限的目錄
with open("my_config.txt", "w") as f:   # 當前目錄，不是 /etc/
    f.write("data")

# 修正 2：優雅地處理錯誤
try:
    with open("output.txt", "w") as f:
        f.write("data")
except PermissionError:
    print("Cannot write to that file. Check your permissions.")

# 修正 3：在嘗試之前檢查檔案是否可寫
import os
filepath = "output.txt"
if os.access(filepath, os.W_OK):
    with open(filepath, "w") as f:
        f.write("data")
else:
    print(f"No write permission for {filepath}")
如何修正：將檔案寫入你自己的目錄（如你的主目錄或專案資料夾）。使用 try/except PermissionError 來處理權限不確定的情況。在 Linux/Mac 上，你可能需要用 chmod 更改檔案權限。

避免技巧：總是寫入你控制的位置 — 你的專案資料夾或專用的輸出目錄。除非你確定有權限，否則避免寫入系統目錄。

24. 整合運用：錯誤處理模式

現在你知道了常見的錯誤，以下是在實際程式碼中 優雅處理它們的模式。

模式 1：基本的 try/except

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")
    result = 0

模式 2：捕獲多個錯誤

try:
    value = int(input("Enter a number: "))
    result = 100 / value
except ValueError:
    print("That's not a valid number!")
except ZeroDivisionError:
    print("You can't divide by zero!")
except Exception as e:
    print(f"Unexpected error: {e}")

模式 3：使用 else 和 finally

try:
    f = open("data.txt", "r")
except FileNotFoundError:
    print("File not found!")
else:
    # 只在沒有引發例外時執行
    content = f.read()
    print(f"Read {len(content)} characters")
    f.close()
finally:
    # 總是執行，不管有沒有例外
    print("File operation complete")

模式 4：帶預設值的安全字典存取

config = {"debug": True, "port": 8080}

# 不冒 KeyError 的風險：
host = config.get("host", "localhost")    # 回傳 "localhost"
port = config.get("port", 3000)           # 回傳 8080（鍵存在）
print(f"Server: {host}:{port}")

模式 5：安全的串列存取

def safe_get(lst, index, default=None):
    """安全地按索引從串列中取得項目。"""
    try:
        return lst[index]
    except IndexError:
        return default

scores = [85, 92, 78]
print(safe_get(scores, 0))     # 85
print(safe_get(scores, 10))    # None
print(safe_get(scores, 10, 0)) # 0
記住：錯誤處理不是防止錯誤永遠不發生 — 而是在錯誤發生時優雅地處理它們。你的程式應該提供有用的錯誤資訊，而不是安靜地或神秘地崩潰。

總結

  • 由下往上閱讀追蹤回溯 — 最後一行告訴你出了什麼問題，上面的行告訴你在哪裡。
  • SyntaxError 和 IndentationError 表示你的程式碼甚至無法被解析 — 檢查冒號、括號、引號和空格。
  • NameError 通常表示打字錯誤或缺少匯入。
  • TypeError 表示你混合了不相容的型別或傳了錯誤的引數數量。
  • ValueError 表示值的類型不對（例如，將 "hello" 轉換為 int）。
  • IndexError 和 KeyError 表示你存取了不存在的東西 — 字典用 .get()，串列檢查 len()。
  • AttributeError 表示你呼叫了該型別上不存在的方法 — 用 type() 檢查型別。
  • FileNotFoundError 和 PermissionError 是檔案系統問題 — 檢查路徑和權限。
  • 使用 try/except 在生產程式碼中優雅地處理錯誤。
  • 不確定時，加入 print() 語句來檢查變數的值和型別。
錯誤是你的朋友，不是敵人。每個錯誤訊息都是 Python 在嘗試幫助你。你遇到和修正的錯誤越多，你學得越快。不要害怕紅色的文字 — 閱讀它、理解它、修正它，然後繼續前進。
