章節 A — 什麼是Python？

如果你來自 MATLAB 背景，或者完全是程式設計的新手， 這個頁面將為你提供全面的概覽：什麼是 Python、為什麼它 已經成為世界上最受歡迎的程式語言之一，以及 它是否適合你想做的事情。

給 MATLAB 使用者的提示：你已經知道如何進行 演算法思維，並且熟悉矩陣、函式和腳本的操作。Python 在許多方面會讓你感到熟悉 — 變數、迴圈、函式、 繪圖 — 但語法和生態系統有所不同。在整個 指南中，我們會指出 MATLAB 的對應概念，讓你能夠將 現有的知識對應到 Python 的世界。

1. 什麼是 Python

Python 是一種高階、直譯式、 通用型程式語言。讓我們來解釋這些詞的含義：

  • 高階 — 你撰寫的程式碼幾乎像 英文一樣易讀。你不需要管理記憶體位址、 CPU 暫存器或其他硬體細節。語言會為你 處理這些（就像 MATLAB 一樣）。
  • 直譯式 — 你可以逐行執行程式碼， 不需要單獨的編譯步驟。你輸入一個 指令，按下 Enter，就能立即看到結果。這 類似於 MATLAB 的命令視窗。
  • 通用型 — Python 不局限於某一個 領域。你可以用它來開發網頁應用、資料分析、自動化、 機器學習、科學計算等等。相比之下，MATLAB 主要是為數值計算而設計的。

簡要歷史

Python 是由荷蘭程式設計師 Guido van Rossum 創建的，於 1991 年首次發布。他以英國喜劇團體 Monty Python 的名字為其命名，而不是以蛇的名字。Guido 的目標是創建一種 有趣且易於閱讀的語言。 如今，Python 由 Python 軟體基金會和一個 龐大的開源社群維護。目前的主要版本是 Python 3（Python 2 已於 2020 年停止支援）。

直譯式 vs. 編譯式 — 簡單的解釋

程式語言通常分為兩大類：

編譯式（C、C++、Rust） 直譯式（Python、MATLAB、JavaScript）
運作方式 你的原始碼在執行之前就被翻譯成機器碼。一個叫做編譯器的特殊程式負責進行翻譯。 你的原始碼在執行時由直譯器逐行讀取和執行。
速度 通常執行時速度較快，因為翻譯已經完成。 通常執行時速度較慢，因為翻譯是即時進行的。
開發速度 較慢 — 每次修改都必須重新編譯。 較快 — 修改一行，立即執行。
錯誤偵測 許多錯誤在編譯時就被發現，在程式執行之前。 錯誤通常在執行時才被發現，當有問題的那行程式碼被執行時。
可以這樣理解：編譯式語言就像在任何人閱讀之前 先把整本書從法文翻譯成英文。直譯式語言就像有一個即時口譯員 在觀眾聆聽的同時逐句翻譯。

2. 為什麼 Python 如此受歡迎

Python 在每個主要調查中（TIOBE 指數、Stack Overflow 開發者調查、GitHub Octoverse）都持續名列 最受歡迎程式語言的前 1–3 名。原因如下：

  1. 可讀、簡潔的語法 — Python 使用 縮排代替大括號，使程式碼視覺上整潔 且易於理解。
  2. 龐大的社群 — 數百萬的開發者 意味著數百萬的教學、Stack Overflow 上的解答，以及 開源套件。
  3. 豐富的函式庫生態系統 — 在 PyPI（Python 套件索引）上有超過 500,000 個 套件。如果你需要做某件事，幾乎 一定有一個函式庫可以使用。
  4. 廣泛應用於業界 — Google、Netflix、 NASA、Instagram、Spotify、JPMorgan 以及數千家其他 組織在生產環境中使用 Python。
  5. 免費且開源 — 永遠不需要授權費用。 相比之下，MATLAB 需要昂貴的 商業授權。
  6. 對初學者友好 — 全球各地的大學 使用 Python 作為入門程式語言。

Hello World — Python vs. C vs. Java

要感受 Python 可讀性的優勢，最好的方式是比較 最簡單的程式 — 在三種語言中印出 "Hello, World!" —：

Python

print("Hello, World!")

就這樣。一行就夠了。

C

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

Java

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Python 讓你專注於你想做什麼，而不是 樣板程式碼。對初學者來說，這意味著你可以在 第一天就開始解決實際問題。

MATLAB 比較：在 MATLAB 中你會寫 disp('Hello, World!') 或 fprintf('Hello, World!\n')。Python 的 print() 本質上是相同的概念 — 一行， 一個函式呼叫。你會感到很熟悉。

3. Python 的應用領域

Python 的多功能性是它最大的優勢之一。以下是 Python 表現出色的主要領域，並附有具體範例：

領域 你可以做什麼 關鍵函式庫／工具
資料科學與分析 清理資料集、計算統計數據、建立視覺化圖表、構建儀表板 pandas、NumPy、Matplotlib、Seaborn、Plotly
機器學習與人工智慧 訓練影像辨識、自然語言處理、推薦系統的模型 scikit-learn、TensorFlow、PyTorch、Keras
網頁開發 建立網站和 REST API（Instagram 的後端就是用 Python 開發的） Django、Flask、FastAPI
自動化與腳本 重新命名 10,000 個檔案、爬取網站、發送自動化電子郵件、產生報告 os、shutil、requests、Beautiful Soup、openpyxl
科學計算 求解微分方程、執行模擬、處理訊號 SciPy、SymPy、NumPy
教育 教授程式設計基礎、建立互動式筆記本 Jupyter Notebook、Google Colab
DevOps 與雲端 自動化伺服器配置、管理雲端基礎設施、撰寫 CI/CD 管道 Ansible、Boto3 (AWS)、Fabric
遊戲原型開發 建立 2D 遊戲和互動式模擬以進行快速原型開發 Pygame、Arcade
桌面應用程式 建立跨平台的圖形介面工具 Tkinter、PyQt、Kivy
為什麼這很重要：如果你學會 Python，你不是 在學習一個單一用途的工具。你正在學習一種可以 從資料分析到網頁開發到自動化到 機器學習都能使用的語言 — 全部使用相同的語法和思維模式。 而使用 MATLAB，你基本上只能侷限在數值計算和 工程模擬的範圍內。

4. Python 的優勢

  • 容易學習 — Python 的語法精簡 且接近自然英語。大多數初學者可以在 幾小時內撰寫出有用的程式。
  • 高度可讀 — 強制縮排和 簡潔的語法使 Python 程式碼在你寫完數月後仍然 容易閱讀（或者閱讀別人的程式碼時也一樣）。
  • 龐大的生態系統 — 套件管理器 pip 讓你可以存取超過五十萬個套件。 需要讀取 Excel 檔案？pip install openpyxl。需要 繪製資料圖表？pip install matplotlib。就是這麼 簡單。
  • 跨平台 — 在 Windows 上撰寫程式碼， 在 macOS 或 Linux 上無需修改即可執行。MATLAB 也能做到， 但 Python 完全免費。
  • 快速開發 — 沒有編譯步驟意味著 你可以快速迭代。Python 非常適合在投入更複雜的實作之前 先進行原型開發。
  • 適合原型開發 — 用 C++ 或 Java 所需 時間的一小部分就能寫出概念驗證。 許多公司先用 Python 進行原型開發，然後在需要時 再優化關鍵部分。
  • 強大的社群支援 — 幾乎你能想到的 任何問題都已經在 Stack Overflow、Reddit 或 Python 社群論壇上被提問和回答過了。
  • 優秀的文件 — 官方 Python 文件、函式庫文件和社群教學 內容豐富且維護良好。

範例：讀取 CSV 檔案

為了說明生態系統如何讓工作變得輕鬆，以下是 讀取 CSV 檔案並計算某一欄平均值的方法 — 這是 資料工作中常見的任務： 

# Python — 讀取 CSV 並計算平均值
import pandas as pd

df = pd.read_csv("data.csv")
average_score = df["score"].mean()
print(f"Average score: {average_score:.2f}")
MATLAB 對應寫法： 
% MATLAB — 讀取 CSV 並計算平均值
T = readtable('data.csv');
averageScore = mean(T.score);
fprintf('Average score: %.2f\n', averageScore);

邏輯幾乎相同。主要差異在於：Python 使用 import 而不是依賴內建的工具箱， 並且使用 f-strings 進行格式化輸出，而不是 fprintf。

5. Python 的弱點

沒有語言是完美的。了解 Python 的 限制很重要，這樣你才能為每項工作選擇正確的工具：

  • 比編譯式語言慢 — 純 Python 程式碼可能比等效的 C 或 C++ 慢 10–100 倍。 對於大多數任務（腳本、資料分析、網頁應用），這並不 重要。對於效能關鍵的內部迴圈，你可以呼叫 優化過的 C 函式庫（NumPy 內部就是這樣做的）或使用 像 Cython 這樣的工具。
  • GIL（全域直譯器鎖） — CPython（標準的 Python 直譯器）有一個鎖，會阻止 CPU 密集型任務進行真正的多執行緒平行處理。這意味著 單一 Python 程序無法完全利用多個 CPU 核心 進行純計算。解決方案包括 multiprocessing 模組和像 Dask 這樣的函式庫。
  • 不適合手機應用程式開發 — 雖然 Kivy 和 BeeWare 等框架存在，但 Python 並不是 建立 iOS 或 Android 應用程式的標準選擇。Swift、Kotlin 或像 Flutter 這樣的跨平台框架更受青睞。
  • 不適合底層系統程式設計 — 用 Python 來撰寫作業系統核心、裝置驅動程式或 嵌入式韌體是不切實際的。C、C++ 和 Rust 才是這方面的首選語言。
  • 執行時錯誤（沒有編譯時型別檢查） — 因為 Python 是動態型別的，某些錯誤 （例如在需要數字的地方傳入字串）只會在 程式碼執行時才被發現。像 mypy 和 型別提示（在 Python 3.5+ 中引入）等工具有助於緩解這個問題， 但它仍然不如 Java 或 C++ 那樣嚴格。
  • 較高的記憶體使用量 — Python 物件 比其 C 語言的對應物佔用更多額外開銷。對於大多數應用程式 這不是問題，但在處理非常大的資料集 或記憶體受限的環境中可能會有影響。
觀點：這些弱點對於 初學者或大多數實際的 Python 使用場景來說很少會造成影響。Python 較慢的速度被大幅縮短的開發時間所彌補。當 效能確實重要時，Python 的生態系統提供了解決方案 （NumPy、Cython、呼叫 C 擴展），讓你能夠兩全其美。

6. Python 與其他語言的比較

以下是 Python 與你可能聽過或使用過的語言的 高層次比較。這是刻意簡化的 — 比較程式語言的內容 可以寫成整本書。

生態系統規模 龐大 中等（工具箱） 大 大 龐大 龐大（npm） 小

MATLAB vs. Python — 關鍵取捨： MATLAB 提供了一個精心打造的、整合式的數值工作環境， 開箱即用。Python 需要組裝套件（NumPy、 SciPy、Matplotlib），但這些套件是免費的、功能極為強大， 而且更廣泛的 Python 生態系統在純數值計算 之外提供了更多的靈活性。如果你的工作範圍超出 矩陣運算 — 例如為資料建立網頁儀表板或 自動化工作流程 — Python 是明顯的贏家。

7. 何時選擇 Python

在以下情況中，Python 是絕佳的選擇：

  • 你想第一次學習程式設計 或為你的技能組合增加一種通用型語言。
  • 你需要分析、清理或視覺化資料 — Python 搭配 pandas 和 Matplotlib 在大多數資料任務上 可以媲美甚至超越 MATLAB。
  • 你想自動化重複性任務 — 重新命名檔案、處理試算表、發送電子郵件、 產生報告。
  • 你正在進入資料科學或機器學習領域 — Python 是這些領域的主導語言。
  • 你需要建立網頁應用程式或 REST API — Django 和 Flask 讓這變得簡單直接。
  • 你想要一個免費的 MATLAB 替代方案來進行 科學計算 — NumPy + SciPy + Matplotlib 涵蓋了 MATLAB 絕大部分的核心功能。
  • 你需要快速建立原型，然後再投入時間 進行更複雜的實作。
  • 你需要跨平台相容性 — 撰寫一次，就能在 Windows、macOS 和 Linux 上執行。
  • 你需要將不同系統串接在一起 — Python 擅長作為「膠水語言」，連接資料庫、 API、檔案系統和其他工具。

範例：自動化任務

假設你有 500 個文字檔案，你需要找到每個 包含「error」這個詞的檔案，並將它們複製到另一個資料夾。在 Python 中： 

import os
import shutil

source = "logs/"
destination = "error_logs/"
os.makedirs(destination, exist_ok=True)

for filename in os.listdir(source):
    filepath = os.path.join(source, filename)
    if filename.endswith(".txt"):
        with open(filepath, "r") as f:
            if "error" in f.read().lower():
                shutil.copy(filepath, destination)
                print(f"Copied: {filename}")

即使你從未見過 Python，這段腳本也很容易閱讀。 它建立一個資料夾、遍歷檔案、檢查是否包含 「error」這個詞，並複製符合條件的檔案。試著用 C 來寫這個 — 需要的程式碼會多得多。

8. 何時不該選擇 Python

Python 並非每種情境的最佳選擇。以下是 其他語言表現更好的場景：

場景 Python 的不足之處 更好的替代方案
手機應用程式開發 原生支援有限，在行動裝置上效能不佳 Swift（iOS）、Kotlin（Android）、Flutter/Dart、React Native
高效能運算（內部迴圈） 純 Python 對於緊湊的數值迴圈來說太慢 C、C++、Rust、Fortran（或在 Python 中使用 NumPy/Cython）
作業系統核心與驅動程式 無法進行底層記憶體控制；依賴執行時直譯器 C、Rust
即時系統 垃圾回收暫停和 GIL 使得確定性時序控制變得困難 C、C++、Ada
瀏覽器前端 瀏覽器原生執行 JavaScript；Python 需要轉譯 JavaScript、TypeScript
3A 遊戲開發 對遊戲引擎來說不夠快（渲染、物理引擎） C++、C#（Unity）、Rust
大型企業系統 動態型別在多人開發的大型程式碼庫中可能成為負擔 Java、C#、Go、TypeScript
重要的細微差別：「不太理想」不代表 「不可能」。人們確實用 Python 建立過手機應用程式（Kivy）、 撰寫過高效能程式碼（NumPy、Cython）、也開發過遊戲 （Pygame）。重點在於其他語言更自然地適合 這些任務。Python 對於你作為初學者會遇到的 絕大多數程式設計需求來說，仍然是絕佳的選擇。

總結

以下是你應該從這個頁面帶走的重點：

  • Python 是一種高階、直譯式、通用型 語言，由 Guido van Rossum 於 1991 年創建。
  • 它之所以受歡迎，是因為它可讀、免費、用途廣泛， 並且有龐大的社群和生態系統支持。
  • Python 被用於資料科學、網頁開發、 自動化、機器學習、科學計算等眾多領域。
  • 它的主要弱點是執行速度和 GIL， 但在大多數使用場景中這些很少會造成實際影響。
  • 從 MATLAB 轉過來，你會發現 Python 的語法很熟悉， 而且它的功能更廣泛且完全免費。
為什麼這很重要：了解 Python 是什麼（以及 不是什麼）能幫助你設定合理的期望並做出明智的 決策。你現在知道 Python 不是最快的語言，但 它是最具生產力的語言之一。你知道它不能做所有事情， 但它可以把大多數事情做好。這個基礎將使你接下來的 學習旅程更加順暢 — 你會理解 Python 為什麼這樣運作，而不僅僅是如何運作。
