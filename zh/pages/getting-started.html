章節 B — Python 入門設定

在你撰寫第一行 Python 程式碼之前，你需要在電腦上安裝 Python，並且對如何執行程式碼有基本的了解。本章節將帶你完成每一個步驟 — 安裝 Python、驗證安裝、以互動式和檔案方式執行程式碼、管理虛擬環境，以及安裝第三方套件。學完之後，你將擁有一個完整可運作的 Python 開發環境。

給 MATLAB 使用者的提示

在 MATLAB 中，所有東西都是捆綁在一起的：你安裝 MATLAB 後就立即擁有編輯器、命令視窗、路徑管理器，以及你購買的所有工具箱。Python 不同。Python 本身只是語言直譯器。你自行選擇編輯器（VS Code、PyCharm 等），使用 pip 單獨管理套件，並使用虛擬環境來組織專案的依賴項目。這種靈活性很強大，但這意味著在開始寫程式之前需要做幾個設定步驟。別擔心 — 一旦你做過一次，就會變成習慣。

1. 如何安裝 Python

Python 是免費且開源的。官方版本可從 python.org/downloads 下載。 請務必下載最新的穩定版本（截至撰寫時為 Python 3.12 或更新版本）。不要安裝 Python 2 — 它已於 2020 年 1 月停止支援。

Windows

  1. 前往 python.org/downloads。
  2. 點擊黃色大按鈕 "Download Python 3.x.x"。這會下載 Windows 安裝程式（.exe 檔案）。
  3. 執行下載的安裝程式。你會看到一個標題為 "Install Python 3.x.x" 的視窗，底部有兩個核取方塊。
  4. 關鍵步驟：在點擊其他任何東西之前，先勾選 "Add python.exe to PATH" 這個選項。這個核取方塊位於安裝程式視窗的最底部，預設是未勾選的。
  5. 點擊 "Install Now"（建議選項）。安裝程式會顯示進度條。
  6. 當你看到 "Setup was successful" 時，點擊 Close。
  7. 開啟一個新的命令提示字元或 PowerShell 視窗（現有的視窗看不到新的 PATH），然後輸入 python --version 來驗證。

警告 — Windows 上的「加入 PATH」

如果你忘記勾選 "Add python.exe to PATH"，Windows 將無法在終端機中辨識 python 指令。你需要重新執行安裝程式並選擇 Modify 來加入它，或者手動將 Python 的安裝目錄（例如 C:\Users\YourName\AppData\Local\Programs\Python\Python312\）加入系統的 PATH 環境變數。為了省去麻煩 — 請務必勾選那個選項。

macOS

  1. 前往 python.org/downloads。
  2. 點擊 "Download Python 3.x.x" 按鈕。這會下載一個 .pkg 安裝程式。
  3. 開啟 .pkg 檔案。你會看到標準的 macOS 安裝精靈。點擊 Continue 跳過簡介、授權合約（點擊 Agree），以及目的地選擇。
  4. 點擊 Install。你可能需要輸入管理員密碼。
  5. 安裝程式完成後，會開啟一個 Finder 視窗顯示 Python 資料夾。你可以關閉它。
  6. 開啟終端機（位於「應用程式」>「工具程式」），輸入 python3 --version 來驗證。

注意：macOS 可能預裝了較舊的系統 Python（舊版本上的 Python 2，或者一個提示你安裝開發工具的佔位程式）。在 macOS 上請務必使用 python3 指令，以確保你執行的是 Python 3。

使用 Homebrew 的替代方法：

# 如果你還沒有 Homebrew，先安裝它：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 然後安裝 Python：
brew install python

Linux

大多數 Linux 發行版都預裝了 Python 3。先檢查一下：

python3 --version

如果沒有安裝 Python 3，或者你想要更新的版本，請使用你的發行版的套件管理器：

# Ubuntu / Debian
sudo apt update
sudo apt install python3 python3-pip python3-venv

# Fedora
sudo dnf install python3 python3-pip

# Arch Linux
sudo pacman -S python python-pip

提示 — 該安裝哪個 Python 版本？

請務必安裝最新的穩定 Python 3 版本。截至撰寫時，建議使用 Python 3.12+。避免使用 Python 3.6 或更舊的版本，因為許多現代函式庫已不再支援它們。你可以在 devguide.python.org/versions 查看目前支援的版本。

2. 如何檢查你的 Python 版本

安裝完成後，開啟終端機（Windows 上的命令提示字元或 PowerShell，macOS/Linux 上的終端機），執行以下其中一個指令： 

# 檢查 Python 版本
python --version
# 輸出：Python 3.12.4

# 在 macOS/Linux 上你可能需要使用 python3
python3 --version
# 輸出：Python 3.12.4

如果你在 Windows 上看到類似 'python' is not recognized as an internal or external command 的錯誤，表示 Python 不在你的 PATH 中。請回到上面的安裝步驟。

找出 Python 的安裝位置

# Windows — 找出 Python 可執行檔的位置
where python
# 輸出：C:\Users\YourName\AppData\Local\Programs\Python\Python312\python.exe

# macOS / Linux — 找出 Python 可執行檔的位置
which python3
# 輸出：/usr/local/bin/python3

當你安裝了多個 Python 版本，想要確認終端機正在使用哪一個時，這些指令非常有用。

同時檢查 pip 版本

# 檢查 pip 版本
pip --version
# 輸出：pip 24.0 from ...site-packages/pip (python 3.12)

# 或在 macOS/Linux 上
pip3 --version

3. 如何從終端機執行 Python（互動模式）

Python 有一個內建的互動式直譯器，通常稱為 REPL（讀取-求值-輸出 迴圈）。這類似於 MATLAB 的命令視窗 — 你輸入一個表達式，按下 Enter，就能立即看到結果。

啟動互動式直譯器

# 啟動互動式 Python
python

# 或在 macOS/Linux 上
python3

你會看到類似這樣的內容：

Python 3.12.4 (main, Jun  7 2024, 00:00:00) [GCC 14.1.1] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

>>> 提示符表示 Python 正在等待你的輸入。試著輸入一些表達式： 

>>> 2 + 3
5
>>> "Hello, " + "World!"
'Hello, World!'
>>> 10 / 3
3.3333333333333335
>>> 2 ** 10
1024
>>> type(42)
<class 'int'>
>>> import math
>>> math.sqrt(144)
12.0
>>> [x ** 2 for x in range(5)]
[0, 1, 4, 9, 16]

離開互動式直譯器

有幾種方式可以離開：

>>> exit()       # 內建函式 — 在任何地方都有效
>>> quit()       # 同樣在任何地方都有效
# 或使用快捷鍵：
# Ctrl+Z 然後 Enter（Windows）
# Ctrl+D           （macOS / Linux）

互動模式 vs. 腳本

互動式直譯器非常適合快速實驗和測試小段程式碼，就像 MATLAB 的命令視窗一樣。然而，對於超過幾行的程式碼，你應該將程式碼寫在 .py 檔案（即「腳本」）中，然後從終端機執行它。我們接下來會介紹這個部分。

4. 如何執行 .py 檔案

真正的 Python 開發是在 .py 檔案中進行的 — 包含 Python 程式碼的純文字檔案。以下是完整的工作流程：

步驟 1：建立 Python 檔案

開啟任何文字編輯器（VS Code、Notepad++、Sublime Text，甚至是記事本），然後輸入以下內容： 

# hello.py — 我的第一個 Python 腳本

name = "World"
print(f"Hello, {name}!")
print("Welcome to Python.")
print(f"2 + 3 = {2 + 3}")

將這個檔案儲存為 hello.py，放在你容易找到的資料夾中（例如 Windows 上的 C:\Users\YourName\projects\hello.py 或 macOS/Linux 上的 ~/projects/hello.py）。

警告 — 副檔名很重要

確保檔案儲存時使用 .py 副檔名，而不是 .py.txt。如果你使用 Windows 記事本，在「存檔類型」下拉選單中選擇 "所有檔案 (*.*)"，然後輸入檔名為 hello.py。更好的做法是使用像 VS Code（免費）這樣的專業程式碼編輯器。

步驟 2：導覽到檔案所在的資料夾

# Windows
cd C:\Users\YourName\projects

# macOS / Linux
cd ~/projects

步驟 3：執行腳本

# Windows
python hello.py

# macOS / Linux
python3 hello.py

你應該會看到以下輸出：

Hello, World!
Welcome to Python.
2 + 3 = 5

工作流程摘要

  1. 在文字編輯器中編輯你的 .py 檔案。
  2. 儲存檔案。
  3. 切換到終端機。
  4. 使用 python filename.py 執行。
  5. 閱讀輸出（或錯誤訊息）。
  6. 重複 — 回到步驟 1 進行修改。

這個「編輯-儲存-執行」循環是 Python 的基本工作流程。像 VS Code 這樣的編輯器讓你可以用一個按鍵就執行腳本，但在底層，它做的是同樣的事情。

5. 如何使用虛擬環境

為什麼這很重要

想像你有兩個專案：專案 A 需要 numpy 1.24，而專案 B 需要 numpy 1.26。如果你將套件安裝在全域（整個系統），你一次只能有一個版本的 numpy，在專案之間切換會變成一場惡夢。虛擬環境解決了這個問題。每個專案都有自己獨立的套件資料夾，所以專案 A 和專案 B 可以各自擁有它們需要的精確版本，而不會有任何衝突。

你可以將虛擬環境想成是給每個專案一份 Python 及其套件的私有副本。這是作為 Python 程式設計師最重要的習慣之一。每個專案都要使用虛擬環境。

建立虛擬環境

Python 3 內建了 venv 模組（不需要額外安裝）。導覽到你的專案資料夾並執行： 

# 建立一個名為 "myenv" 的虛擬環境
python -m venv myenv

# 或在 macOS/Linux 上
python3 -m venv myenv

這會在你的專案目錄中建立一個名為 myenv 的新資料夾。這個資料夾包含一份 Python 直譯器的副本、一個 pip 可執行檔，以及一個 site-packages 目錄，安裝的套件會放在這裡。你通常會將它命名為 venv、.venv 或 env — 任何名稱都可以。

啟動虛擬環境

建立環境還不夠 — 你必須啟動它，這樣你的終端機才會使用環境內的 Python 和 pip，而不是全域的。 

# Windows（命令提示字元）
myenv\Scripts\activate

# Windows（PowerShell）
myenv\Scripts\Activate.ps1

# Windows（Git Bash）
source myenv/Scripts/activate

# macOS / Linux
source myenv/bin/activate

當環境啟動時，你會在終端機提示符的開頭看到環境名稱（用括號括起來）： 

(myenv) C:\Users\YourName\projects>
(myenv) user@machine:~/projects$

環境啟動期間，python 和 pip 指的是虛擬環境內的版本 — 而不是全域的。

停用虛擬環境

# 在所有平台上都有效 — 直接輸入：
deactivate

(myenv) 前綴會消失，你的終端機會恢復使用全域的 Python。

完整的虛擬環境工作流程

# 1. 建立專案資料夾
mkdir my_project
cd my_project

# 2. 建立虛擬環境
python -m venv venv

# 3. 啟動虛擬環境
# Windows：  venv\Scripts\activate
# Mac/Linux：source venv/bin/activate

# 4. 安裝套件（只會安裝到虛擬環境中）
pip install numpy requests

# 5. 進行你的工作...
python my_script.py

# 6. 完成後，停用虛擬環境
deactivate

警告 — 不要將 venv 資料夾提交到版本控制

虛擬環境資料夾（venv/、myenv/ 等）可能有數百 MB，而且是特定於你的作業系統的。永遠不要將它加入版本控制（Git）。相反，將它加入你的 .gitignore 檔案。其他人可以使用你的 requirements.txt（下一節會介紹）來重新建立環境。

MATLAB 比較

MATLAB 沒有虛擬環境。當你安裝一個工具箱，它會對你電腦上的每個 MATLAB 專案都可用。這比較簡單，但可能會導致版本衝突。Python 的虛擬環境系統讓你能夠按專案精細控制依賴項目 — 這對可重現性和協作來說是一個重大優勢。

6. 如何使用 pip 安裝套件

Python 的標準函式庫非常廣泛，但生態系統真正的力量來自第三方套件 — 由社群撰寫並發布在 PyPI（Python 套件索引）上的函式庫。有超過 500,000 個套件可用。你使用 pip（Python 的套件安裝程式）來安裝它們。

安裝套件

# 確保你的虛擬環境已經啟動！

# 安裝單一套件
pip install numpy

# 安裝特定版本
pip install numpy==1.26.4

# 一次安裝多個套件
pip install numpy requests matplotlib pandas

pip 會從 PyPI 下載套件並安裝它（以及它所需的任何依賴項目）。你會看到顯示下載進度的輸出，以及類似 Successfully installed numpy-1.26.4 的成功訊息。

列出已安裝的套件

# 查看所有已安裝的套件及其版本
pip list

# 範例輸出：
# Package    Version
# ---------- -------
# numpy      1.26.4
# pip        24.0
# requests   2.31.0
# setuptools 69.5.1

將依賴項目凍結到 requirements.txt

# 匯出所有已安裝套件的精確版本
pip freeze > requirements.txt

# 查看檔案
cat requirements.txt
# 輸出：
# certifi==2024.2.2
# charset-normalizer==3.3.2
# numpy==1.26.4
# requests==2.31.0
# urllib3==2.2.1

requirements.txt 檔案是一個包含固定版本套件的純文字列表。將這個檔案分享給協作者或放入你的 Git 儲存庫，這樣任何人都可以重新建立你的精確環境： 

# 其他人（或你在新電腦上）可以用以下指令安裝所有套件：
pip install -r requirements.txt

升級和解除安裝套件

# 將套件升級到最新版本
pip install --upgrade numpy

# 升級 pip 本身
pip install --upgrade pip

# 解除安裝套件
pip uninstall requests

搜尋套件

要尋找套件，請在你的網頁瀏覽器中直接瀏覽 pypi.org。以下是作為 MATLAB 使用者轉換到 Python 時會遇到的一些熱門套件：

套件用途MATLAB 對應功能
numpy數值陣列和數學運算MATLAB 核心陣列
scipy科學計算（最佳化、積分等）MATLAB 內建函式
matplotlib繪圖和視覺化MATLAB 繪圖函式

7. 初學者常犯的設定錯誤

幾乎每個 Python 初學者都會遇到以下一個或多個問題。現在先讀過它們，這樣你就能避免日後數小時的挫折。

錯誤 1 — 未將 Python 加入 PATH（Windows）

症狀：你在終端機中輸入 python，卻出現 'python' is not recognized as an internal or external command'。

解決方法：重新執行 Python 安裝程式，選擇 Modify，確保 PATH 選項已勾選。或者，使用 python.exe 的完整路徑（例如 C:\Users\YourName\AppData\Local\Programs\Python\Python312\python.exe），但這很麻煩。

錯誤 2 — 使用了錯誤的 Python 版本

症狀：使用現代功能（f-strings、海象運算子、match 語句）的程式碼拋出 SyntaxError。

解決方法：執行 python --version 來檢查。如果你看到 Python 2.x，你需要安裝 Python 3 並使用 python3 指令。如果你看到 Python 3.6 或更舊的版本，請升級到最新版本。

錯誤 3 — 混淆 pip 和 pip3

症狀：你用 pip install numpy 安裝了套件，但當你用 python3 執行腳本時，卻出現 ModuleNotFoundError: No module named 'numpy'。

解決方法：在某些系統上，pip 可能連結到 Python 2，而 pip3 連結到 Python 3。最安全的安裝套件方式是：python3 -m pip install numpy。這能保證套件安裝到你實際執行的同一個 Python 版本中。

錯誤 4 — 將套件安裝在全域而非虛擬環境中

症狀：專案之間的套件版本衝突。系統 Python 在一次錯誤安裝後損壞。權限錯誤（ERROR: Could not install packages due to an EnvironmentError）。

解決方法：在執行 pip install 之前，務必先建立並啟動虛擬環境。確認你的提示符顯示 (venv) 後再安裝。

錯誤 5 — 忘記啟動虛擬環境

症狀：你建立了虛擬環境，也在其中安裝了套件，但你的腳本找不到它們。

解決方法：確認你的終端機提示符中有顯示 (venv) 或 (myenv)。如果沒有，再次執行啟動指令。你每次開啟新的終端機視窗時都必須啟動環境。

錯誤 6 — 在 macOS/Linux 上執行 python 而非 python3

症狀：在某些 macOS/Linux 系統上，python 指向 Python 2（或根本不存在），而 python3 才是正確的指令。

解決方法：除非你在虛擬環境內（在虛擬環境中 python 始終是正確的），否則在 macOS/Linux 上請務必使用 python3 和 pip3。

錯誤 7 — 修改了程式碼但執行的是舊版本

症狀：你修改了程式碼，但輸出結果沒有改變。

解決方法：確認你在重新執行之前已儲存了檔案。這是一個出乎意料的常見疏忽，特別是對於習慣了編輯器在執行前自動儲存的 MATLAB 使用者。

8. python、python3、pip、pip3、venv 和 conda 的差異

這些術語在你剛入門時可能會令人困惑。以下是一個清楚的比較：

指令／工具 是什麼 何時使用 備註
python Python 直譯器 執行腳本、啟動 REPL 在 Windows 上，這通常是 Python 3。在 macOS/Linux 上，較舊的系統上可能指向 Python 2，或者根本不存在。
python3 Python 3 直譯器（明確指定） 在 macOS/Linux 上執行腳本 始終指向 Python 3。在 macOS/Linux 上使用這個指令比較安全。在虛擬環境內，python 和 python3 是相同的。
pip Python 套件安裝程式 安裝第三方套件 在某些系統上可能連結到 Python 2。最安全的用法：python -m pip install ...
pip3 Python 3 的套件安裝程式（明確指定） 在 macOS/Linux 上安裝套件 保證套件安裝到 Python 3。在虛擬環境內，pip 和 pip3 是等效的。
venv Python 內建的虛擬環境模組 建立隔離的專案環境 Python 3.3+ 內建。輕量級，不需要額外安裝。用法：python -m venv myenv
conda 第三方套件和環境管理器（來自 Anaconda/Miniconda） 資料科學專案、管理複雜的依賴項目 同時管理 Python 套件和非 Python 依賴項目（例如 C 函式庫）。比 venv 重。在資料科學領域很受歡迎。你不需要 conda 就能開始學習 Python — venv + pip 就足夠了。
virtualenv 第三方虛擬環境工具 較舊的專案或需要額外功能時 比 venv 更早出現。建立環境的速度更快，支援較舊的 Python。對大多數初學者來說，內建的 venv 就夠用了。
pyenv 管理多個 Python 版本的工具 當你需要多個 Python 版本並存時 不是套件管理器 — 它管理的是 Python 本身。對於需要在不同 Python 版本的專案上工作的開發者很有用。僅限 macOS/Linux（有一個 Windows 版本叫做 pyenv-win）。

給初學者的建議設定

不要把事情搞得太複雜。在你學習 Python 的頭幾個月，你只需要：

  • 從 python.org 安裝的 Python 3
  • 用於虛擬環境的 venv（內建，不需要安裝）
  • 用於安裝套件的 pip（隨 Python 一起安裝）
  • 作為文字編輯器的 VS Code（免費，有優秀的 Python 支援）

你可以在有具體需求時再去探索 conda、pyenv 和其他工具。

總結

在這個章節中，你學到了如何：

  • 在 Windows、macOS 和 Linux 上安裝 Python（以及為什麼 Windows 上的「加入 PATH」核取方塊很重要）。
  • 使用 python --version 和 where/which 驗證你的安裝。
  • 使用互動式直譯器進行快速實驗，以及如何離開它。
  • 使用「編輯-儲存-執行」工作流程撰寫並執行 .py 檔案。
  • 使用 venv 建立虛擬環境來隔離專案的依賴項目。
  • 使用 pip 安裝套件，並使用 requirements.txt 管理依賴項目。
  • 避免常見的設定錯誤，這些錯誤會讓大多數初學者受挫。
  • 區分 python、python3、pip、pip3、venv 和 conda。
