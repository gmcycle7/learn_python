Dictionaries

Dictionaries are one of the most powerful and frequently used data structures in Python. If lists let you store ordered sequences of values, dictionaries let you store key-value pairs — connecting a label (the key) to a piece of data (the value). Once you understand dictionaries, you will see them everywhere: configuration files, JSON data, function keyword arguments, and more.

MATLAB comparison: MATLAB has containers.Map, which works similarly — you store values indexed by keys rather than by numeric position. However, containers.Map is relatively niche in MATLAB code. In Python, dictionaries (dict) are absolutely central to the language. They are used everywhere — for JSON data, function **kwargs, class attributes under the hood, module namespaces, and much more. Mastering dictionaries is essential.

1. What is a Dictionary?

Think of a real-world dictionary: you look up a word (the key) and find its definition (the value). A Python dictionary works the same way — every entry is a key-value pair, and you use the key to retrieve the corresponding value.

Creating a Dictionary with {}

The most common way to create a dictionary is with curly braces, using colons to separate keys from values: 

# A simple dictionary: English word -> Spanish translation
english_to_spanish = {
    "hello": "hola",
    "goodbye": "adiós",
    "please": "por favor",
    "thank you": "gracias"
}

print(english_to_spanish)
# {'hello': 'hola', 'goodbye': 'adiós', 'please': 'por favor', 'thank you': 'gracias'}

Creating a Dictionary with dict()

You can also create a dictionary using the built-in dict() constructor. This works well when your keys are simple strings that look like valid Python variable names: 

# Using dict() with keyword arguments
person = dict(name="Alice", age=30, city="London")
print(person)
# {'name': 'Alice', 'age': 30, 'city': 'London'}

# Using dict() with a list of tuples
pairs = [("x", 10), ("y", 20), ("z", 30)]
coordinates = dict(pairs)
print(coordinates)
# {'x': 10, 'y': 20, 'z': 30}

Empty Dictionary

# Two ways to create an empty dictionary
empty1 = {}
empty2 = dict()

print(type(empty1))  # <class 'dict'>
print(len(empty1))   # 0
Watch out: {} creates an empty dictionary, not an empty set. To create an empty set, you must use set(). This is a common source of confusion for beginners.

Key Rules: Keys Must Be Immutable

Dictionary keys must be immutable (unchangeable) types. This means you can use:

  • Strings — the most common key type
  • Numbers (int, float) — perfectly valid
  • Tuples — only if they contain immutable elements
  • Booleans — technically valid but rarely useful

You cannot use lists, dictionaries, or sets as keys because they are mutable:

# Valid keys
valid = {
    "name": "Alice",       # string key
    42: "the answer",      # integer key
    3.14: "pi",            # float key
    (1, 2): "coordinates"  # tuple key
}

# Invalid key - this will raise a TypeError
# bad = {[1, 2]: "a list as key"}  # TypeError: unhashable type: 'list'
Why must keys be immutable? Python uses a technique called hashing to look up keys quickly. A hash is a fixed-size number computed from the key's value. If the key could change after being added (like a list that gets an item appended), its hash would change too, and Python would no longer be able to find it. Immutable types guarantee that the hash stays the same.

Values, on the other hand, can be anything — strings, numbers, lists, other dictionaries, functions, objects, etc.

2. Accessing Values

There are two main ways to get a value out of a dictionary: bracket notation and the .get() method. Choosing the right one depends on whether you expect the key to always be present.

Bracket Notation: dict[key]

Use square brackets with the key to retrieve a value. If the key does not exist, Python raises a KeyError: 

student = {"name": "Bob", "grade": "A", "age": 21}

print(student["name"])   # Bob
print(student["grade"])  # A

# Trying to access a key that doesn't exist:
# print(student["email"])  # KeyError: 'email'

The .get() Method: Safe Access

The .get() method returns None if the key is missing, instead of crashing your program: 

student = {"name": "Bob", "grade": "A", "age": 21}

# .get() returns None if the key doesn't exist
print(student.get("email"))       # None
print(student.get("name"))        # Bob

# You can provide a default value as the second argument
print(student.get("email", "N/A"))    # N/A
print(student.get("grade", "N/A"))    # A  (key exists, so default is ignored)
When to use which:
  • Use dict[key] when you are certain the key exists, or when a missing key indicates a bug that should crash loudly.
  • Use dict.get(key) when the key might be missing and you want to handle that gracefully.
  • Use dict.get(key, default) when you want a fallback value for missing keys.

3. Modifying Dictionaries

Dictionaries are mutable — you can add, change, and remove key-value pairs after creation.

Adding and Updating Values

student = {"name": "Bob", "age": 21}

# Add a new key-value pair
student["email"] = "bob@example.com"
print(student)
# {'name': 'Bob', 'age': 21, 'email': 'bob@example.com'}

# Update an existing value
student["age"] = 22
print(student)
# {'name': 'Bob', 'age': 22, 'email': 'bob@example.com'}
Note: The syntax for adding and updating is the same — dict[key] = value. If the key already exists, the old value is replaced. If it does not exist, a new entry is created.

The .update() Method

Use .update() to merge another dictionary (or an iterable of key-value pairs) into an existing dictionary. Existing keys are overwritten; new keys are added: 

student = {"name": "Bob", "age": 21}

# Update with another dictionary
student.update({"age": 22, "grade": "A", "email": "bob@example.com"})
print(student)
# {'name': 'Bob', 'age': 22, 'grade': 'A', 'email': 'bob@example.com'}

# Update with keyword arguments
student.update(city="New York", age=23)
print(student)
# {'name': 'Bob', 'age': 23, 'grade': 'A', 'email': 'bob@example.com', 'city': 'New York'}

Removing Entries

Python offers several ways to remove key-value pairs from a dictionary: 

student = {"name": "Bob", "age": 22, "grade": "A", "email": "bob@example.com"}

# del — removes a specific key (raises KeyError if missing)
del student["email"]
print(student)
# {'name': 'Bob', 'age': 22, 'grade': 'A'}

# .pop(key) — removes and returns the value (raises KeyError if missing)
grade = student.pop("grade")
print(grade)     # A
print(student)   # {'name': 'Bob', 'age': 22}

# .pop(key, default) — returns default instead of raising KeyError
missing = student.pop("phone", "not found")
print(missing)   # not found

# .popitem() — removes and returns the last inserted key-value pair
student["city"] = "London"
student["year"] = 3
last = student.popitem()
print(last)      # ('year', 3)
print(student)   # {'name': 'Bob', 'age': 22, 'city': 'London'}

# .clear() — removes ALL entries
student.clear()
print(student)   # {}
Common mistake: Using del dict[key] without checking if the key exists first. If the key is missing, you get a KeyError. Either check with if key in dict first, or use dict.pop(key, None) for safe removal.

4. Dictionary Methods

.keys(), .values(), and .items()

These three methods give you different views of a dictionary's contents: 

fruit_colors = {"apple": "red", "banana": "yellow", "grape": "purple"}

# .keys() — all keys
print(fruit_colors.keys())
# dict_keys(['apple', 'banana', 'grape'])

# .values() — all values
print(fruit_colors.values())
# dict_values(['red', 'yellow', 'purple'])

# .items() — all key-value pairs as tuples
print(fruit_colors.items())
# dict_items([('apple', 'red'), ('banana', 'yellow'), ('grape', 'purple')])

# Convert to a list if needed
keys_list = list(fruit_colors.keys())
print(keys_list)  # ['apple', 'banana', 'grape']
These are views, not copies. The objects returned by .keys(), .values(), and .items() are views that reflect changes made to the dictionary. If you add a key to the dictionary, the view will automatically include it.

.copy() — Shallow Copy

original = {"a": 1, "b": 2, "c": 3}
duplicate = original.copy()

duplicate["d"] = 4
print(original)   # {'a': 1, 'b': 2, 'c': 3}  (unchanged)
print(duplicate)  # {'a': 1, 'b': 2, 'c': 3, 'd': 4}
Shallow copy warning: .copy() creates a shallow copy. If your dictionary contains mutable values (like lists or other dictionaries), those inner objects are shared, not duplicated. Use import copy; copy.deepcopy(dict) if you need a fully independent copy.

Checking if a Key Exists with in

student = {"name": "Bob", "age": 22}

print("name" in student)     # True
print("email" in student)    # False
print("Bob" in student)      # False — "in" checks KEYS, not values

# Common pattern: check before accessing
if "email" in student:
    print(student["email"])
else:
    print("No email on file")
MATLAB comparison: In MATLAB with containers.Map, you use isKey(map, key) to check if a key exists. In Python, the in keyword reads more naturally: if key in dict.

len() — Number of Key-Value Pairs

student = {"name": "Bob", "age": 22, "grade": "A"}
print(len(student))  # 3

5. Looping Over Dictionaries

You will frequently need to iterate over dictionaries. Python gives you several clean ways to do this.

Loop Over Keys (Default)

fruit_colors = {"apple": "red", "banana": "yellow", "grape": "purple"}

# By default, iterating over a dict gives you the keys
for fruit in fruit_colors:
    print(fruit)

# Output:
# apple
# banana
# grape

Loop Over Keys and Values Together

fruit_colors = {"apple": "red", "banana": "yellow", "grape": "purple"}

# .items() gives you both key and value
for fruit, color in fruit_colors.items():
    print(f"The {fruit} is {color}")

# Output:
# The apple is red
# The banana is yellow
# The grape is purple
Why .items() is the most common pattern: Most of the time when you loop over a dictionary, you need both the key and the value. Using for key, value in dict.items() is cleaner and more efficient than looping over keys and then using dict[key] to get each value.

Loop Over Values Only

scores = {"Alice": 95, "Bob": 87, "Charlie": 92}

# If you only care about the values
for score in scores.values():
    print(score)

# Output:
# 95
# 87
# 92

# Useful for calculations
total = sum(scores.values())
average = total / len(scores)
print(f"Average score: {average:.1f}")  # Average score: 91.3

6. Nested Dictionaries

Dictionary values can be anything — including other dictionaries. This is called nesting and is extremely common in real-world data (think JSON).

Dictionaries Inside Dictionaries

# Student records — each student is a nested dictionary
students = {
    "alice": {
        "full_name": "Alice Johnson",
        "age": 22,
        "grades": {"math": 95, "physics": 88, "cs": 92}
    },
    "bob": {
        "full_name": "Bob Smith",
        "age": 21,
        "grades": {"math": 78, "physics": 85, "cs": 90}
    }
}

Accessing Nested Values

# Chain square brackets to drill into nested dictionaries
print(students["alice"]["full_name"])           # Alice Johnson
print(students["bob"]["grades"]["physics"])     # 85

# Safer approach with .get() for deeply nested data
math_grade = students.get("charlie", {}).get("grades", {}).get("math", "N/A")
print(math_grade)  # N/A  (charlie doesn't exist)

Real-World Example: Configuration File

# Application configuration (similar to what you'd find in a JSON config)
config = {
    "database": {
        "host": "localhost",
        "port": 5432,
        "name": "myapp_db",
        "credentials": {
            "user": "admin",
            "password": "secret123"
        }
    },
    "logging": {
        "level": "INFO",
        "file": "/var/log/myapp.log",
        "max_size_mb": 50
    },
    "features": {
        "dark_mode": True,
        "notifications": False
    }
}

# Access deeply nested values
db_host = config["database"]["host"]
log_level = config["logging"]["level"]
print(f"Connecting to {db_host}, log level: {log_level}")
# Connecting to localhost, log level: INFO
JSON and dictionaries: JSON (JavaScript Object Notation) is the most common data format for web APIs. When you load JSON data in Python using import json, it becomes a dictionary (or a list of dictionaries). This is one of the reasons dictionaries are so important to learn.

7. Dictionary Comprehensions

Just like list comprehensions, Python supports dictionary comprehensions — a concise way to create dictionaries from iterables.

Basic Syntax

# Syntax: {key_expr: value_expr for item in iterable}

# Create a dictionary of squares
squares = {n: n ** 2 for n in range(1, 6)}
print(squares)
# {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

Practical Examples

# Convert a list of words to a word-length dictionary
words = ["hello", "world", "python", "code"]
word_lengths = {word: len(word) for word in words}
print(word_lengths)
# {'hello': 5, 'world': 5, 'python': 6, 'code': 4}

# Filter: only include items that meet a condition
scores = {"Alice": 95, "Bob": 67, "Charlie": 82, "Diana": 91}
high_scores = {name: score for name, score in scores.items() if score >= 85}
print(high_scores)
# {'Alice': 95, 'Diana': 91}

# Swap keys and values
original = {"a": 1, "b": 2, "c": 3}
swapped = {v: k for k, v in original.items()}
print(swapped)
# {1: 'a', 2: 'b', 3: 'c'}

# Transform values: convert Celsius to Fahrenheit
celsius = {"London": 15, "Paris": 18, "Tokyo": 22, "Sydney": 25}
fahrenheit = {city: round(temp * 9/5 + 32, 1) for city, temp in celsius.items()}
print(fahrenheit)
# {'London': 59.0, 'Paris': 64.4, 'Tokyo': 71.6, 'Sydney': 77.0}
MATLAB comparison: MATLAB does not have dictionary comprehensions. You would typically use a loop with containers.Map or build a struct field by field. Python comprehensions let you build an entire dictionary in a single, readable expression.

8. Common Use Cases

Here are four patterns that come up again and again in real Python code. Mastering these will make you a more effective programmer.

Counting Occurrences

One of the most common uses for a dictionary is counting how many times each item appears in a sequence: 

# Count character frequency in a string
text = "mississippi"
freq = {}
for char in text:
    freq[char] = freq.get(char, 0) + 1

print(freq)
# {'m': 1, 'i': 4, 's': 4, 'p': 2}

# Or use the built-in collections.Counter (shortcut!)
from collections import Counter
freq2 = Counter(text)
print(freq2)
# Counter({'i': 4, 's': 4, 'p': 2, 'm': 1})
Why freq.get(char, 0) + 1 works: If the character is not yet in the dictionary, .get(char, 0) returns 0. We add 1 to it, so the first occurrence of any character starts at 1. On subsequent encounters, the current count is retrieved and incremented. This is a clean alternative to checking if char in freq first.

Grouping Data

# Group students by their grade
students = [
    ("Alice", "A"), ("Bob", "B"), ("Charlie", "A"),
    ("Diana", "B"), ("Eve", "A"), ("Frank", "C")
]

groups = {}
for name, grade in students:
    if grade not in groups:
        groups[grade] = []
    groups[grade].append(name)

print(groups)
# {'A': ['Alice', 'Charlie', 'Eve'], 'B': ['Bob', 'Diana'], 'C': ['Frank']}

# Cleaner version with .setdefault()
groups2 = {}
for name, grade in students:
    groups2.setdefault(grade, []).append(name)

print(groups2)
# {'A': ['Alice', 'Charlie', 'Eve'], 'B': ['Bob', 'Diana'], 'C': ['Frank']}

Looking Up Values (Replacing Long if/elif Chains)

# Instead of this:
def get_day_name_ugly(day_num):
    if day_num == 1:
        return "Monday"
    elif day_num == 2:
        return "Tuesday"
    elif day_num == 3:
        return "Wednesday"
    # ... etc.

# Use a dictionary lookup:
day_names = {
    1: "Monday", 2: "Tuesday", 3: "Wednesday",
    4: "Thursday", 5: "Friday", 6: "Saturday", 7: "Sunday"
}

def get_day_name(day_num):
    return day_names.get(day_num, "Invalid day")

print(get_day_name(3))  # Wednesday
print(get_day_name(9))  # Invalid day
Performance note: Dictionary lookups are O(1) on average — constant time regardless of how many keys the dictionary has. A long chain of if/elif statements is O(n) — it gets slower as you add more cases. For large lookup tables, a dictionary is both cleaner and faster.

Configuration Storage

# A common pattern: using a dictionary to store settings
default_settings = {
    "theme": "light",
    "font_size": 14,
    "auto_save": True,
    "language": "en",
    "max_retries": 3
}

# User overrides some settings
user_settings = {
    "theme": "dark",
    "font_size": 16
}

# Merge: user settings override defaults
final_settings = {**default_settings, **user_settings}
print(final_settings)
# {'theme': 'dark', 'font_size': 16, 'auto_save': True, 'language': 'en', 'max_retries': 3}
The ** unpacking operator: The {**dict1, **dict2} syntax creates a new dictionary by unpacking both dictionaries. If they share keys, the second dictionary's values win. This is a clean, Pythonic way to merge dictionaries. In Python 3.9+, you can also use the | operator: dict1 | dict2.

Common Mistakes to Avoid

Mistake What Happens Fix
dict[missing_key] KeyError Use dict.get(key) or check with in
Using a list as a key TypeError: unhashable type Convert to a tuple first: tuple(my_list)
{} for an empty set Creates an empty dict instead Use set() for an empty set
Modifying a dict while looping over it RuntimeError: dictionary changed size during iteration Loop over a copy: for k in list(dict.keys()):
Assuming dictionaries are ordered in older Python Insertion order is only guaranteed in Python 3.7+ Use Python 3.7+ (or collections.OrderedDict)
Checking value in dict Checks keys, not values Use value in dict.values() to check values

Exercises

Test your understanding with these exercises. Try to solve each one on your own before revealing the solution.

Exercise 1: Build a Phone Book

Create a dictionary called phone_book with at least three name-to-phone-number entries. Then print a specific person's number, add a new entry, and delete one entry.

Show Solution 
phone_book = {
    "Alice": "555-1234",
    "Bob": "555-5678",
    "Charlie": "555-9012"
}

# Print Bob's number
print(phone_book["Bob"])  # 555-5678

# Add a new entry
phone_book["Diana"] = "555-3456"

# Delete Charlie
del phone_book["Charlie"]

print(phone_book)
# {'Alice': '555-1234', 'Bob': '555-5678', 'Diana': '555-3456'}

Exercise 2: Word Frequency Counter

Given the string "the cat sat on the mat the cat", write code that counts how many times each word appears. Store the result in a dictionary.

Show Solution 
sentence = "the cat sat on the mat the cat"
words = sentence.split()

word_count = {}
for word in words:
    word_count[word] = word_count.get(word, 0) + 1

print(word_count)
# {'the': 3, 'cat': 2, 'sat': 1, 'on': 1, 'mat': 1}

Exercise 3: Invert a Dictionary

Write code that takes the dictionary {"a": 1, "b": 2, "c": 3} and creates a new dictionary where the keys and values are swapped: {1: "a", 2: "b", 3: "c"}. Use a dictionary comprehension.

Show Solution 
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
print(inverted)
# {1: 'a', 2: 'b', 3: 'c'}

Exercise 4: Merge Two Dictionaries

You have two dictionaries: 

defaults = {"color": "blue", "size": 10, "visible": True}
overrides = {"color": "red", "border": 2}

Create a new dictionary that contains all keys from both, where overrides takes priority over defaults for shared keys.

Show Solution 
defaults = {"color": "blue", "size": 10, "visible": True}
overrides = {"color": "red", "border": 2}

# Method 1: ** unpacking
merged = {**defaults, **overrides}
print(merged)
# {'color': 'red', 'size': 10, 'visible': True, 'border': 2}

# Method 2: .update() (modifies defaults in place)
defaults_copy = defaults.copy()
defaults_copy.update(overrides)
print(defaults_copy)
# {'color': 'red', 'size': 10, 'visible': True, 'border': 2}

# Method 3: | operator (Python 3.9+)
# merged = defaults | overrides

Exercise 5: Student Grade Averages

Given the following nested dictionary of student grades, compute and print each student's average grade: 

students = {
    "Alice": {"math": 90, "science": 85, "english": 92},
    "Bob": {"math": 78, "science": 82, "english": 88},
    "Charlie": {"math": 95, "science": 91, "english": 87}
}
Show Solution 
students = {
    "Alice": {"math": 90, "science": 85, "english": 92},
    "Bob": {"math": 78, "science": 82, "english": 88},
    "Charlie": {"math": 95, "science": 91, "english": 87}
}

for name, grades in students.items():
    avg = sum(grades.values()) / len(grades)
    print(f"{name}: average = {avg:.1f}")

# Output:
# Alice: average = 89.0
# Bob: average = 82.7
# Charlie: average = 91.0

Exercise 6: Group by First Letter

Given a list of words, group them by their first letter into a dictionary. For example, ["apple", "banana", "avocado", "blueberry", "cherry", "apricot"] should produce: 

{"a": ["apple", "avocado", "apricot"], "b": ["banana", "blueberry"], "c": ["cherry"]}
Show Solution 
words = ["apple", "banana", "avocado", "blueberry", "cherry", "apricot"]

groups = {}
for word in words:
    first_letter = word[0]
    groups.setdefault(first_letter, []).append(word)

print(groups)
# {'a': ['apple', 'avocado', 'apricot'], 'b': ['banana', 'blueberry'], 'c': ['cherry']}

# Alternative using a regular if check:
groups2 = {}
for word in words:
    letter = word[0]
    if letter not in groups2:
        groups2[letter] = []
    groups2[letter].append(word)

Summary

  • A dictionary stores key-value pairs. Keys must be immutable; values can be anything.
  • Use dict[key] when you are sure the key exists; use dict.get(key, default) for safe access.
  • .keys(), .values(), and .items() give you views of the dictionary's contents.
  • The most common loop pattern is for key, value in dict.items().
  • Dictionaries can be nested — values can be other dictionaries, lists, etc.
  • Dictionary comprehensions let you build dictionaries concisely from iterables.
  • Common patterns include counting, grouping, lookups, and configuration storage.
  • Dictionaries are central to Python — they power JSON handling, function keyword arguments, and much of the language internals.
Why this matters: Dictionaries are arguably the most important data structure in Python after lists. You will use them daily whether you are parsing API responses, building configuration objects, counting items, or organizing data. The patterns you learned here — especially counting with .get(), grouping with .setdefault(), and comprehensions — will appear in nearly every Python project you work on.
