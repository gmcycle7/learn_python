Python for Automation

This bonus page is your practical handbook for using Python to automate everyday tasks: managing files, processing text, running shell commands, and building command-line tools. Every example is a complete, runnable script you can adapt to your own work immediately.

For MATLAB users: In MATLAB you might use system() to call shell commands or write .m scripts to batch-process data files. Python offers the same capabilities with a much richer standard library — no toolboxes required. If you have ever written a MATLAB script that loops over files in a folder, cleans up data, and saves results, you will feel right at home here.

1. Why Python is Great for Automation

Python has become the go-to language for automation scripts, and for good reason:

  • Readable syntax — Scripts you write today are easy to understand six months from now. No cryptic bash one-liners.
  • Cross-platform — The same Python script runs on Windows, macOS, and Linux without modification (when you use the standard library for file paths and system calls).
  • Batteries included — The standard library ships with modules for file I/O, JSON, CSV, regex, subprocesses, argument parsing, and much more — no pip install needed.
  • Replaces bash scripts — For anything beyond a few lines, Python is more maintainable, more portable, and easier to debug than shell scripts.
  • Easy to extend — When you need more power, thousands of third-party packages are one pip install away.
Standard library modules used on this page: os, os.path, pathlib, shutil, glob, json, csv, re, subprocess, sys, argparse, time, datetime. All of these come with every Python installation.

2. File System Operations

File management is the bread and butter of automation. Python gives you two main approaches: the older os / os.path module and the modern pathlib module (recommended for new code).

2.1 Listing Files in a Directory

# --- Method 1: os.listdir (returns a plain list of names) ---
import os

entries = os.listdir(".")          # current directory
print(entries)
# ['report.csv', 'data', 'script.py', ...]

# --- Method 2: pathlib.Path.iterdir (returns Path objects) ---
from pathlib import Path

for item in Path(".").iterdir():
    print(item.name, "dir" if item.is_dir() else "file")

# --- Method 3: glob (pattern matching) ---
import glob

csv_files = glob.glob("*.csv")     # all CSVs in current dir
print(csv_files)

# pathlib equivalent
csv_files = list(Path(".").glob("*.csv"))
print(csv_files)

2.2 Walking Directory Trees

# --- os.walk: visit every directory recursively ---
import os

for dirpath, dirnames, filenames in os.walk("/home/user/projects"):
    for fname in filenames:
        full_path = os.path.join(dirpath, fname)
        print(full_path)

# --- pathlib.rglob: recursive glob ---
from pathlib import Path

# Find all .py files anywhere under a folder
for py_file in Path("/home/user/projects").rglob("*.py"):
    print(py_file)

2.3 Creating and Deleting Folders

import os
import shutil

# Create a nested folder structure (like mkdir -p)
os.makedirs("output/reports/2025", exist_ok=True)

# pathlib equivalent
from pathlib import Path
Path("output/reports/2025").mkdir(parents=True, exist_ok=True)

# Delete a single empty directory
os.rmdir("output/reports/2025")

# Delete a directory tree (CAREFUL — this is permanent!)
shutil.rmtree("output")
Warning: shutil.rmtree() permanently deletes files — there is no recycle bin, no undo. Always double-check the path, and consider printing what you would delete before actually deleting.

2.4 Copying and Moving Files

import shutil

# Copy a file (preserves content, not necessarily metadata)
shutil.copy("data.csv", "backup/data.csv")

# Copy a file AND its metadata (timestamps, permissions)
shutil.copy2("data.csv", "backup/data.csv")

# Move (or rename) a file
shutil.move("old_name.txt", "new_name.txt")

# Copy an entire directory tree
shutil.copytree("src_folder", "dst_folder")

2.5 Renaming Files

import os
from pathlib import Path

# os approach
os.rename("report_draft.txt", "report_final.txt")

# pathlib approach
Path("report_draft.txt").rename("report_final.txt")

2.6 Checking File Size and Modification Time

import os
import datetime

info = os.stat("data.csv")
print(f"Size: {info.st_size} bytes")
print(f"Modified: {datetime.datetime.fromtimestamp(info.st_mtime)}")

# pathlib approach
from pathlib import Path

p = Path("data.csv")
print(f"Size: {p.stat().st_size} bytes")
print(f"Suffix: {p.suffix}")        # '.csv'
print(f"Stem: {p.stem}")            # 'data'
print(f"Exists: {p.exists()}")

2.7 Complete Example: Organize a Downloads Folder

Runnable script! Save this as organize_downloads.py and point it at any messy folder. It sorts files into sub-folders by extension. 
#!/usr/bin/env python3
"""Organize files in a directory into sub-folders by extension."""

import os
import shutil
from pathlib import Path

def organize_folder(folder_path):
    """Move files into sub-folders named by their extension."""
    folder = Path(folder_path)

    if not folder.is_dir():
        print(f"Error: {folder} is not a directory")
        return

    moved = 0
    for item in folder.iterdir():
        # Skip directories and hidden files
        if item.is_dir() or item.name.startswith("."):
            continue

        # Determine the target sub-folder
        ext = item.suffix.lower()          # e.g. '.pdf'
        if ext == "":
            dest_folder = folder / "no_extension"
        else:
            dest_folder = folder / ext.lstrip(".")   # 'pdf', 'jpg', etc.

        dest_folder.mkdir(exist_ok=True)
        dest_file = dest_folder / item.name

        # Avoid overwriting existing files
        if dest_file.exists():
            print(f"  Skipped (already exists): {item.name}")
            continue

        shutil.move(str(item), str(dest_file))
        moved += 1
        print(f"  Moved: {item.name} -> {dest_folder.name}/")

    print(f"\nDone! Moved {moved} file(s).")

if __name__ == "__main__":
    import sys
    target = sys.argv[1] if len(sys.argv) > 1 else "."
    organize_folder(target)
# Usage:
# python organize_downloads.py ~/Downloads
#   Moved: report.pdf -> pdf/
#   Moved: photo.jpg -> jpg/
#   Moved: notes.txt -> txt/
#   Done! Moved 3 file(s).

3. Working with Text Files

3.1 Reading, Writing, and Appending

# --- Read an entire file ---
with open("notes.txt", "r") as f:
    content = f.read()
print(content)

# --- Read line by line (memory-friendly for large files) ---
with open("notes.txt", "r") as f:
    for line in f:
        print(line.strip())        # strip() removes the trailing newline

# --- Write (overwrites existing content) ---
with open("output.txt", "w") as f:
    f.write("Line 1\n")
    f.write("Line 2\n")

# --- Append (adds to the end) ---
with open("output.txt", "a") as f:
    f.write("Line 3 (appended)\n")
MATLAB equivalent: fopen / fgets / fprintf / fclose. Python's with open(...) automatically closes the file for you — no need to remember fclose.

3.2 Processing CSV Files

import csv

# --- Reading a CSV ---
with open("employees.csv", "r", newline="") as f:
    reader = csv.reader(f)
    header = next(reader)          # first row = column names
    print("Columns:", header)

    for row in reader:
        name, dept, salary = row
        print(f"  {name} works in {dept}, earns ${salary}")

# --- Reading into dictionaries (column names as keys) ---
with open("employees.csv", "r", newline="") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"], row["salary"])

# --- Writing a CSV ---
data = [
    ["name", "score"],
    ["Alice", 95],
    ["Bob", 87],
    ["Carol", 92],
]

with open("scores.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerows(data)         # write all rows at once

3.3 Processing JSON Files

import json

# --- Reading JSON ---
with open("config.json", "r") as f:
    config = json.load(f)          # returns a dict (or list)
print(config["database"]["host"])

# --- Writing JSON ---
data = {
    "name": "AutoBot",
    "version": 2,
    "features": ["file_ops", "scheduling"],
}
with open("output.json", "w") as f:
    json.dump(data, f, indent=2)   # indent for human-readable output

# --- Working with JSON strings ---
json_string = '{"x": 1, "y": 2}'
obj = json.loads(json_string)      # loads = load from string
print(obj["x"])                    # 1

back_to_string = json.dumps(obj)
print(back_to_string)              # '{"x": 1, "y": 2}'
Tip: json.load() reads from a file. json.loads() reads from a string. Same pattern for output: json.dump() writes to a file, json.dumps() returns a string. The trailing s stands for string.

3.4 Processing Log Files

# Count ERROR, WARNING, and INFO lines in a log file
from collections import Counter

level_counts = Counter()

with open("app.log", "r") as f:
    for line in f:
        for level in ("ERROR", "WARNING", "INFO"):
            if level in line:
                level_counts[level] += 1
                break

for level, count in level_counts.most_common():
    print(f"  {level}: {count}")

3.5 Searching for Patterns with Regex

import re

text = "Contact us at support@example.com or sales@example.com"

# Find all email addresses
emails = re.findall(r"[\w.+-]+@[\w-]+\.[\w.]+", text)
print(emails)
# ['support@example.com', 'sales@example.com']

# Search for a pattern (returns first match or None)
match = re.search(r"\d{3}-\d{4}", "Call 555-1234 today")
if match:
    print(f"Found: {match.group()}")   # Found: 555-1234

# Replace patterns
cleaned = re.sub(r"\s+", " ", "too   many    spaces")
print(cleaned)   # 'too many spaces'

# Split on a pattern
parts = re.split(r"[;,]\s*", "one, two; three,four")
print(parts)     # ['one', 'two', 'three', 'four']

3.6 Complete Example: Extract Data from Multiple Log Files

Runnable script! This script scans a directory of log files, extracts all ERROR lines, and writes a summary report. 
#!/usr/bin/env python3
"""Scan log files for ERROR lines and produce a summary report."""

import re
from pathlib import Path
from collections import defaultdict

def scan_logs(log_dir, output_file="error_report.txt"):
    """Find all ERROR lines in .log files and write a summary."""
    log_dir = Path(log_dir)
    errors_by_file = defaultdict(list)

    # Pattern: timestamp followed by ERROR and a message
    pattern = re.compile(r"(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}).*ERROR\s+(.*)")

    for log_file in sorted(log_dir.glob("*.log")):
        with open(log_file, "r") as f:
            for line_num, line in enumerate(f, 1):
                match = pattern.search(line)
                if match:
                    timestamp, message = match.groups()
                    errors_by_file[log_file.name].append(
                        (line_num, timestamp, message.strip())
                    )

    # Write the report
    total_errors = sum(len(errs) for errs in errors_by_file.values())

    with open(output_file, "w") as out:
        out.write(f"Error Report\n{'=' * 60}\n")
        out.write(f"Scanned: {log_dir}\n")
        out.write(f"Total errors found: {total_errors}\n\n")

        for filename, errs in sorted(errors_by_file.items()):
            out.write(f"--- {filename} ({len(errs)} errors) ---\n")
            for line_num, timestamp, message in errs:
                out.write(f"  Line {line_num} [{timestamp}]: {message}\n")
            out.write("\n")

    print(f"Report written to {output_file} ({total_errors} errors found)")

if __name__ == "__main__":
    import sys
    folder = sys.argv[1] if len(sys.argv) > 1 else "."
    scan_logs(folder)

4. Running Shell Commands

Python's subprocess module lets you run any command-line program from within your script. It replaces the older os.system() function (which you should avoid).

4.1 subprocess.run Basics

import subprocess

# Run a simple command
result = subprocess.run(["echo", "Hello from Python!"])
print(f"Return code: {result.returncode}")   # 0 means success
Why a list? Passing the command as a list (["echo", "Hello"]) is safer than a single string because Python handles quoting and escaping for you. It also avoids shell injection vulnerabilities.

4.2 Capturing Output

import subprocess

# Capture stdout as a string
result = subprocess.run(
    ["python", "--version"],
    capture_output=True,           # capture stdout and stderr
    text=True                      # decode bytes to str
)
print(f"stdout: {result.stdout.strip()}")
print(f"stderr: {result.stderr.strip()}")
print(f"return code: {result.returncode}")

4.3 Checking Return Codes

import subprocess

# Method 1: check manually
result = subprocess.run(["ls", "nonexistent_file"], capture_output=True, text=True)
if result.returncode != 0:
    print(f"Command failed: {result.stderr.strip()}")

# Method 2: raise an exception on failure
try:
    subprocess.run(
        ["ls", "nonexistent_file"],
        capture_output=True, text=True,
        check=True                 # raises CalledProcessError on non-zero exit
    )
except subprocess.CalledProcessError as e:
    print(f"Command failed with code {e.returncode}")
    print(f"stderr: {e.stderr.strip()}")

4.4 Piping Commands

import subprocess

# Equivalent of: cat /etc/hosts | grep "localhost"
cat_proc = subprocess.run(
    ["cat", "/etc/hosts"],
    capture_output=True, text=True
)

grep_proc = subprocess.run(
    ["grep", "localhost"],
    input=cat_proc.stdout,         # pipe the output of cat into grep
    capture_output=True, text=True
)

print(grep_proc.stdout)
Why not shell=True? You might see subprocess.run("cat file | grep pattern", shell=True) in older code. While it works, using shell=True opens the door to shell injection attacks and makes your script platform-dependent. Prefer the list form and pipe data through Python instead.

4.5 Complete Example: System Health Check Script

Runnable script! A quick system info reporter that works on Linux and macOS. Adapt the commands for Windows as needed. 
#!/usr/bin/env python3
"""Quick system health check script."""

import subprocess
import platform
import shutil

def run_cmd(cmd):
    """Run a command and return its stdout, or an error message."""
    try:
        result = subprocess.run(
            cmd, capture_output=True, text=True, timeout=10
        )
        return result.stdout.strip()
    except FileNotFoundError:
        return "(command not found)"
    except subprocess.TimeoutExpired:
        return "(timed out)"

def health_check():
    print("=" * 50)
    print("SYSTEM HEALTH CHECK")
    print("=" * 50)

    # 1. Basic system info (cross-platform)
    print(f"\nPlatform:   {platform.system()} {platform.release()}")
    print(f"Python:     {platform.python_version()}")

    # 2. Disk usage (cross-platform via shutil)
    total, used, free = shutil.disk_usage("/")
    gb = 1024 ** 3
    print(f"\nDisk usage: {used / gb:.1f} GB used / {total / gb:.1f} GB total "
          f"({free / gb:.1f} GB free)")

    # 3. Uptime (Linux/macOS)
    uptime = run_cmd(["uptime", "-p"])
    if uptime != "(command not found)":
        print(f"Uptime:     {uptime}")

    # 4. Memory info (Linux)
    mem = run_cmd(["free", "-h"])
    if mem != "(command not found)":
        print(f"\nMemory:\n{mem}")

    # 5. Network connectivity
    ping = run_cmd(["ping", "-c", "1", "-W", "2", "8.8.8.8"])
    if "1 received" in ping or "1 packets received" in ping:
        print("\nNetwork:    Connected (ping to 8.8.8.8 OK)")
    else:
        print("\nNetwork:    Possibly offline")

    print("\n" + "=" * 50)

if __name__ == "__main__":
    health_check()

5. Command-Line Scripts

A well-built CLI script accepts arguments from the command line so you do not have to edit the code every time you run it.

5.1 sys.argv for Basic Arguments

import sys

# sys.argv is a list of strings:
#   sys.argv[0] = the script name
#   sys.argv[1] = first argument, etc.

print(f"Script name: {sys.argv[0]}")
print(f"Arguments:   {sys.argv[1:]}")

# Example usage: python myscript.py hello world
# Output:
#   Script name: myscript.py
#   Arguments:   ['hello', 'world']
When to use sys.argv: Quick one-off scripts where you just need one or two positional arguments. For anything more complex, use argparse (next section).

5.2 argparse for Proper CLI Tools

#!/usr/bin/env python3
"""A simple greeter with argparse."""

import argparse

parser = argparse.ArgumentParser(description="Greet someone.")
parser.add_argument("name", help="Name of the person to greet")
parser.add_argument("-n", "--count", type=int, default=1,
                    help="Number of times to greet (default: 1)")
parser.add_argument("-u", "--upper", action="store_true",
                    help="Print greeting in uppercase")

args = parser.parse_args()

greeting = f"Hello, {args.name}!"
if args.upper:
    greeting = greeting.upper()

for _ in range(args.count):
    print(greeting)
# Usage examples:
# python greet.py Alice
#   Hello, Alice!
#
# python greet.py Bob -n 3 --upper
#   HELLO, BOB!
#   HELLO, BOB!
#   HELLO, BOB!
#
# python greet.py --help
#   usage: greet.py [-h] [-n COUNT] [-u] name
#   Greet someone.
#   positional arguments:
#     name                  Name of the person to greet
#   optional arguments:
#     -n COUNT, --count COUNT  Number of times to greet (default: 1)
#     -u, --upper           Print greeting in uppercase

5.4 Complete Example: CLI File Search Tool

Runnable script! A mini grep-like tool that searches files for a pattern and reports matches. 
#!/usr/bin/env python3
"""Search files for a text pattern (like a simple grep)."""

import argparse
import re
from pathlib import Path

def search_files(directory, pattern, extension, ignore_case):
    """Search for a regex pattern in files and yield matches."""
    flags = re.IGNORECASE if ignore_case else 0
    regex = re.compile(pattern, flags)

    search_path = Path(directory)
    glob_pattern = f"*.{extension}" if extension else "*"

    for filepath in sorted(search_path.rglob(glob_pattern)):
        if not filepath.is_file():
            continue

        try:
            with open(filepath, "r", errors="ignore") as f:
                for line_num, line in enumerate(f, 1):
                    if regex.search(line):
                        yield filepath, line_num, line.rstrip()
        except (PermissionError, OSError):
            continue

def main():
    parser = argparse.ArgumentParser(
        description="Search files for a text pattern."
    )
    parser.add_argument("pattern", help="Regex pattern to search for")
    parser.add_argument("directory", nargs="?", default=".",
                        help="Directory to search (default: current)")
    parser.add_argument("-e", "--ext", default=None,
                        help="Only search files with this extension (e.g. py)")
    parser.add_argument("-i", "--ignore-case", action="store_true",
                        help="Case-insensitive search")
    args = parser.parse_args()

    count = 0
    for filepath, line_num, line in search_files(
        args.directory, args.pattern, args.ext, args.ignore_case
    ):
        # Show relative path for cleaner output
        rel = filepath.relative_to(args.directory)
        print(f"{rel}:{line_num}: {line}")
        count += 1

    print(f"\n--- {count} match(es) found ---")

if __name__ == "__main__":
    main()
# Usage:
# python filesearch.py "TODO" ./src --ext py
# python filesearch.py "error" /var/log --ext log -i

6.2 Simple Timing with the time Module

import time

start = time.time()

# ... do some work ...
total = sum(range(10_000_000))

6.3 Progress Reporting

import sys
from pathlib import Path

files = list(Path("data").glob("*.csv"))
total = len(files)

for i, f in enumerate(files, 1):
    # Process the file ...
    process(f)

    # Print progress on the same line
    pct = i / total * 100
    sys.stdout.write(f"\rProcessing: {i}/{total} ({pct:.0f}%)")
    sys.stdout.flush()

print("\nDone!")
Tip: The \r (carriage return) moves the cursor back to the start of the line, so the next write() overwrites it. This gives a clean, updating progress counter without flooding the terminal with lines.

6.4 Complete Example: Batch File Info Reporter

Runnable script! Scans a directory for image files and reports their sizes and modification dates. 
#!/usr/bin/env python3
"""Report file info (size, date) for images in a directory."""

import os
import sys
import datetime
from pathlib import Path

IMAGE_EXTENSIONS = {".jpg", ".jpeg", ".png", ".gif", ".bmp", ".tiff", ".webp"}

def format_size(size_bytes):
    """Convert bytes to a human-readable string."""
    for unit in ("B", "KB", "MB", "GB"):
        if size_bytes < 1024:
            return f"{size_bytes:.1f} {unit}"
        size_bytes /= 1024
    return f"{size_bytes:.1f} TB"

def report_images(directory):
    """Print a report of all image files in directory."""
    dir_path = Path(directory)

    if not dir_path.is_dir():
        print(f"Error: {directory} is not a valid directory")
        sys.exit(1)

    # Gather image files
    images = [
        f for f in dir_path.iterdir()
        if f.is_file() and f.suffix.lower() in IMAGE_EXTENSIONS
    ]

    if not images:
        print("No image files found.")
        return

    images.sort(key=lambda f: f.name)
    total_size = 0

    print(f"{'File':<35} {'Size':>10}  {'Modified'}")
    print("-" * 70)

    for img in images:
        stat = img.stat()
        size = stat.st_size
        total_size += size
        modified = datetime.datetime.fromtimestamp(stat.st_mtime)

        print(f"{img.name:<35} {format_size(size):>10}  {modified:%Y-%m-%d %H:%M}")

    print("-" * 70)
    print(f"{'Total: ' + str(len(images)) + ' files':<35} {format_size(total_size):>10}")

if __name__ == "__main__":
    target = sys.argv[1] if len(sys.argv) > 1 else "."
    report_images(target)
# Usage:
# python image_report.py ~/Pictures
#
# File                                    Size  Modified
# ----------------------------------------------------------------------
# beach_sunset.jpg                      2.3 MB  2025-06-15 14:30
# diagram.png                         145.2 KB  2025-07-01 09:12
# screenshot_2025.png                  890.4 KB  2025-08-20 11:45
# ----------------------------------------------------------------------
# Total: 3 files                        3.3 MB

7. Working with Paths

Correct path handling is critical for cross-platform scripts. Python offers two approaches: the traditional os.path module and the modern pathlib module.

7.1 os.path vs. pathlib Comparison

Operation os.path pathlib
Join paths os.path.join("a", "b", "c.txt") Path("a") / "b" / "c.txt"
Get filename os.path.basename(p) Path(p).name
Get directory os.path.dirname(p) Path(p).parent
Get extension os.path.splitext(p)[1] Path(p).suffix
Get stem (no ext) os.path.splitext(os.path.basename(p))[0] Path(p).stem
File exists? os.path.exists(p) Path(p).exists()
Is a file? os.path.isfile(p) Path(p).is_file()
Is a directory? os.path.isdir(p) Path(p).is_dir()
Absolute path os.path.abspath(p) Path(p).resolve()
Home directory os.path.expanduser("~") Path.home()
Current directory os.getcwd() Path.cwd()
Which should you use? For new code, prefer pathlib. It is more readable (the / operator is intuitive), returns rich objects instead of raw strings, and handles platform differences automatically. You will encounter os.path in older code, so it is good to know both.

7.2 Cross-Platform Path Handling

from pathlib import Path

# WRONG — hardcoded separator (breaks on Windows or Linux)
path = "data/reports/2025/summary.csv"     # works on Linux/macOS
path = "data\\reports\\2025\\summary.csv"   # works on Windows only

# RIGHT — let Python choose the separator

# On Linux:   data/reports/2025/summary.csv
# On Windows: data\reports\2025\summary.csv

7.4 Common Path Operations

from pathlib import Path

p = Path("/home/user/projects/data/results.csv")

print(p.name)       # 'results.csv'
print(p.stem)       # 'results'
print(p.suffix)     # '.csv'
print(p.parent)     # /home/user/projects/data
print(p.parents[0]) # /home/user/projects/data   (same as .parent)
print(p.parents[1]) # /home/user/projects
print(p.parents[2]) # /home/user

# Change the extension
new_p = p.with_suffix(".json")
print(new_p)         # /home/user/projects/data/results.json

# Change just the filename
new_p = p.with_name("output.csv")
print(new_p)         # /home/user/projects/data/output.csv

# Check if a path matches a pattern
print(p.match("*.csv"))              # True
print(p.match("data/*.csv"))         # True

# Get all parts as a tuple
print(p.parts)
# ('/', 'home', 'user', 'projects', 'data', 'results.csv')
MATLAB comparison: MATLAB uses fullfile() to build paths and fileparts() to decompose them. Python's pathlib gives you the same functionality but with a much richer, object-oriented API. The / operator replaces fullfile(), and properties like .stem, .suffix, and .parent replace the multiple return values of fileparts().

8.1 Watch a Folder for New Files

#!/usr/bin/env python3
"""Poll a directory and react when new files appear."""

import time
from pathlib import Path

def watch_folder(folder, interval=2):
    """Watch a folder and print a message when new files arrive."""
    folder = Path(folder)
    known_files = set(folder.iterdir())

    print(f"Watching {folder} for new files (Ctrl+C to stop)...")

    try:
        while True:
            time.sleep(interval)
            current_files = set(folder.iterdir())
            new_files = current_files - known_files

            for f in new_files:
                if f.is_file():
                    print(f"  New file detected: {f.name} "
                          f"({f.stat().st_size} bytes)")

            known_files = current_files
    except KeyboardInterrupt:
        print("\nStopped watching.")

if __name__ == "__main__":
    import sys
    target = sys.argv[1] if len(sys.argv) > 1 else "."
    watch_folder(target)

8.2 Backup Script

#!/usr/bin/env python3
"""Create a timestamped backup of a directory."""

import shutil
import datetime
from pathlib import Path

def backup_directory(source, backup_root="backups"):
    """Copy source directory into backup_root with a timestamp."""
    source = Path(source)
    backup_root = Path(backup_root)

    if not source.is_dir():
        print(f"Error: {source} is not a directory")
        return

    # Create a timestamp-based folder name
    timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
    dest = backup_root / f"{source.name}_{timestamp}"

    print(f"Backing up {source} -> {dest}")
    shutil.copytree(source, dest)

    # Count what was copied
    file_count = sum(1 for _ in dest.rglob("*") if _.is_file())
    print(f"Done! {file_count} file(s) backed up.")

if __name__ == "__main__":
    import sys
    if len(sys.argv) < 2:
        print("Usage: python backup.py  [backup_root]")
        sys.exit(1)
    src = sys.argv[1]
    dst = sys.argv[2] if len(sys.argv) > 2 else "backups"
    backup_directory(src, dst)

8.3 Find and Replace Across Files

#!/usr/bin/env python3
"""Find and replace a string across all files matching a pattern."""

from pathlib import Path

def find_and_replace(directory, glob_pattern, old_text, new_text, dry_run=True):
    """Replace old_text with new_text in files matching glob_pattern."""
    directory = Path(directory)
    changed_files = 0

    for filepath in sorted(directory.rglob(glob_pattern)):
        if not filepath.is_file():
            continue

        try:
            content = filepath.read_text()
        except (UnicodeDecodeError, PermissionError):
            continue

        if old_text not in content:
            continue

        count = content.count(old_text)

        if dry_run:
            print(f"  [DRY RUN] {filepath}: {count} occurrence(s)")
        else:
            new_content = content.replace(old_text, new_text)
            filepath.write_text(new_content)
            print(f"  Updated {filepath}: {count} replacement(s)")

        changed_files += 1

    mode = "DRY RUN" if dry_run else "DONE"
    print(f"\n[{mode}] {changed_files} file(s) affected.")

if __name__ == "__main__":
    import sys
    if len(sys.argv) < 4:
        print("Usage: python replace.py    [--apply]")
        print("  Without --apply, runs in dry-run mode (shows what would change)")
        sys.exit(1)

    apply = "--apply" in sys.argv
    find_and_replace(
        directory=sys.argv[1],
        glob_pattern="*.py",
        old_text=sys.argv[2],
        new_text=sys.argv[3],
        dry_run=not apply
    )
Safety first: Notice the dry_run=True default. Always preview what a mass-edit script would change before making actual modifications. This pattern — dry-run by default, --apply flag to execute — prevents accidents.

8.4 Generate Reports from Data

#!/usr/bin/env python3
"""Read CSV data files and generate a summary report."""

import csv
from pathlib import Path
from collections import defaultdict

def generate_report(data_dir, output_file="summary_report.txt"):
    """Read all CSV files and produce a summary."""
    data_dir = Path(data_dir)
    all_stats = []

    for csv_file in sorted(data_dir.glob("*.csv")):
        with open(csv_file, "r", newline="") as f:
            reader = csv.DictReader(f)
            rows = list(reader)

        if not rows:
            continue

        # Gather basic stats
        stats = {
            "file": csv_file.name,
            "rows": len(rows),
            "columns": len(rows[0]),
            "column_names": list(rows[0].keys()),
        }
        all_stats.append(stats)

    # Write the report
    with open(output_file, "w") as out:
        out.write("DATA SUMMARY REPORT\n")
        out.write("=" * 50 + "\n\n")

        for s in all_stats:
            out.write(f"File: {s['file']}\n")
            out.write(f"  Rows: {s['rows']}\n")
            out.write(f"  Columns ({s['columns']}): {', '.join(s['column_names'])}\n\n")

        out.write(f"Total files processed: {len(all_stats)}\n")
        out.write(f"Total rows across all files: "
                  f"{sum(s['rows'] for s in all_stats)}\n")

    print(f"Report written to {output_file}")

if __name__ == "__main__":
    import sys
    folder = sys.argv[1] if len(sys.argv) > 1 else "."
    generate_report(folder)

8.5 Clean Up Temporary Files

#!/usr/bin/env python3
"""Delete temporary and cache files from a project directory."""

import os
from pathlib import Path

# Patterns to clean up
JUNK_PATTERNS = [
    "**/*.pyc",              # compiled Python bytecode
    "**/__pycache__",        # bytecode cache directories
    "**/*.tmp",              # temp files
    "**/*.log",              # log files
    "**/.DS_Store",          # macOS folder metadata
    "**/Thumbs.db",          # Windows thumbnail cache
]

def cleanup(directory, dry_run=True):
    """Remove temporary files and directories matching JUNK_PATTERNS."""
    directory = Path(directory)
    removed_files = 0
    removed_dirs = 0
    freed_bytes = 0

    for pattern in JUNK_PATTERNS:
        for item in directory.glob(pattern):
            if item.is_file():
                size = item.stat().st_size
                if dry_run:
                    print(f"  [DEL FILE] {item} ({size:,} bytes)")
                else:
                    item.unlink()
                    print(f"  Deleted: {item}")
                removed_files += 1
                freed_bytes += size
            elif item.is_dir():
                if dry_run:
                    print(f"  [DEL DIR]  {item}/")
                else:
                    import shutil
                    shutil.rmtree(item)
                    print(f"  Removed: {item}/")
                removed_dirs += 1

    mode = "DRY RUN" if dry_run else "CLEANED"
    print(f"\n[{mode}] {removed_files} file(s), {removed_dirs} dir(s), "
          f"~{freed_bytes / 1024:.1f} KB freed")

if __name__ == "__main__":
    import sys
    target = sys.argv[1] if len(sys.argv) > 1 else "."
    apply = "--apply" in sys.argv
    cleanup(target, dry_run=not apply)
# Usage:
# python cleanup.py ./my_project              (preview what would be deleted)
# python cleanup.py ./my_project --apply      (actually delete)

Summary

  • Python's standard library provides everything you need for file management, text processing, shell commands, and CLI tools — no third-party packages required.
  • Use pathlib.Path for all path operations in new code. It is more readable and cross-platform than os.path.
  • Always use with open(...) for file I/O to ensure files are properly closed.
  • Use subprocess.run() with a list argument to run shell commands safely. Avoid shell=True.
  • Use argparse to turn scripts into proper CLI tools with help text and validation.
  • Build safety into your scripts: use dry-run modes, print before deleting, and catch exceptions gracefully.
  • The recipes on this page (folder watcher, backup, find-and-replace, cleanup) are starting templates — adapt them to your specific workflows.
You now have a practical automation toolkit! These patterns cover the vast majority of day-to-day scripting needs. When you are ready for more advanced automation (web scraping, email, databases, APIs), Python's ecosystem has mature packages for all of them. But the standard library techniques on this page will take you surprisingly far.
