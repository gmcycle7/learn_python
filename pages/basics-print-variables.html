Print, Comments, Variables & Numbers

This is the very first "real Python" page. By the end you will be able to display output on the screen, leave notes in your code, store values in variables, and do arithmetic. These four skills form the foundation of every Python program you will ever write.

Why this matters: Every programming language needs a way to show results, remember values, and crunch numbers. Master these basics and everything that follows—strings, lists, functions—will feel like natural extensions.

1. The print() Function

print() is Python's built-in function for displaying output to the screen (the "console" or "terminal"). It is the single most useful debugging and learning tool you have. Whenever you are unsure what a value is, just print() it.

How it works: You call print() by writing the word print followed by parentheses. Inside those parentheses you place whatever you want to display. Python converts it to text and writes it to the console, followed by a newline character (so the next output appears on a new line).

1.1 Basic Printing

The simplest use: pass a string (text inside quotes) to print().

# Example 1 — Print a simple string
print("Hello, world!")
# Output: Hello, world!

# Example 2 — Single quotes work too
print('Python is fun!')
# Output: Python is fun!

# Example 3 — Print a number directly
print(42)
# Output: 42

1.2 Printing Multiple Values

You can pass several values to print() separated by commas. Python will print them all on one line, separated by a space by default.

# Example 4 — Multiple values
print("The answer is", 42)
# Output: The answer is 42

# Example 5 — Mix types freely
print("Pi is approximately", 3.14159, "and e is approximately", 2.71828)
# Output: Pi is approximately 3.14159 and e is approximately 2.71828

1.3 The sep Parameter

The sep (separator) parameter controls what goes between each value. The default is a single space (" ").

# Example 6 — Custom separator
print("2026", "03", "31", sep="-")
# Output: 2026-03-31

# Example 7 — No separator
print("P", "y", "t", "h", "o", "n", sep="")
# Output: Python

# Example 8 — Tab separator
print("Name", "Age", "City", sep="\t")
# Output: Name    Age    City

1.4 The end Parameter

By default print() finishes with a newline character (\n), so the next print() appears on the next line. The end parameter lets you change that.

# Example 9 — Printing on the same line
print("Loading", end="")
print("...", end="")
print(" Done!")
# Output: Loading... Done!

# Example 10 — End with a custom string
print("Item 1", end=" | ")
print("Item 2", end=" | ")
print("Item 3")
# Output: Item 1 | Item 2 | Item 3

1.5 Printing Variables

You will almost always print the contents of variables rather than hard-coded text. Simply put the variable name inside print()—no quotes around it.

# Example 11 — Printing a variable
name = "Alice"
age = 30
print(name)          # Output: Alice
print(age)           # Output: 30
print(name, "is", age, "years old.")
# Output: Alice is 30 years old.
Common Mistake — Forgetting parentheses:
In Python 2, print was a statement and you could write print "hello". In Python 3, print() is a function and the parentheses are required. If you see old tutorials or Stack Overflow answers using print "hello", know that this will cause a SyntaxError in Python 3. 
# WRONG in Python 3:
print "hello"   # SyntaxError: Missing parentheses in call to 'print'

# CORRECT:
print("hello")
MATLAB Note: print() is similar to MATLAB's disp() for simple output and fprintf() for formatted output. The key difference: Python's print() automatically adds a newline at the end (like disp()), while MATLAB's fprintf() does not unless you add \n yourself. In Python, if you want formatted output, look into f-strings (covered in the Strings page), which replace fprintf-style formatting. 
% MATLAB
disp('Hello, world!')
fprintf('The answer is %d\n', 42)

# Python equivalent
print('Hello, world!')
print(f'The answer is {42}')

2. Comments

Comments are notes you leave in your code for yourself (or for other humans reading it). Python completely ignores comments when running your program—they exist only to make the code easier to understand.

Why this matters: Code is read far more often than it is written. A well-placed comment can save you (or a colleague) hours of confusion when you revisit code weeks or months later.

2.1 Single-Line Comments with #

Everything after a # on a line is a comment. Python ignores it.

# This is a comment — Python skips this entire line
print("Hello")  # This comment is after code — Python runs the print, ignores the rest

# You can use comments to temporarily disable code:
# print("This line will NOT run")

2.2 Multi-Line Comments with Triple Quotes

Python does not have a dedicated multi-line comment syntax (like /* ... */ in C). However, you can use triple-quoted strings ("""...""" or '''...''') as a stand-in. If a triple-quoted string is not assigned to a variable, Python evaluates it and then discards it—effectively making it a comment.

"""
This is a multi-line comment (technically a string literal).
Python evaluates it but discards the result,
so it behaves like a comment.
"""

'''
You can also use single triple-quotes.
This is useful for longer explanations.
'''
Docstrings: When a triple-quoted string appears as the very first statement inside a function, class, or module, it becomes a docstring—a special kind of documentation that Python stores and makes accessible via the help() function. You will learn about docstrings in detail on the Functions page. 
def greet(name):
    """Return a greeting string for the given name."""
    return f"Hello, {name}!"

help(greet)
# Shows: Return a greeting string for the given name.

2.3 When to Use Comments

  • Explain "why," not "what." Good: # Convert to meters because the sensor returns millimeters. Bad: # Divide x by 1000 (the code already says that).
  • Mark TODOs: # TODO: handle the case where the file is empty.
  • Disable code temporarily during debugging (called "commenting out").
  • Section headers in long scripts: # --- Data Loading ---.
Common Mistake — Over-commenting: Do not comment every single line. If your variable is called total_price, you do not need a comment saying # the total price. Use clear variable names and save comments for non-obvious logic.
MATLAB Note: MATLAB uses % for single-line comments and %{ ... %} for block comments. In Python, use # for single lines and """...""" for multi-line blocks. The mapping is straightforward: 
% MATLAB single-line comment
%{
  MATLAB block comment
%}

# Python single-line comment
"""
Python multi-line comment
(or docstring if inside a function/class)
"""

3. Variables

3.1 What Is a Variable?

A variable is a name that refers to a value stored in your computer's memory. Think of it as a labeled box: the label is the variable name, and the contents of the box is the value. You can put something in the box, look at what is inside, replace it with something else, or use it in calculations.

Plain English: A variable is just a way to give a name to a piece of data so you can use it later. Instead of writing 3.14159 everywhere, you write pi = 3.14159 once and then use the name pi wherever you need it.

3.2 Assignment with =

You create (or update) a variable using the = operator. The name goes on the left; the value goes on the right.

# Example 12 — Creating variables
message = "Hello, Python!"
count = 5
temperature = 98.6

print(message)        # Output: Hello, Python!
print(count)          # Output: 5
print(temperature)    # Output: 98.6

You can reassign a variable at any time—the old value is simply replaced.

# Example 13 — Reassignment
x = 10
print(x)   # Output: 10

x = 20
print(x)   # Output: 20

x = x + 5  # Use the current value of x (20) to compute a new value (25)
print(x)   # Output: 25

3.3 Naming Rules and Conventions

Python has strict rules (break them and you get an error) and community conventions (break them and your code is harder to read).

Rule / ConventionExampleExplanation
Must start with a letter or underscorename, _privateCannot start with a digit

Case-sensitiveName ≠ name ≠ NAMEThese are three different variables
Convention: use snake_casetotal_price, user_nameLowercase words separated by underscores
Convention: UPPER_CASE for constantsMAX_RETRIES = 5Signals "do not change this value"

# Good variable names
student_name = "Bob"
max_score = 100
is_active = True
_internal_counter = 0

# Legal but not recommended (not snake_case)
studentName = "Bob"   # This is camelCase — common in JavaScript, not Python
StudentName = "Bob"   # This is PascalCase — reserved for class names in Python

3.4 Dynamic Typing

In Python, you do not declare a variable's type. Python figures it out automatically based on the value you assign. This is called dynamic typing.

# Example 14 — Dynamic typing
x = 10          # x is an integer
print(type(x))  # Output: <class 'int'>

x = "hello"     # Now x is a string — no error!
print(type(x))  # Output: <class 'str'>

x = 3.14        # Now x is a float
print(type(x))  # Output: <class 'float'>
The type() function: You can check what type a variable holds at any point by calling type(variable_name). This is extremely useful for debugging. You will use it often as you learn.

3.5 Multiple Assignment

Python lets you assign multiple variables in a single line. This is a convenient shorthand.

# Example 15 — Multiple assignment on one line
x, y, z = 1, 2, 3
print(x, y, z)   # Output: 1 2 3

# Assign the same value to multiple variables
a = b = c = 0
print(a, b, c)   # Output: 0 0 0

# Swap two variables without a temporary variable
x, y = 10, 20
print(x, y)       # Output: 10 20
x, y = y, x       # Swap!
print(x, y)       # Output: 20 10
Why this matters: The variable swap trick (x, y = y, x) is something Python makes beautifully simple. In most other languages (including MATLAB), you need a temporary variable to swap two values. In Python it is one line.
Common Mistake — Using reserved words: Python has about 35 keywords (like if, for, while, True, False, None, class, return) that you cannot use as variable names. You will get a SyntaxError. 
# WRONG — 'class' is a reserved keyword
class = "Biology 101"   # SyntaxError: invalid syntax

# FIX — use a slightly different name
course_class = "Biology 101"
Additionally, avoid shadowing built-in names like print, list, type, sum, max, etc. It will not cause an immediate error, but it will break those functions for the rest of your code. 
# BAD — shadows the built-in list() function
list = [1, 2, 3]
# Now list() no longer works as expected!

# FIX — use a descriptive name
my_list = [1, 2, 3]
Common Mistake — Starting with a number: 
# WRONG
2nd_place = "Silver"   # SyntaxError: invalid syntax

# FIX
second_place = "Silver"
place_2 = "Silver"
MATLAB Note: Variable creation in MATLAB and Python is very similar—both use = and neither requires a type declaration. The main differences:
  • MATLAB convention is camelCase; Python convention is snake_case.
  • MATLAB uses ; at the end of a line to suppress output; Python does not print assignment results automatically (there is nothing to suppress).
  • Python variables are case-sensitive. MATLAB variables are also case-sensitive, so this is the same.
  • MATLAB's who/whos commands have no direct equivalent in Python, but you can use dir() or type() to inspect variables.
% MATLAB
studentName = 'Alice';   % camelCase, semicolon to suppress output
x = 10;

# Python
student_name = "Alice"   # snake_case, no semicolons needed
x = 10

4. Numbers: Integers and Floats

Python has two main numeric types you will use daily:

  • int (integer) — whole numbers with no decimal point: 0, 42, -7, 1000000
  • float (floating-point) — numbers with a decimal point: 3.14, -0.001, 2.0
# Example 16 — int vs float
a = 10       # int
b = 10.0     # float (the .0 makes it a float)
c = -3       # int
d = 0.5      # float

print(type(a))  # <class 'int'>
print(type(b))  # <class 'float'>
Key difference: 10 is an int and 10.0 is a float. They are equal in value (10 == 10.0 is True), but they are different types. This rarely causes problems, but it is good to be aware of.

4.1 Arithmetic Operators

Python supports all the arithmetic you would expect, plus a few extras.

OperatorNameExampleResult
+Addition10 + 313
-Subtraction10 - 37
*Multiplication10 * 330
/Division (true)10 / 33.3333...
//Floor division10 // 33
%Modulo (remainder)10 % 31
**Exponentiation10 ** 31000
 
# Example 17 — All arithmetic operators
a = 17
b = 5

print("a + b  =", a + b)     # 22
print("a - b  =", a - b)     # 12
print("a * b  =", a * b)     # 85
print("a / b  =", a / b)     # 3.4
print("a // b =", a // b)    # 3    (floor division — rounds down)
print("a % b  =", a % b)     # 2    (remainder of 17 / 5)
print("a ** b =", a ** b)    # 1419857  (17 to the power of 5)

4.2 Integer Division vs Float Division

This is one of the most important things to understand:

  • / (single slash) always returns a float, even if the result is a whole number.
  • // (double slash) returns the result rounded down to the nearest integer (floor division).
# Example 18 — Division comparison
print(10 / 2)     # 5.0   (float, even though 10 divides evenly by 2)
print(10 // 2)    # 5     (int)

print(7 / 2)      # 3.5   (float)
print(7 // 2)     # 3     (int — rounds DOWN, not to nearest)

print(-7 / 2)     # -3.5  (float)
print(-7 // 2)    # -4    (rounds DOWN toward negative infinity, not toward zero!)
Common Mistake — Division surprise: In Python 3, / always returns a float. This surprises people coming from C, Java, or Python 2 where dividing two integers gives an integer. 
print(10 / 5)    # 2.0, NOT 2
print(type(10 / 5))  # <class 'float'>
If you want an integer result, use //. Also watch out: // rounds toward negative infinity, not toward zero. So -7 // 2 is -4, not -3.

4.3 Order of Operations (Precedence)

Python follows the standard mathematical order of operations, often remembered as PEMDAS:

  1. Parentheses ()
  2. Exponentiation **
  3. Multiplication *, Division /, floor division //, modulo % (left to right)
  4. Addition +, Subtraction - (left to right)
# Example 19 — Order of operations
print(2 + 3 * 4)        # 14  (multiplication first: 3*4=12, then 2+12=14)
print((2 + 3) * 4)      # 20  (parentheses first: 2+3=5, then 5*4=20)
print(2 ** 3 ** 2)       # 512 (right-to-left for **: 3**2=9, then 2**9=512)
print(10 - 2 * 3 + 1)   # 5   (2*3=6, then 10-6+1=5)
Tip: When in doubt, use parentheses. They make your intent clear and prevent bugs. (a + b) * c is instantly readable; a + b * c requires you to remember precedence rules.

4.4 Common Built-in Math Functions

Python provides several math functions that do not require any import.

# Example 20 — Built-in math functions
print(abs(-42))          # 42    (absolute value)
print(abs(3.7))          # 3.7

print(round(3.14159))    # 3     (round to nearest integer)
print(round(3.14159, 2)) # 3.14  (round to 2 decimal places)
print(round(2.5))        # 2     (banker's rounding — rounds to nearest EVEN number)
print(round(3.5))        # 4

print(min(4, 7, 2, 9))   # 2     (smallest value)
print(max(4, 7, 2, 9))   # 9     (largest value)

print(pow(2, 10))        # 1024  (same as 2 ** 10)
Common Mistake — Banker's rounding: Python's round() uses "round half to even" (banker's rounding). This means round(0.5) is 0, round(1.5) is 2, round(2.5) is 2, round(3.5) is 4. This is statistically unbiased but surprises most beginners. If you need traditional "round half up" behavior, you will need a different approach (e.g., math.floor(x + 0.5)).

4.5 The math Module

For more advanced math, Python provides the math module. You must import it before use.

# Example 21 — The math module
import math

# Square root
print(math.sqrt(144))     # 12.0
print(math.sqrt(2))       # 1.4142135623730951

# Constants
print(math.pi)            # 3.141592653589793
print(math.e)             # 2.718281828459045

# Ceiling and floor
print(math.ceil(4.2))     # 5   (round UP to nearest integer)
print(math.floor(4.8))    # 4   (round DOWN to nearest integer)
print(math.ceil(-4.2))    # -4  (toward positive infinity)
print(math.floor(-4.8))   # -5  (toward negative infinity)

# Logarithms
print(math.log(100, 10))  # 2.0  (log base 10 of 100)
print(math.log(math.e))   # 1.0  (natural log of e)

# Trigonometry (arguments in radians)
print(math.sin(math.pi / 2))  # 1.0
print(math.cos(0))            # 1.0

# Convert degrees to radians and back
print(math.radians(180))      # 3.141592653589793
print(math.degrees(math.pi))  # 180.0
MATLAB Note: MATLAB treats all numbers as double (64-bit floating point) by default. Python distinguishes between int and float. This means:
  • In MATLAB, x = 10 creates a double. In Python, x = 10 creates an int and x = 10.0 creates a float.
  • MATLAB math functions (sqrt, sin, abs, etc.) are available directly. In Python, you need import math first for most of them (abs(), round(), min(), max() are built-in, but sqrt(), sin(), cos() are in the math module).
  • MATLAB uses ^ for exponentiation; Python uses **.
  • MATLAB's mod(a, b) is Python's a % b.
% MATLAB
x = sqrt(144);        % 12 (double)
y = 10 ^ 3;           % 1000
r = mod(17, 5);       % 2

# Python
import math
x = math.sqrt(144)    # 12.0 (float)
y = 10 ** 3           # 1000 (int)
r = 17 % 5            # 2 (int)

4.6 Type Checking and Conversion

You can check a value's type with type() and convert between types using int(), float(), and str().

# Example 22 — Type checking
x = 42
print(type(x))         # <class 'int'>
print(isinstance(x, int))   # True
print(isinstance(x, float)) # False
# Example 23 — Type conversion
# float to int (truncates — does NOT round)
print(int(3.9))        # 3
print(int(-3.9))       # -3

# int to float
print(float(42))       # 42.0

# string to number
print(int("100"))      # 100
print(float("3.14"))   # 3.14

# number to string
print(str(42))         # "42"
print(str(3.14))       # "3.14"
Common Mistake — int() truncates, it does not round: 
print(int(9.99))    # 9, NOT 10
print(int(-2.7))    # -2, NOT -3

# If you want rounding, use round() first:
print(round(9.99))  # 10
print(int(round(9.99)))  # 10

4.7 Augmented Assignment Operators

Python provides shorthand for updating a variable with an arithmetic operation:

# Example 24 — Augmented assignment
x = 10

x += 5    # Same as: x = x + 5   → x is now 15
x -= 3    # Same as: x = x - 3   → x is now 12
x *= 2    # Same as: x = x * 2   → x is now 24
x /= 4    # Same as: x = x / 4   → x is now 6.0 (float!)
x //= 2   # Same as: x = x // 2  → x is now 3.0
x **= 3   # Same as: x = x ** 3  → x is now 27.0
x %= 5    # Same as: x = x % 5   → x is now 2.0

print(x)  # 2.0
Note: Once /= is used, the variable becomes a float and stays a float for all subsequent operations, even //=. If you started with x = 10 (int) and did x /= 2, x is now 5.0 (float), not 5 (int).

4.8 Large Numbers and Underscores

Python handles arbitrarily large integers (no overflow!) and lets you use underscores as visual separators in numbers.

# Example 25 — Large numbers and underscores
big = 1_000_000_000     # Much easier to read than 1000000000
print(big)               # 1000000000

# Python ints have unlimited size
huge = 2 ** 100
print(huge)              # 1267650600228229401496703205376

# Underscores work in floats too
speed_of_light = 299_792_458.0   # meters per second
print(speed_of_light)            # 299792458.0

Exercise 4 — Arithmetic Practice

A rectangle has a width of 7.5 and a height of 12.3. Write a Python script that:

  1. Stores the width and height in variables.
  2. Calculates and prints the area (width × height).
  3. Calculates and prints the perimeter (2 × (width + height)).
  4. Calculates and prints the diagonal using the Pythagorean theorem (use math.sqrt()).
  5. Rounds each result to 2 decimal places.
Show Solution 
import math

# Step 1: Store dimensions
width = 7.5
height = 12.3

# Step 2: Area
area = width * height
print("Area:", round(area, 2))         # 92.25

# Step 3: Perimeter
perimeter = 2 * (width + height)
print("Perimeter:", round(perimeter, 2))  # 39.6

# Step 4: Diagonal (Pythagorean theorem)
diagonal = math.sqrt(width ** 2 + height ** 2)
print("Diagonal:", round(diagonal, 2))   # 14.41
