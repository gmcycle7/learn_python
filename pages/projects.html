Section G — Mini Projects

This is where theory meets practice. Each project below is a complete, runnable Python script that you can copy, save to a .py file, and execute immediately. The projects are ordered roughly by complexity — start with Project 1 and work your way through.

Note for MATLAB users: These projects are the Python equivalent of the small scripts you might write in MATLAB to automate a task, analyse data, or generate a plot. You will see familiar concepts — functions, loops, file I/O, plotting — but expressed in Python syntax. Each project includes notes that map the Python approach back to MATLAB equivalents where helpful.

Project 1: Simple Calculator

Goal

Build an interactive calculator that asks the user for two numbers and an arithmetic operation (+, -, *, /), performs the calculation, and prints the result. The program handles invalid input gracefully instead of crashing.

Concepts Used

functions conditionals input() try / except while loop f-strings

Complete Code

# ============================================================
# simple_calculator.py — Interactive arithmetic calculator
# ============================================================

def add(a, b):
    """Return the sum of a and b."""
    return a + b

def subtract(a, b):
    """Return the difference of a and b."""
    return a - b

def multiply(a, b):
    """Return the product of a and b."""
    return a * b

def divide(a, b):
    """Return the quotient of a divided by b.
    Raises ValueError if b is zero."""
    if b == 0:
        raise ValueError("Cannot divide by zero!")
    return a / b

def get_number(prompt):
    """Ask the user for a number. Keep asking until valid input is given."""
    while True:
        try:
            return float(input(prompt))
        except ValueError:
            print("  Invalid input. Please enter a numeric value.")

def main():
    """Main loop — runs the calculator until the user quits."""
    print("=" * 40)
    print("  Simple Python Calculator")
    print("=" * 40)

    # Map operator symbols to functions
    operations = {
        "+": add,
        "-": subtract,
        "*": multiply,
        "/": divide,
    }

    while True:
        # --- Get the first number ---
        num1 = get_number("\nEnter the first number: ")

        # --- Get the operator ---
        print("Choose an operation: +, -, *, /")
        op = input("Enter operator: ").strip()

        if op not in operations:
            print(f"  Unknown operator '{op}'. Try again.")
            continue  # restart the loop

        # --- Get the second number ---
        num2 = get_number("Enter the second number: ")

        # --- Perform the calculation ---
        try:
            result = operations[op](num1, num2)
            print(f"\n  Result: {num1} {op} {num2} = {result}")
        except ValueError as e:
            print(f"\n  Error: {e}")

        # --- Ask whether to continue ---
        again = input("\nAnother calculation? (y/n): ").strip().lower()
        if again != "y":
            print("Goodbye!")
            break

if __name__ == "__main__":
    main()

Block-by-Block Explanation

1. Arithmetic functions (lines 5–22)

Each operation is wrapped in its own function (add, subtract, multiply, divide). This is cleaner than putting all the logic inside one big if/elif chain and makes each operation independently testable. The divide function explicitly checks for division by zero and raises a ValueError with a clear message.

2. Input helper — get_number() (lines 24–29)

This function uses a while True loop combined with try/except. If the user types something that cannot be converted to a float (for example "abc"), the ValueError is caught and the user is prompted again. The loop only exits when a valid number is returned.

3. Operation dispatch dictionary (lines 39–44)

Instead of writing if op == "+": ... for each operator, we store the functions in a dictionary keyed by the operator string. This is a common Python pattern — dispatch tables — that replaces verbose conditional chains.

MATLAB comparison: In MATLAB you might use a switch statement. Python has no switch (though Python 3.10+ has match/case). The dictionary dispatch pattern is the idiomatic Python approach and is both concise and extensible.

4. Main loop (lines 47–68)

The while True loop keeps the calculator running until the user types something other than y. Inside the loop we collect input, look up the operation, call it with the two numbers, and print the result. The try/except around the calculation call catches the division-by-zero error.

5. Entry-point guard (line 70)

if __name__ == "__main__": ensures main() runs only when the script is executed directly, not when it is imported as a module. This is a Python best practice you will see in every well-structured script.

How to Run

# Save the code as simple_calculator.py, then:
python simple_calculator.py

The program will prompt you interactively in the terminal. Enter numbers and an operator when asked.

Modify & Extend

  • Add more operations: exponentiation (**), modulus (%), integer division (//).
  • Add a history list that stores past calculations and prints them on request.
  • Support expressions like "3 + 5" entered on a single line (use str.split()).
  • Write results to a log file so you have a record of all calculations.

Project 2: File Organizer

Goal

Automatically sort the files in a folder into subfolders based on their file extension. For example, .txt files move into a TextFiles folder, .jpg and .png files move into an Images folder, and so on. Files whose extensions are not in the mapping go into a Misc folder.

Concepts Used

os shutil pathlib dictionaries loops conditionals file I/O

Complete Code

# ============================================================
# file_organizer.py — Sort files in a folder by extension
# ============================================================

import os
import shutil
from pathlib import Path

# ----- Configuration -----
# Map each extension (lowercase) to a target subfolder name.
# Add or remove entries to match your needs.
EXTENSION_MAP = {
    # Documents
    ".txt":  "TextFiles",
    ".pdf":  "Documents",
    ".docx": "Documents",
    ".xlsx": "Spreadsheets",
    ".csv":  "Spreadsheets",
    # Images
    ".jpg":  "Images",
    ".jpeg": "Images",
    ".png":  "Images",
    ".gif":  "Images",
    ".bmp":  "Images",
    # Code / scripts
    ".py":   "PythonScripts",
    ".m":    "MatlabFiles",
    ".js":   "Code",
    ".html": "Code",
    ".css":  "Code",
    # Data
    ".json": "Data",
    ".xml":  "Data",
    # Archives
    ".zip":  "Archives",
    ".tar":  "Archives",
    ".gz":   "Archives",
}

# Fallback folder for extensions not listed above
DEFAULT_FOLDER = "Misc"


def organize(folder_path):
    """Move every file in *folder_path* into a subfolder by extension.

    Parameters
    ----------
    folder_path : str or Path
        The directory whose files you want to organize.
    """
    source = Path(folder_path)

    # Safety check — make sure the path exists and is a directory
    if not source.is_dir():
        print(f"Error: '{source}' is not a valid directory.")
        return

    moved_count = 0  # track how many files we move

    # Iterate over every item in the folder
    for item in source.iterdir():
        # Skip directories — we only want to move files
        if item.is_dir():
            continue

        # Get the file extension in lowercase (e.g. ".TXT" -> ".txt")
        ext = item.suffix.lower()

        # Look up the target folder; fall back to DEFAULT_FOLDER
        target_name = EXTENSION_MAP.get(ext, DEFAULT_FOLDER)

        # Build the full path for the destination subfolder
        target_dir = source / target_name

        # Create the subfolder if it doesn't exist yet
        target_dir.mkdir(exist_ok=True)

        # Build the destination file path
        destination = target_dir / item.name

        # Handle name collisions: if a file with the same name already
        # exists in the target folder, add a numeric suffix.
        counter = 1
        while destination.exists():
            stem = item.stem        # filename without extension
            destination = target_dir / f"{stem}_{counter}{ext}"
            counter += 1

        # Move the file
        shutil.move(str(item), str(destination))
        print(f"  Moved: {item.name}  ->  {target_name}/")
        moved_count += 1

    # Summary
    print(f"\nDone! Moved {moved_count} file(s).")


if __name__ == "__main__":
    # Ask the user which folder to organize
    folder = input("Enter the path of the folder to organize: ").strip()

    # If the user presses Enter with no input, use the current directory
    if not folder:
        folder = "."

    print(f"\nOrganizing files in: {Path(folder).resolve()}\n")
    organize(folder)

Block-by-Block Explanation

1. Imports (lines 5–7)

os provides basic OS interaction, shutil provides the move() function for moving files across directories, and pathlib.Path gives us an object-oriented way to manipulate file paths (much cleaner than joining strings with os.path.join).

2. Extension mapping dictionary (lines 12–39)

A dictionary maps lowercase extensions to folder names. Using a dictionary makes the mapping easy to read and modify. The .get(ext, DEFAULT_FOLDER) call on line 72 returns the mapped folder or "Misc" if the extension is unknown.

3. The organize() function (lines 43–92)

This is the core logic. source.iterdir() yields every entry in the directory. We skip subdirectories, look up each file's extension, create the target subfolder with mkdir(exist_ok=True) (which does not error if it already exists), handle name collisions with a while loop, and finally call shutil.move().

Tip: The exist_ok=True parameter on mkdir() is the Python equivalent of "create if not exists." Without it, calling mkdir() on an existing directory would raise a FileExistsError.

4. Entry point (lines 95–103)

Prompts the user for a folder path, defaults to the current directory if nothing is entered, and calls organize().

MATLAB comparison: In MATLAB you might use dir() to list files and movefile() to move them. Python's pathlib is similar to MATLAB's fullfile() for building paths, but it also provides .suffix, .stem, and .name as convenient properties.

How to Run

# Save as file_organizer.py, then:
python file_organizer.py

# When prompted, enter a folder path (e.g., C:\Users\you\Downloads)
# or press Enter to organize the current directory.
Warning: This script moves files. Test it on a folder with copies of files first, not on your only copy of important data.

Modify & Extend

  • Add a --dry-run flag that prints what would happen without actually moving files.
  • Add a --undo feature by writing a log of moves that can be reversed.
  • Make the extension map configurable via a JSON file.
  • Add recursive organization (organize files inside subfolders too).

Project 3: CSV Analyzer

Goal

Read a CSV file, identify numeric columns, and calculate basic statistics for each one: count, mean, minimum, and maximum. Print a clean summary table to the terminal. This project uses only the standard library — no pandas or numpy required.

Concepts Used

csv module file I/O dictionaries lists math (basic) string formatting

Complete Code

# ============================================================
# csv_analyzer.py — Read a CSV and compute basic statistics
# ============================================================

import csv
import sys


def read_csv(filepath):
    """Read a CSV file and return (headers, rows).

    Parameters
    ----------
    filepath : str
        Path to the CSV file.

    Returns
    -------
    headers : list of str
        Column names from the first row.
    rows : list of list of str
        All data rows (each row is a list of string values).
    """
    with open(filepath, newline="", encoding="utf-8") as f:
        reader = csv.reader(f)
        headers = next(reader)          # first row = column names
        rows = [row for row in reader]   # remaining rows = data
    return headers, rows


def is_numeric(value):
    """Return True if *value* can be converted to a float."""
    try:
        float(value)
        return True
    except (ValueError, TypeError):
        return False


def compute_stats(values):
    """Given a list of floats, return a dict with count, mean, min, max.

    Parameters
    ----------
    values : list of float

    Returns
    -------
    dict with keys: 'count', 'mean', 'min', 'max'
    """
    n = len(values)
    if n == 0:
        return {"count": 0, "mean": None, "min": None, "max": None}

    total = sum(values)
    return {
        "count": n,
        "mean": total / n,
        "min": min(values),
        "max": max(values),
    }


def analyze(filepath):
    """Read the CSV at *filepath* and print statistics for numeric columns."""
    headers, rows = read_csv(filepath)

    # For each column, collect numeric values
    # Key: column index, Value: list of floats
    numeric_data = {}

    for col_idx, header in enumerate(headers):
        col_values = []
        for row in rows:
            # Some rows may have fewer columns; skip gracefully
            if col_idx < len(row) and is_numeric(row[col_idx]):
                col_values.append(float(row[col_idx]))
        # Only keep columns that have at least one numeric value
        if col_values:
            numeric_data[col_idx] = col_values

    if not numeric_data:
        print("No numeric columns found in the CSV.")
        return

    # ----- Print the report -----
    print(f"\nFile: {filepath}")
    print(f"Total rows (excluding header): {len(rows)}")
    print(f"Total columns: {len(headers)}")
    print(f"Numeric columns found: {len(numeric_data)}\n")

    # Table header
    print(f"{'Column':<20} {'Count':>8} {'Mean':>12} {'Min':>12} {'Max':>12}")
    print("-" * 66)

    for col_idx, values in numeric_data.items():
        stats = compute_stats(values)
        name = headers[col_idx]
        print(
            f"{name:<20} "
            f"{stats['count']:>8} "
            f"{stats['mean']:>12.4f} "
            f"{stats['min']:>12.4f} "
            f"{stats['max']:>12.4f}"
        )

    print()


if __name__ == "__main__":
    # Accept file path from command line or prompt for it
    if len(sys.argv) > 1:
        csv_path = sys.argv[1]
    else:
        csv_path = input("Enter the path to the CSV file: ").strip()

    analyze(csv_path)

Block-by-Block Explanation

1. read_csv() (lines 9–30)

Opens the CSV file using Python's built-in csv.reader. The first row is consumed separately with next(reader) to get column headers. The remaining rows are collected into a list of lists using a list comprehension.

2. is_numeric() (lines 33–39)

A small helper that tries to convert a value to float. If the conversion fails, the value is not numeric. This is the standard Python "ask forgiveness, not permission" (EAFP) pattern.

3. compute_stats() (lines 42–60)

Takes a list of floats and returns count, mean, min, and max as a dictionary. Uses the built-in sum(), min(), and max() functions. No external libraries needed.

MATLAB comparison: In MATLAB you would call mean(), min(), max(), and numel() directly on a column vector. Python's built-in functions work on any iterable (lists, tuples, etc.) rather than requiring arrays.

4. analyze() (lines 63–101)

The main analysis function. It reads the CSV, loops through each column to collect numeric values, computes stats for each numeric column, and prints a formatted table using f-string alignment specifiers like <20 (left-align, 20 chars) and >12.4f (right-align, 12 chars, 4 decimal places).

5. Entry point (lines 104–109)

Accepts the CSV path as a command-line argument or prompts for it interactively if no argument is given.

How to Run

# Create a test CSV file first:
# data.csv:
#   Name,Age,Score,City
#   Alice,30,88.5,NYC
#   Bob,25,92.3,LA
#   Charlie,35,76.1,Chicago

# Then run:
python csv_analyzer.py data.csv

# Or run without arguments and enter the path when prompted:
python csv_analyzer.py

Modify & Extend

  • Add standard deviation and median to the statistics.
  • Write the summary to an output CSV or text file.
  • Add support for different delimiters (tabs, semicolons) via a command-line flag.
  • Handle missing values (empty cells) by counting them separately.

Project 4: Simple Plot Generator

Goal

Generate sample data and create two publication-ready charts: a line plot showing a trend over time, and a bar chart comparing categories. Save both plots to image files using savefig().

Concepts Used

matplotlib lists list comprehensions math module savefig() subplots

Complete Code

# ============================================================
# plot_generator.py — Create a line plot and bar chart
# ============================================================

import math
import matplotlib.pyplot as plt


def generate_line_plot(filename="line_plot.png"):
    """Create and save a line plot of a sine-based signal.

    This simulates plotting experimental data over time — a very
    common task for anyone coming from MATLAB.
    """
    # --- Generate data ---
    # 100 evenly spaced x values from 0 to 4*pi
    num_points = 100
    x = [i * (4 * math.pi) / (num_points - 1) for i in range(num_points)]

    # Two signals: a sine wave and a damped sine wave
    y_sine    = [math.sin(xi) for xi in x]
    y_damped  = [math.exp(-0.2 * xi) * math.sin(xi) for xi in x]

    # --- Create the plot ---
    fig, ax = plt.subplots(figsize=(8, 4))

    ax.plot(x, y_sine,   label="sin(x)",                  color="steelblue", linewidth=2)
    ax.plot(x, y_damped, label="exp(-0.2x) * sin(x)",     color="tomato",    linewidth=2, linestyle="--")

    # --- Labels and styling ---
    ax.set_title("Line Plot: Sine Waves", fontsize=14, fontweight="bold")
    ax.set_xlabel("x (radians)")
    ax.set_ylabel("Amplitude")
    ax.legend(loc="upper right")
    ax.grid(True, alpha=0.3)

    # --- Save to file ---
    fig.tight_layout()
    fig.savefig(filename, dpi=150)
    print(f"  Line plot saved to: {filename}")
    plt.close(fig)  # free memory


def generate_bar_chart(filename="bar_chart.png"):
    """Create and save a bar chart comparing programming languages."""

    # --- Data ---
    languages   = ["Python", "JavaScript", "Java", "C++", "MATLAB", "R"]
    popularity  = [30, 22, 18, 12, 8, 6]        # percentage (fictional)
    colors      = ["#3776AB", "#F7DF1E", "#ED8B00", "#00599C", "#EF6C00", "#276DC3"]

    # --- Create the plot ---
    fig, ax = plt.subplots(figsize=(8, 5))

    bars = ax.bar(languages, popularity, color=colors, edgecolor="white", linewidth=0.8)

    # Add value labels on top of each bar
    for bar, val in zip(bars, popularity):
        ax.text(
            bar.get_x() + bar.get_width() / 2,   # x position: center of bar
            bar.get_height() + 0.5,                # y position: just above bar
            f"{val}%",                              # label text
            ha="center", va="bottom", fontsize=10, fontweight="bold"
        )

    # --- Labels and styling ---
    ax.set_title("Bar Chart: Language Popularity", fontsize=14, fontweight="bold")
    ax.set_ylabel("Popularity (%)")
    ax.set_ylim(0, 35)          # leave room for labels above bars
    ax.grid(axis="y", alpha=0.3)

    # --- Save to file ---
    fig.tight_layout()
    fig.savefig(filename, dpi=150)
    print(f"  Bar chart saved to: {filename}")
    plt.close(fig)


def main():
    """Generate both plots."""
    print("Generating plots...\n")
    generate_line_plot()
    generate_bar_chart()
    print("\nAll plots generated successfully.")


if __name__ == "__main__":
    main()

Block-by-Block Explanation

1. Data generation (lines 17–22)

We create 100 x-values from 0 to 4π using a list comprehension and compute two y-series: a plain sine wave and a damped sine wave. This approach uses only the standard math module. If you have numpy installed, you could use numpy.linspace() and vectorized operations instead.

MATLAB comparison: This is the Python equivalent of x = linspace(0, 4*pi, 100); y = sin(x); followed by plot(x, y). In Python with matplotlib, the workflow is nearly identical: generate data, call ax.plot(), set labels, save or show.

2. Creating the figure (line 25)

plt.subplots() returns a Figure and an Axes object. This is the "object-oriented" matplotlib API, which is recommended over the simpler plt.plot() style because it gives you full control over each subplot.

3. Styling and saving (lines 31–40)

We set the title, axis labels, legend, and grid. Then fig.tight_layout() adjusts spacing so nothing is clipped, and fig.savefig() writes the plot to a PNG file at 150 DPI. Finally plt.close(fig) frees memory.

4. Bar chart with value labels (lines 57–64)

After creating the bars, we loop over them to place text labels above each bar using ax.text(). The zip(bars, popularity) pairs each bar object with its value, making it easy to position the label.

How to Run

# Install matplotlib first (one-time):
pip install matplotlib

# Then run:
python plot_generator.py

# Two image files will be created in the current directory:
#   line_plot.png
#   bar_chart.png

Modify & Extend

  • Read data from a CSV file instead of generating it in code.
  • Add a scatter plot or histogram as a third chart.
  • Create a multi-panel figure with all charts in one image using plt.subplots(1, 3).
  • Accept the output filename and DPI as command-line arguments.

Project 5: Log Parser

Goal

Read a log file, extract all lines that contain error messages, count how many times each unique error occurs, and write a summary report to a new file. This is the kind of text-processing task Python excels at.

Concepts Used

file I/O string methods dictionaries sorting loops f-strings

Complete Code

# ============================================================
# log_parser.py — Parse a log file and summarize errors
# ============================================================

import sys
from datetime import datetime


def parse_log(filepath):
    """Read a log file and extract error lines.

    Assumes each log line has the format:
        [TIMESTAMP] LEVEL: message
    For example:
        [2025-01-15 10:23:45] ERROR: Connection timed out

    Returns
    -------
    error_lines : list of str
        The full text of each error line.
    error_counts : dict
        Maps each unique error message to its occurrence count.
    total_lines : int
        Total number of lines in the file.
    """
    error_lines = []
    error_counts = {}
    total_lines = 0

    with open(filepath, "r", encoding="utf-8") as f:
        for line in f:
            total_lines += 1
            stripped = line.strip()

            # Check if this line contains an ERROR-level message
            if "ERROR" in stripped.upper():
                error_lines.append(stripped)

                # Extract the message part (after "ERROR:")
                # If "ERROR:" exists, take everything after it; otherwise use the full line
                if "ERROR:" in stripped:
                    msg = stripped.split("ERROR:", 1)[1].strip()
                elif "error:" in stripped:
                    msg = stripped.split("error:", 1)[1].strip()
                else:
                    msg = stripped  # fallback: use whole line

                # Count occurrences of this exact message
                error_counts[msg] = error_counts.get(msg, 0) + 1

    return error_lines, error_counts, total_lines


def write_report(error_lines, error_counts, total_lines, log_path, report_path):
    """Write an error summary report to *report_path*.

    Parameters
    ----------
    error_lines : list of str
    error_counts : dict
    total_lines : int
    log_path : str
    report_path : str
    """
    # Sort errors by count, highest first
    sorted_errors = sorted(error_counts.items(), key=lambda x: x[1], reverse=True)

    with open(report_path, "w", encoding="utf-8") as f:
        f.write("=" * 60 + "\n")
        f.write("  LOG ERROR SUMMARY REPORT\n")
        f.write(f"  Generated: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}\n")
        f.write("=" * 60 + "\n\n")

        f.write(f"Source log file: {log_path}\n")
        f.write(f"Total lines in log: {total_lines}\n")
        f.write(f"Total error lines:  {len(error_lines)}\n")
        f.write(f"Unique errors:      {len(error_counts)}\n\n")

        # --- Error frequency table ---
        f.write("-" * 60 + "\n")
        f.write(f"{'Count':>6}  {'Error Message'}\n")
        f.write("-" * 60 + "\n")
        for msg, count in sorted_errors:
            f.write(f"{count:>6}  {msg}\n")
        f.write("-" * 60 + "\n\n")

        # --- Full error lines ---
        f.write("FULL ERROR LINES:\n\n")
        for i, line in enumerate(error_lines, start=1):
            f.write(f"  [{i}] {line}\n")
        f.write("\n" + "=" * 60 + "\n")
        f.write("  END OF REPORT\n")
        f.write("=" * 60 + "\n")


def main():
    """Parse the log file and generate the report."""
    # Get file paths
    if len(sys.argv) > 1:
        log_path = sys.argv[1]
    else:
        log_path = input("Enter path to the log file: ").strip()

    report_path = log_path.rsplit(".", 1)[0] + "_error_report.txt"

    print(f"Parsing: {log_path}")
    error_lines, error_counts, total_lines = parse_log(log_path)

    if not error_lines:
        print("No errors found in the log file.")
        return

    # Write the report
    write_report(error_lines, error_counts, total_lines, log_path, report_path)

    # Print a quick summary to the terminal as well
    print(f"\nTotal lines:  {total_lines}")
    print(f"Error lines:  {len(error_lines)}")
    print(f"Unique errors: {len(error_counts)}")
    print(f"\nTop errors:")
    for msg, count in sorted(error_counts.items(), key=lambda x: x[1], reverse=True)[:5]:
        print(f"  {count:>4}x  {msg}")
    print(f"\nFull report written to: {report_path}")


if __name__ == "__main__":
    main()

Block-by-Block Explanation

Project 6: Batch File Renamer

Goal

Rename all files in a folder according to a pattern: add a prefix, sequential numbering, and optionally the current date. For example, rename photo.jpg to vacation_001_2025-01-15_photo.jpg.

Concepts Used

Complete Code

# ============================================================
# batch_renamer.py — Rename files with prefix, number, and date
# ============================================================

import os
from pathlib import Path
from datetime import datetime


def preview_renames(folder_path, prefix="file", include_date=True, start_num=1,
                    num_digits=3, extensions=None):
    """Build a list of (old_name, new_name) pairs WITHOUT renaming.

    Parameters
    ----------
    folder_path : str or Path
        The directory containing files to rename.
    prefix : str
        String to prepend to each filename.
    include_date : bool
        Whether to include today's date in the new name.
    start_num : int
        Starting number for the sequential counter.
    num_digits : int
        Zero-pad the counter to this many digits (e.g. 3 -> 001, 002).
    extensions : set or None
        If provided, only rename files with these extensions (e.g. {".jpg", ".png"}).
        If None, rename all files.

    Returns
    -------
    rename_pairs : list of (Path, Path)
        Each tuple is (old_path, new_path).
    """
    source = Path(folder_path)
    if not source.is_dir():
        print(f"Error: '{source}' is not a valid directory.")
        return []

    # Get the date string (e.g. "2025-01-15")
    date_str = datetime.now().strftime("%Y-%m-%d") if include_date else ""

    # Collect and sort files (sorted by name for consistent ordering)
    files = sorted(
        [f for f in source.iterdir() if f.is_file()],
        key=lambda f: f.name.lower()
    )

    # Filter by extension if specified
    if extensions:
        extensions = {ext.lower() for ext in extensions}
        files = [f for f in files if f.suffix.lower() in extensions]

    rename_pairs = []
    for i, filepath in enumerate(files, start=start_num):
        # Build the new filename
        number_part = str(i).zfill(num_digits)     # e.g. "001"

        if include_date:
            new_name = f"{prefix}_{number_part}_{date_str}{filepath.suffix}"
        else:
            new_name = f"{prefix}_{number_part}{filepath.suffix}"

        new_path = filepath.parent / new_name
        rename_pairs.append((filepath, new_path))

    return rename_pairs


def execute_renames(rename_pairs):
    """Perform the actual file renames.

    Parameters
    ----------
    rename_pairs : list of (Path, Path)
    """
    for old_path, new_path in rename_pairs:
        # Safety: don't overwrite existing files
        if new_path.exists():
            print(f"  SKIP (already exists): {new_path.name}")
            continue
        os.rename(str(old_path), str(new_path))
        print(f"  Renamed: {old_path.name}  ->  {new_path.name}")


def main():
    """Interactive batch file renamer."""
    print("=" * 50)
    print("  Batch File Renamer")
    print("=" * 50)

    folder = input("\nFolder to rename files in: ").strip() or "."
    prefix = input("Prefix for new filenames [file]: ").strip() or "file"

    date_choice = input("Include date in filename? (y/n) [y]: ").strip().lower()
    include_date = date_choice != "n"

    ext_input = input("Filter by extensions (comma-separated, e.g. .jpg,.png) or Enter for all: ").strip()
    extensions = None
    if ext_input:
        extensions = {e.strip() for e in ext_input.split(",")}

    # Generate the preview
    rename_pairs = preview_renames(folder, prefix, include_date, extensions=extensions)

    if not rename_pairs:
        print("\nNo files to rename.")
        return

    # Show preview
    print(f"\n--- Preview ({len(rename_pairs)} files) ---\n")
    for old_path, new_path in rename_pairs:
        print(f"  {old_path.name}  ->  {new_path.name}")

    # Confirm
    confirm = input(f"\nProceed with renaming {len(rename_pairs)} files? (y/n): ").strip().lower()
    if confirm == "y":
        print()
        execute_renames(rename_pairs)
        print(f"\nDone! Renamed {len(rename_pairs)} file(s).")
    else:
        print("Cancelled. No files were renamed.")


if __name__ == "__main__":
    main()

Project 7: Folder Report Generator

Goal

Scan an entire directory tree (including all subdirectories) and produce a report showing: total number of files, total size, file counts and sizes grouped by extension, and the largest files. This is useful for understanding what is taking up disk space.

Concepts Used

os.walk() pathlib dictionaries sorting string formatting functions

Complete Code

# ============================================================
# folder_report.py — Scan a directory tree and generate a report
# ============================================================

import os
import sys
from pathlib import Path
from datetime import datetime


def format_size(size_bytes):
    """Convert a size in bytes to a human-readable string.

    Examples: 1024 -> '1.00 KB', 1048576 -> '1.00 MB'
    """
    for unit in ["B", "KB", "MB", "GB", "TB"]:
        if size_bytes < 1024:
            return f"{size_bytes:.2f} {unit}"
        size_bytes /= 1024
    return f"{size_bytes:.2f} PB"


def scan_directory(root_path):
    """Walk a directory tree and collect file information.

    Parameters
    ----------
    root_path : str or Path
        The top-level directory to scan.

    Returns
    -------
    file_list : list of dict
        Each dict has keys: 'path', 'name', 'ext', 'size'.
    dir_count : int
        Number of subdirectories found.
    """
    root = Path(root_path)
    file_list = []
    dir_count = 0

    for dirpath, dirnames, filenames in os.walk(root):
        dir_count += len(dirnames)

        for fname in filenames:
            full_path = Path(dirpath) / fname
            try:
                size = full_path.stat().st_size
            except (OSError, PermissionError):
                # Skip files we can't access
                continue

            ext = full_path.suffix.lower() if full_path.suffix else "(no extension)"

            file_list.append({
                "path": str(full_path),
                "name": fname,
                "ext": ext,
                "size": size,
            })

    return file_list, dir_count


def generate_report(root_path, file_list, dir_count, top_n=10):
    """Generate and print a formatted report.

    Parameters
    ----------
    root_path : str
    file_list : list of dict
    dir_count : int
    top_n : int
        Number of largest files to show.
    """
    total_size = sum(f["size"] for f in file_list)
    total_files = len(file_list)

    # --- Group by extension ---
    ext_stats = {}  # ext -> {"count": int, "size": int}
    for f in file_list:
        ext = f["ext"]
        if ext not in ext_stats:
            ext_stats[ext] = {"count": 0, "size": 0}
        ext_stats[ext]["count"] += 1
        ext_stats[ext]["size"] += f["size"]

    # Sort extensions by total size (descending)
    sorted_exts = sorted(ext_stats.items(), key=lambda x: x[1]["size"], reverse=True)

    # --- Largest files ---
    largest_files = sorted(file_list, key=lambda x: x["size"], reverse=True)[:top_n]

    # --- Build the report string ---
    lines = []
    lines.append("=" * 70)
    lines.append("  FOLDER REPORT")
    lines.append(f"  Generated: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
    lines.append("=" * 70)
    lines.append("")
    lines.append(f"  Root directory:   {root_path}")
    lines.append(f"  Total files:      {total_files:,}")
    lines.append(f"  Total folders:    {dir_count:,}")
    lines.append(f"  Total size:       {format_size(total_size)}")
    lines.append("")

    # --- Extension breakdown ---
    lines.append("-" * 70)
    lines.append(f"  {'Extension':<18} {'Files':>8} {'Total Size':>14} {'% of Total':>12}")
    lines.append("-" * 70)
    for ext, stats in sorted_exts:
        pct = (stats["size"] / total_size * 100) if total_size > 0 else 0
        lines.append(
            f"  {ext:<18} {stats['count']:>8,} {format_size(stats['size']):>14} {pct:>11.1f}%"
        )
    lines.append("-" * 70)
    lines.append("")

    # --- Largest files ---
    lines.append(f"  TOP {top_n} LARGEST FILES:")
    lines.append("")
    for i, f in enumerate(largest_files, start=1):
        lines.append(f"  {i:>3}. {format_size(f['size']):>10}  {f['path']}")
    lines.append("")
    lines.append("=" * 70)

    report = "\n".join(lines)
    return report


def main():
    """Scan a directory and print / save the report."""
    if len(sys.argv) > 1:
        root_path = sys.argv[1]
    else:
        root_path = input("Enter directory to scan: ").strip() or "."

    root_path = str(Path(root_path).resolve())
    print(f"\nScanning: {root_path}\n")

    file_list, dir_count = scan_directory(root_path)

    if not file_list:
        print("No files found.")
        return

    report = generate_report(root_path, file_list, dir_count)

    # Print to terminal
    print(report)

    # Save to file
    report_file = "folder_report.txt"
    with open(report_file, "w", encoding="utf-8") as f:
        f.write(report)
    print(f"\nReport saved to: {report_file}")


if __name__ == "__main__":
    main()

Block-by-Block Explanation

1. format_size() (lines 10–17)

A utility function that converts raw byte counts into human-readable strings like "3.45 MB". It divides by 1024 repeatedly, stepping through units (B, KB, MB, GB, TB) until the number is less than 1024.

2. scan_directory() (lines 20–58)

Uses os.walk(), which recursively traverses a directory tree. For each file it records the path, name, extension, and size. The try/except block handles permission errors gracefully — some system files may not be accessible.

Project 8: Simple Numerical Analysis

Goal

Generate a synthetic dataset with noise, calculate descriptive statistics, fit a straight line (linear regression) using basic math, and plot the data along with the fitted line. This project is especially relevant for MATLAB users who are accustomed to data analysis and curve fitting.

Complete Code

# ============================================================
# numerical_analysis.py — Generate data, compute stats, fit line, plot
# ============================================================
# This script uses only the standard library + matplotlib.
# (It does NOT require numpy, though numpy would simplify much of this.)

import math
import random
import matplotlib.pyplot as plt


# ----- Data Generation -----

def generate_data(n=50, slope=2.5, intercept=10, noise_level=5.0, seed=42):
    """Generate n data points along a line with Gaussian noise.

    y = slope * x + intercept + noise

    Parameters
    ----------
    n : int            Number of data points.
    slope : float      True slope of the line.
    intercept : float  True y-intercept.
    noise_level : float  Standard deviation of the added noise.
    seed : int         Random seed for reproducibility.

    Returns
    -------
    x_data : list of float
    y_data : list of float
    """
    random.seed(seed)
    x_data = [i * 1.0 for i in range(n)]
    y_data = [slope * x + intercept + random.gauss(0, noise_level) for x in x_data]
    return x_data, y_data


# ----- Statistics -----

def calc_mean(values):
    """Return the arithmetic mean of a list of numbers."""
    return sum(values) / len(values)


def calc_std(values):
    """Return the sample standard deviation."""
    mean = calc_mean(values)
    variance = sum((v - mean) ** 2 for v in values) / (len(values) - 1)
    return math.sqrt(variance)


def calc_median(values):
    """Return the median of a list of numbers."""
    s = sorted(values)
    n = len(s)
    mid = n // 2
    if n % 2 == 0:
        return (s[mid - 1] + s[mid]) / 2
    else:
        return s[mid]


def print_stats(label, values):
    """Print a summary of descriptive statistics."""
    print(f"\n  Statistics for {label}:")
    print(f"    Count:    {len(values)}")
    print(f"    Mean:     {calc_mean(values):.4f}")
    print(f"    Median:   {calc_median(values):.4f}")
    print(f"    Std Dev:  {calc_std(values):.4f}")
    print(f"    Min:      {min(values):.4f}")
    print(f"    Max:      {max(values):.4f}")


# ----- Linear Regression (least squares) -----

def linear_fit(x, y):
    """Fit a line y = mx + b using the least-squares method.

    This implements the closed-form solution:
        m = (n * sum(x*y) - sum(x) * sum(y)) / (n * sum(x^2) - (sum(x))^2)
        b = mean(y) - m * mean(x)

    Parameters
    ----------
    x : list of float
    y : list of float

    Returns
    -------
    m : float   slope
    b : float   intercept
    r_squared : float   coefficient of determination (R^2)
    """
    n = len(x)
    sum_x   = sum(x)
    sum_y   = sum(y)
    sum_xy  = sum(xi * yi for xi, yi in zip(x, y))
    sum_x2  = sum(xi ** 2 for xi in x)
    sum_y2  = sum(yi ** 2 for yi in y)

    # Slope
    denom = n * sum_x2 - sum_x ** 2
    m = (n * sum_xy - sum_x * sum_y) / denom

    # Intercept
    b = (sum_y - m * sum_x) / n

    # R-squared
    ss_res = sum((yi - (m * xi + b)) ** 2 for xi, yi in zip(x, y))
    ss_tot = sum((yi - calc_mean(y)) ** 2 for yi in y)
    r_squared = 1 - ss_res / ss_tot if ss_tot != 0 else 0

    return m, b, r_squared


# ----- Plotting -----

def plot_results(x, y, m, b, r_squared, filename="numerical_analysis.png"):
    """Plot the data points and the fitted line, and save to file."""
    # Compute the fitted line values
    y_fit = [m * xi + b for xi in x]

    fig, ax = plt.subplots(figsize=(8, 5))

    # Scatter plot of the raw data
    ax.scatter(x, y, color="steelblue", alpha=0.6, s=30, label="Data points")

    # Fitted line
    ax.plot(x, y_fit, color="tomato", linewidth=2,
            label=f"Fit: y = {m:.2f}x + {b:.2f} (R² = {r_squared:.4f})")

    ax.set_title("Numerical Analysis: Linear Fit", fontsize=14, fontweight="bold")
    ax.set_xlabel("x")
    ax.set_ylabel("y")
    ax.legend()
    ax.grid(True, alpha=0.3)

    fig.tight_layout()
    fig.savefig(filename, dpi=150)
    print(f"\n  Plot saved to: {filename}")
    plt.close(fig)


# ----- Main -----

def main():
    """Run the full analysis pipeline."""
    print("=" * 50)
    print("  Simple Numerical Analysis")
    print("=" * 50)

    # Step 1: Generate data
    x, y = generate_data(n=50, slope=2.5, intercept=10, noise_level=5.0)
    print(f"\nGenerated {len(x)} data points.")

    # Step 2: Descriptive statistics
    print_stats("x", x)
    print_stats("y", y)

    # Step 3: Linear regression
    m, b, r_sq = linear_fit(x, y)
    print(f"\n  Linear Fit Results:")
    print(f"    Slope (m):     {m:.4f}   (true value: 2.5)")
    print(f"    Intercept (b): {b:.4f}   (true value: 10.0)")
    print(f"    R-squared:     {r_sq:.4f}")

    # Step 4: Plot
    plot_results(x, y, m, b, r_sq)
    print("\nAnalysis complete.")


if __name__ == "__main__":
    main()

1. Data generation (lines 14–36)

Creates n data points along the line y = 2.5x + 10 with Gaussian noise added. Setting random.seed(42) ensures the results are reproducible every time you run the script.

If you install numpy, you can write nearly identical code: x = np.arange(50); y = 2.5*x + 10 + 5*np.random.randn(50).

Project 9: Command Line Utility

# ============================================================
# wordcount.py — A CLI word/line/character counter for text files
# ============================================================
# Usage:
#   python wordcount.py file1.txt file2.txt
#   python wordcount.py *.txt --sort words --top 5
#   python wordcount.py myfile.txt --no-total

import argparse
import sys
from pathlib import Path


def count_file(filepath):
    """Count lines, words, and characters in a single file.

    Parameters
    ----------
    filepath : str or Path
        Path to the text file.

    Returns
    -------
    dict with keys: 'lines', 'words', 'chars', 'file'
    Returns None if the file cannot be read.
    """
    try:
        with open(filepath, "r", encoding="utf-8") as f:
            content = f.read()
    except FileNotFoundError:
        print(f"  Error: File not found: {filepath}", file=sys.stderr)
        return None
    except PermissionError:
        print(f"  Error: Permission denied: {filepath}", file=sys.stderr)
        return None
    except UnicodeDecodeError:
        print(f"  Error: Cannot read (not a text file?): {filepath}", file=sys.stderr)
        return None

    lines = content.count("\n")
    words = len(content.split())
    chars = len(content)

    return {
        "lines": lines,
        "words": words,
        "chars": chars,
        "file": str(filepath),
    }


def print_results(results, show_total=True, sort_by=None, top_n=None):
    """Print a formatted table of word-count results.

    Parameters
    ----------
    results : list of dict
        Each dict from count_file().
    show_total : bool
        Whether to print a total row at the bottom.
    sort_by : str or None
        Sort results by 'lines', 'words', or 'chars' (descending).
    top_n : int or None
        Show only the top N files (after sorting).
    """
    if not results:
        print("No results to display.")
        return

    # Optional sorting
    if sort_by and sort_by in ("lines", "words", "chars"):
        results = sorted(results, key=lambda r: r[sort_by], reverse=True)

    # Optional top-N filtering
    if top_n is not None and top_n > 0:
        results = results[:top_n]

    # Print header
    print(f"\n{'Lines':>10} {'Words':>10} {'Chars':>10}  {'File'}")
    print("-" * 50)

    # Print each file's counts
    total_lines = 0
    total_words = 0
    total_chars = 0

    for r in results:
        print(f"{r['lines']:>10,} {r['words']:>10,} {r['chars']:>10,}  {r['file']}")
        total_lines += r["lines"]
        total_words += r["words"]
        total_chars += r["chars"]

    # Print total (only if more than one file and show_total is True)
    if show_total and len(results) > 1:
        print("-" * 50)
        print(f"{total_lines:>10,} {total_words:>10,} {total_chars:>10,}  TOTAL")

    print()


def build_parser():
    """Create and return the argument parser."""
    parser = argparse.ArgumentParser(

        epilog="Example: python wordcount.py report.txt notes.txt --sort words",
    )

    # Positional arguments: one or more file paths
    parser.add_argument(
        "files",
        nargs="+",
        help="One or more text files to analyze.",
    )

    # Optional flags
    parser.add_argument(
        "--sort",
        choices=["lines", "words", "chars"],
        default=None,
        help="Sort results by the specified metric (descending).",
    )

    parser.add_argument(
        "--top",
        type=int,
        default=None,
        metavar="N",
        help="Show only the top N files (requires --sort).",
    )

    parser.add_argument(
        "--no-total",
        action="store_true",
        help="Do not show the total row.",
    )

    return parser


def main():
    """Parse arguments and run the word counter."""
    parser = build_parser()
    args = parser.parse_args()

    # Count each file
    results = []
    for filepath in args.files:
        result = count_file(filepath)
        if result is not None:
            results.append(result)

    if not results:
        print("No valid files to analyze.", file=sys.stderr)
        sys.exit(1)

    # Display results
    print_results(
        results,
        show_total=not args.no_total,
        sort_by=args.sort,
        top_n=args.top,
    )


if __name__ == "__main__":
    main()

Block-by-Block Explanation

1. count_file() (lines 14–49)

Reads a single file and computes three metrics: line count (number of newline characters), word count (using str.split(), which splits on any whitespace), and character count (len(content)). The function handles three common errors gracefully: file not found, permission denied, and non-text files.

2. print_results() (lines 52–90)

Formats and prints a table of results. Supports optional sorting by any metric and an optional top-N filter. The format specifiers >10, right-align numbers in a 10-character field with comma separators for thousands.

3. build_parser() (lines 93–129)

This is where argparse shines. We define:

  • files — a positional argument that accepts one or more file paths (nargs="+").
  • --sort — an optional flag with restricted choices.
  • --top N — an optional integer flag.
  • --no-total — a boolean flag (present = True, absent = False).

Running python wordcount.py --help automatically generates a help message from these definitions.

Tip: argparse is the standard-library way to build CLI tools in Python. For simple scripts, sys.argv works fine, but for anything with flags, optional arguments, or help text, argparse is the right choice. There are also third-party alternatives like click and typer that offer even more convenience.
MATLAB comparison: MATLAB scripts do not have a built-in argument parser like argparse. MATLAB users typically pass arguments via function parameters or use inputParser. The Python approach makes it easy to build scripts that behave like proper command-line tools.

4. main() (lines 132–153)

Parses arguments, counts each file, filters out failures, and passes the results to the printer. Exits with status code 1 if no valid files were processed.

How to Run

# Basic usage — count a single file:
python wordcount.py myfile.txt

python wordcount.py file1.txt file2.txt file3.txt

python wordcount.py report.txt notes.txt --no-total
