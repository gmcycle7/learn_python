Section C5 — Conditionals (if / elif / else)

Every useful program needs to make decisions. Should you display an error message? Is the user old enough? Did the sensor reading exceed a threshold? In Python, you make decisions with conditional statements — if, elif, and else. This page covers everything you need to know about branching logic in Python.

Note for MATLAB users: You already know if ... elseif ... else ... end. Python's version is almost identical in logic, but the syntax differs in three important ways: (1) Python uses elif instead of elseif, (2) every condition line ends with a colon, and (3) there is no end keyword — Python uses indentation alone to define blocks.

1. The if Statement

The simplest conditional is a plain if. It runs a block of code only when a condition is True.

Syntax

if condition:
    # This block runs only when condition is True
    do_something()

Key rules:

  • The line must end with a colon (:).
  • The indented block below is the body. Python uses indentation (typically 4 spaces) to define which lines belong to the if.
  • The condition is any expression that evaluates to True or False.

Example 1 — Basic if

temperature = 35

if temperature > 30:
    print("It's hot outside!")

# Output: It's hot outside!

Because temperature is 35, the condition 35 > 30 is True, so the indented print() line executes.

Example 2 — Nothing happens when False

temperature = 20

if temperature > 30:
    print("It's hot outside!")

print("Done checking.")
# Output: Done checking.

The condition is False, so the indented line is skipped entirely. The program continues with the next unindented line.

Indentation is not optional. In most languages, indentation is just a style choice. In Python, it is part of the syntax. If you indent incorrectly, Python raises an IndentationError. Always use 4 spaces per level (your editor can do this automatically).

2. if / else

Often you want to do one thing when a condition is true and a different thing when it is false. That is what else is for.

Syntax

if condition:
    # runs when True
else:
    # runs when False

Example 3 — Pass or fail

score = 72

if score >= 60:
    print("You passed!")
else:
    print("You failed. Try again.")

# Output: You passed!

Example 4 — Even or odd

number = 7

if number % 2 == 0:
    print(f"{number} is even")
else:
    print(f"{number} is odd")

# Output: 7 is odd

The modulo operator % gives the remainder after division. If number % 2 equals 0, the number is even; otherwise it is odd.

3. elif — Multiple Conditions

When you have more than two branches, use elif (short for "else if"). Python checks each condition from top to bottom and runs the first block whose condition is True. If none match, the else block runs (if present).

Syntax

if condition_1:
    # runs if condition_1 is True
elif condition_2:
    # runs if condition_1 is False AND condition_2 is True
elif condition_3:
    # runs if both above are False AND condition_3 is True
else:
    # runs if ALL conditions above are False

You can have as many elif branches as you need. The else at the end is optional but recommended as a catch-all.

Example 5 — Letter grades

score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Score: {score}, Grade: {grade}")
# Output: Score: 85, Grade: B

Python tests each condition in order. 85 >= 90 is False, so it moves on. 85 >= 80 is True, so grade is set to "B" and all remaining branches are skipped.

MATLAB comparison: 
% MATLAB
if score >= 90
    grade = 'A';
elseif score >= 80
    grade = 'B';
elseif score >= 70
    grade = 'C';
elseif score >= 60
    grade = 'D';
else
    grade = 'F';
end

Notice MATLAB uses elseif (one word) and requires end. Python uses elif and uses indentation instead of end. The logic is identical.

4. Nested if Statements

You can place an if statement inside another if statement. This is called nesting. The inner if only runs when the outer condition is already True.

Example 6 — Nested check

age = 25
has_ticket = True

if age >= 18:
    if has_ticket:
        print("Welcome to the concert!")
    else:
        print("You need a ticket.")
else:
    print("Sorry, you must be 18 or older.")

# Output: Welcome to the concert!

Each level of nesting adds another 4 spaces of indentation. While nesting works, deeply nested code (3+ levels) becomes hard to read. In practice, you can often simplify nested conditions by combining them with and (see below).

Example 7 — Flattened version using and

age = 25
has_ticket = True

if age >= 18 and has_ticket:
    print("Welcome to the concert!")
elif age < 18:
    print("Sorry, you must be 18 or older.")
else:
    print("You need a ticket.")

This is cleaner and easier to read. Prefer flattening when possible.

5. Combining Conditions with and, or, not

Python provides three logical operators for combining conditions:

Operator Meaning True when…
and Both must be true A and B → True only if A is True and B is True
or At least one must be true A or B → True if A is True or B is True (or both)
not Reverses the truth value not A → True if A is False

Example 8 — Using and

username = "admin"
password = "secret123"

if username == "admin" and password == "secret123":
    print("Login successful!")
else:
    print("Invalid credentials.")

# Output: Login successful!

Both conditions must be True for the combined expression to be True.

Example 9 — Using or

day = "Saturday"

if day == "Saturday" or day == "Sunday":
    print("It's the weekend!")
else:
    print("It's a weekday.")

# Output: It's the weekend!

Example 10 — Using not

is_raining = False

if not is_raining:
    print("No umbrella needed.")

# Output: No umbrella needed.

not flips False to True, so the block runs.

Example 11 — Combining multiple operators

temperature = 22
is_sunny = True

if temperature > 20 and temperature < 35 and is_sunny:
    print("Perfect day for a walk!")

# Output: Perfect day for a walk!
Operator precedence: not is evaluated first, then and, then or. When in doubt, use parentheses to make your intent clear: if (a or b) and c:
MATLAB comparison: MATLAB uses && for logical AND and || for logical OR. Python spells them out as English words: and and or. MATLAB's ~ (tilde for NOT) becomes not in Python.

Conditions rely on comparison operators. Here is the complete list:

Chained comparisons: Python lets you chain comparisons naturally. Instead of writing x > 0 and x < 100, you can write 0 < x < 100. This reads just like math notation and is perfectly valid Python.

Example 12 — Chained comparison

age = 25

if 18 <= age <= 65:
    print("Working age")

# Output: Working age

7. Ternary Expressions (Inline if/else)

Sometimes you want to assign a value based on a condition in a single line. Python has a concise syntax for this called a ternary expression (also called a conditional expression).

Syntax

value = a if condition else b

This reads as: "Set value to a if the condition is true; otherwise, set it to b."

Example 13 — Simple ternary

age = 20

status = "adult" if age >= 18 else "minor"
print(status)
# Output: adult

Example 14 — Ternary in a print statement

score = 45

print("Pass" if score >= 50 else "Fail")
# Output: Fail

Example 15 — Ternary for choosing a value

x = -7

absolute_value = x if x >= 0 else -x
print(absolute_value)
# Output: 7
When to use ternaries: They are great for simple, one-line assignments. If the logic is complex or involves multiple branches, use a full if/elif/else block instead. Readability always wins over cleverness.
MATLAB note: MATLAB does not have a built-in ternary operator. The closest equivalent is a one-line if or using indexing tricks. Python's ternary syntax is cleaner and more readable.

8. Truthiness and Falsiness

In Python, you do not always need an explicit comparison. Every value has an inherent truthiness. When you write if x:, Python evaluates whether x is "truthy" or "falsy."

Falsy values (evaluate to False)

ValueType
Falsebool
NoneNoneType
0int
0.0float
"" (empty string)str
[] (empty list)list
{} (empty dict)dict
() (empty tuple)tuple
set() (empty set)set

Everything else is truthy. Non-zero numbers, non-empty strings, non-empty collections — all truthy.

Example 16 — Checking for an empty list

shopping_list = []

if shopping_list:
    print("Let's go shopping!")
else:
    print("Nothing to buy.")

# Output: Nothing to buy.

An empty list is falsy, so if shopping_list: evaluates to False. This is the Pythonic way to check whether a collection has items — you do not need to write if len(shopping_list) > 0:.

Example 17 — Checking for a non-empty string

user_input = "hello"

if user_input:
    print(f"You typed: {user_input}")
else:
    print("You didn't type anything.")

# Output: You typed: hello
Why this matters: Understanding truthiness is essential for writing clean, idiomatic Python. Instead of verbose comparisons like if name != "": or if count != 0:, experienced Python programmers write if name: and if count:. It is shorter, cleaner, and considered best practice.

9. Common Mistakes

Conditionals are simple in concept, but beginners often trip on a few predictable pitfalls. Here are the most common ones and how to fix them.

Mistake 1: Using = instead of ==

The single equals sign = is for assignment (storing a value). The double equals sign == is for comparison (checking equality). Mixing them up is one of the most common beginner errors. 

# WRONG — this is assignment, not comparison
x = 10
if x = 10:       # SyntaxError!
    print("ten")

# CORRECT — use == for comparison
x = 10
if x == 10:
    print("ten")  # Output: ten
Good news: Unlike C or JavaScript, Python will raise a SyntaxError if you accidentally use = inside an if. The error message will point you to the problem.

Mistake 2: Forgetting the colon

# WRONG — missing colon at the end
if score > 90
    print("A+")

# CORRECT
if score > 90:
    print("A+")

Every if, elif, and else line must end with a colon. Forgetting it produces a SyntaxError.

Mistake 3: Wrong indentation

# WRONG — inconsistent indentation
if temperature > 30:
    print("Hot")
      print("Drink water")   # IndentationError!

# CORRECT — both lines have the same indentation
if temperature > 30:
    print("Hot")
    print("Drink water")

All lines in the same block must have exactly the same indentation. Mixing tabs and spaces is a recipe for trouble — configure your editor to insert 4 spaces when you press Tab.

Mistake 4: Writing elif as else if

# WRONG — Python does not use "else if"
if x > 0:
    print("positive")
else if x < 0:       # SyntaxError!
    print("negative")

# CORRECT — use elif (one word)
if x > 0:
    print("positive")
elif x < 0:
    print("negative")

Mistake 5: Checking multiple values incorrectly

# WRONG — does not do what you think
color = "red"
if color == "red" or "blue":   # Always True!
    print("Match")

# WHY? Python reads this as: (color == "red") or ("blue")
# The string "blue" is truthy, so the whole expression is always True.

# CORRECT
if color == "red" or color == "blue":
    print("Match")

# EVEN BETTER — use the 'in' operator
if color in ("red", "blue"):
    print("Match")
Why this matters: The color == "red" or "blue" bug is extremely common among beginners. It looks logical in English ("if color equals red or blue") but Python interprets it differently. Always use in for checking membership in a set of values.

11. Real-World Examples

Example 18 — Simple calculator

a = 10
b = 3
operator = "+"

if operator == "+":
    result = a + b
elif operator == "-":
    result = a - b
elif operator == "*":
    result = a * b
elif operator == "/":
    if b != 0:
        result = a / b
    else:
        result = "Error: division by zero"
else:
    result = "Unknown operator"

print(f"{a} {operator} {b} = {result}")
# Output: 10 + 3 = 13

Example 19 — Leap year checker

year = 2024

if year % 400 == 0:
    is_leap = True
elif year % 100 == 0:
    is_leap = False
elif year % 4 == 0:
    is_leap = True
else:
    is_leap = False

print(f"{year} is a leap year: {is_leap}")
# Output: 2024 is a leap year: True

Example 20 — FizzBuzz (classic programming exercise)

n = 15

if n % 3 == 0 and n % 5 == 0:
    print("FizzBuzz")
elif n % 3 == 0:
    print("Fizz")
elif n % 5 == 0:
    print("Buzz")
else:
    print(n)

# Output: FizzBuzz

Note: the order matters. The combined check (n % 3 == 0 and n % 5 == 0) must come first. If you checked for n % 3 == 0 first, it would print "Fizz" and skip the FizzBuzz case entirely.

12. Exercises

Practice makes permanent. Try these exercises yourself before looking at the solutions. Type the code out rather than copying and pasting — the muscle memory helps.

Exercise 1: Age group classifier

Write a program that takes an age variable and prints which age group the person belongs to:

  • 0–12: "Child"
  • 13–17: "Teenager"
  • 18–64: "Adult"
  • 65+: "Senior"
Show solution 
age = 30

if age < 0:
    print("Invalid age")
elif age <= 12:
    print("Child")
elif age <= 17:
    print("Teenager")
elif age <= 64:
    print("Adult")
else:
    print("Senior")

# Output: Adult

Exercise 2: Temperature advisor

Given a temp in Celsius, print advice:

  • Below 0: "Freezing! Stay inside."
  • 0–15: "Cold. Wear a jacket."
  • 16–25: "Nice weather. Enjoy!"
  • 26–35: "Warm. Stay hydrated."
  • Above 35: "Dangerously hot!"
Show solution 
temp = 22

if temp < 0:
    print("Freezing! Stay inside.")
elif temp <= 15:
    print("Cold. Wear a jacket.")
elif temp <= 25:
    print("Nice weather. Enjoy!")
elif temp <= 35:
    print("Warm. Stay hydrated.")
else:
    print("Dangerously hot!")

# Output: Nice weather. Enjoy!

Exercise 3: Password strength checker

Given a password string, check its strength:

  • Fewer than 6 characters: "Weak"
  • 6–11 characters: "Medium"
  • 12 or more characters: "Strong"
  • Bonus: if the password contains at least one digit, add " (has number)" to the output.
Show solution 
password = "hello2world"

length = len(password)
has_digit = any(char.isdigit() for char in password)

if length < 6:
    strength = "Weak"
elif length < 12:
    strength = "Medium"
else:
    strength = "Strong"

if has_digit:
    strength += " (has number)"

print(f"Password strength: {strength}")
# Output: Password strength: Medium (has number)

Exercise 4: Ternary practice

Rewrite this code as a single line using a ternary expression: 

x = 42
if x % 2 == 0:
    parity = "even"
else:
    parity = "odd"
Show solution 
x = 42
parity = "even" if x % 2 == 0 else "odd"
print(parity)  # Output: even

Summary

  • if checks a condition; the indented block runs only when it is True.
  • elif (not elseif!) provides additional branches.
  • else is the catch-all that runs when no condition matched.
  • Every condition line ends with a colon.
  • Indentation defines the block — there is no end keyword.
  • Use and, or, and not to combine conditions.
  • Ternary syntax: value = a if condition else b.
  • Python values have inherent truthiness — empty collections and zero are falsy.
  • Use == for comparison, not =. Always end with a colon. Indent consistently.
You now have the power of decision-making in your programs! Combined with the data types you have already learned (numbers, strings, lists, dictionaries), conditionals let you write programs that respond intelligently to different inputs. Next up: loops — the ability to repeat actions.
